# BTC 1h K 线预测模型结构说明

**文档流水号**：0051  
**模型版本**：v3.1-high-vol-fusion  
**适用对象**：管理层 / 非算法细节读者  
**数据与标的**：Binance Vision · BTCUSDT · 1h · 近 365 天  
**当前主 checkpoint**：`checkpoints/real_btc_high_vol_top20_focus/stage3_predictor_best_combo.pt`  
**更新日期**：2026-06-16  

---

## 1. 一句话概括

本模型是一个 **三阶段 Transformer 流水线**：先把 K 线自动切成“形态片段”并离散编码，再基于这些形态 token 预测未来 5 根 K 线的涨跌幅度；推理阶段还会用一个小型 MLP 把 5 个不同提前量的预测融合成一根更贴近真实价格的最终曲线。

---

## 2. 为什么要这样设计

| 问题 | 设计思路 |
|------|----------|
| 固定窗口切分无法适应不同行情 | 用因果 MHA **自动学习切分点**（波动突变处优先切分） |
| 连续向量难以表达“形态模式” | 每段用 **VQ 向量量化** 映射到 16 个离散形态码 |
| 多步预测容易滞后、幅度偏小 | Stage 3 联合 IC/MSE/方向损失 + 推理端 **5-lead MLP 融合** |
| 高波动段更重要 | 训练时对 **高波动样本加权**（top 20% 重点学习） |

---

## 3. 整体架构（三阶段）

```text
┌─────────────────────────────────────────────────────────────────────────┐
│  输入：128 根 1h K 线（每根 17 维因果特征）                                │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                    ┌───────────────┴───────────────┐
                    ▼                               │
         ┌──────────────────────┐                  │
         │  Stage 1 / 2         │                  │
         │  AutoSegment VQ-VAE  │                  │
         │  · 自动切分 MHA       │                  │
         │  · 片段 MHA + CNN     │                  │
         │  · VQ 码本 (16 codes) │                  │
         └──────────┬───────────┘                  │
                    │ 形态 token 序列 [≤16 段]      │
                    ▼                               │
         ┌──────────────────────┐                  │
         │  Stage 3             │                  │
         │  KlinePatternPredictor                  │
         │  · Causal Transformer                   │
         │  · 注意力池化 + 旁路特征                 │
         │  · 输出未来 5 步 log_ret                 │
         └──────────┬───────────┘                  │
                    │                               │
                    ▼                               │
         ┌──────────────────────┐                  │
         │  推理后处理（非训练）  │                  │
         │  · 残差校准           │                  │
         │  · 5-lead MLP 融合    │◄── 每根 K 线收集 │
         │  · 还原为 close 价格  │    lead1..5 预测 │
         └──────────┬───────────┘                  │
                    ▼                               │
              最终预测 close 曲线                   │
```

**训练顺序**：Stage 1 → Stage 2 → Stage 3（Stage 3 可冻结 Stage 1/2 权重，只训预测头）。  
**推理顺序**：加载 Stage 2 embedding + Stage 3 预测器 → 滚动窗口逐步预测 → MLP 融合。

## 4. 输入数据与特征

### 4.1 数据来源

- 来源：Binance Vision 公开 1h K 线（OHLCV）
- 切分：按时间顺序 70% 训练 / 15% 验证 / 15% 测试（**严格因果，无未来泄漏**）
- 标准化：每列做 **trailing 120 bar z-score**（仅使用当前及历史）

### 4.2 单根 K 线特征（共 17 维）

**基础形态 5 维**

| 特征 | 含义 |
|------|------|
| log_ret | 对数收益率 |
| body_ratio | 实体占振幅比例 |
| upper_wick | 上影线比例 |
| lower_wick | 下影线比例 |
| log_vol | 成交量对数 |

**多尺度趋势 12 维**（窗口 20 / 60 / 120 bar，每窗口 4 项）

| 指标 | 含义 |
|------|------|
| slope | log 价格线性回归斜率 |
| resid_std | 回归残差标准差 |
| r² | 拟合优度 |
| strength | \|slope\| / resid_std，趋势强度 |

### 4.3 预测目标

- **主任务**：未来 **5 根** 1h K 线的 **log_ret**（对数收益率）
- **辅助任务**（训练期）：累计涨跌方向、末段形态码监督、VQ 重建、切分正则等

## 5. Stage 1/2：自动切分 + 形态编码（Embedding 层）

模块：`AutoSegmentVQEncoder`（`transformer_kit/auto_segment_encoder.py`）

### 5.1 自动切分（BarSequenceSegmentingMHA）

- 对 128 根 K 线做 **因果 Transformer** 编码
- 每个 bar 位置输出“是否在此切分”的 logit
- 训练时用 Gumbel-Softmax 可微近似；推理时用硬切分
- 约束：单段 4~32 bar，最多 16 段；波动率高的位置有 **伪标签监督** 鼓励切分

### 5.2 片段编码（双分支）

每个切分片段独立编码为 256 维向量：

| 分支 | 结构 | 作用 |
|------|------|------|
| MHA 分支 | 2 层 Segment Transformer + 1 个 summary token | 捕获段内时序依赖 |
| CNN 分支 | 1D 卷积（可选，默认开启） | 捕获局部形态纹理 |
| 融合 | LayerNorm(MHA + CNN) | 两路特征相加 |

### 5.3 向量量化（VQ）

- 码本大小：**16** 个离散形态码
- 更新方式：EMA 码本 + dead-code / dominant-code **再平衡**
- 防坍缩：可微熵惩罚、segment 向量方差惩罚、逆频率 EMA 更新
- 输出：每段一个 **形态 token**（256 维），最多 16 个 token 组成序列

### 5.4 Stage 1 vs Stage 2

| 阶段 | 训练目标 |
|------|----------|
| Stage 1 | 切分 + VQ + 片段重建（学习有意义的形态离散化） |
| Stage 2 | 在 Stage 1 基础上继续优化 VQ 重建与码本多样性 |

## 6. Stage 3：未来收益预测（KlinePatternPredictor）

模块：`KlinePatternPredictor`（`transformer_kit/pattern_model.py`）

### 6.1 主干结构

```text
形态 token 序列 [B, S≤16, 256]
        + 段位置 embedding
        ↓
Causal Transformer（2 层, 4 heads, d_model=256）
        ↓
Segment Attention Pool（可学习 query 汇聚全序列）
        + Raw Context 旁路（17~27 维手工摘要特征 → MLP → 256 维）
        ↓
Future Head：MLP → [B, 5, 1]  （未来 5 步 log_ret）
```

**Raw Context 旁路**：从输入窗口直接提取最近 1/3/6/12/24/48 bar 累计收益、波动率、末根形态等，与 Transformer 输出 **相加**，避免信息在 token 化过程中丢失。

### 6.2 训练损失（联合优化）

Stage 3 不是单纯 MSE，而是多目标加权：

| 损失项 | 作用 |
|--------|------|
| 逐步 MSE | 逐步 log_ret 回归 |
| 逐步 IC (1-corr) | 提升逐步预测与真实的相关性 |
| 累计 IC | 提升 5 步累计收益排序能力 |
| 符号一致损失 | 逐步涨跌方向对齐 |
| 成对排序损失 | batch 内截面排序（可选） |
| 方向分类头 | 未来累计涨跌二分类 |
| 反滞后损失 | 惩罚“预测像过去走势复制” |
| 高波动样本加权 | top 20% 波动样本损失 × 更高权重 |

当前版本 checkpoint 名称 `high_vol_top20_focus` 即表示 **重点优化高波动区间** 的预测表现。

### 6.3 关键超参数（默认值）

| 参数 | 值 |
|------|-----|
| context_bars | 128 |
| pred_horizon | 5 |
| d_model | 256 |
| n_heads | 4 |
| trunk_layers | 2 |
| num_codes | 16 |
| max_segments | 16 |
| pool_mode | attention |

## 7. 推理后处理：滚动预测 + MLP 融合

> 以下模块 **不参与主模型训练**，在回测/可视化脚本中单独拟合（`examples/plot_backtest_candlestick_predictions.py`）。

### 7.1 滚动预测

- 输入窗口每次 **前进 1 根 K 线**
- 每个 anchor 点输出未来 5 步预测 close
- 因此每根目标 K 线会收到来自 **5 个不同 anchor** 的预测（lead1~lead5）

### 7.2 残差校准

- 在 train+valid 上拟合逐步 affine / 残差校准
- 修正模型输出的系统性幅度偏差

### 7.3 5-input MLP 融合器

```text
对每根目标 K 线 t：
  输入 x = [lead1_pred_ret, lead2_pred_ret, ..., lead5_pred_ret]  （5 维）
  输出 y = 融合后的真实 log_ret(t)
  还原 close(t) = close(t-1) × exp(y)
```

| 项目 | 配置 |
|------|------|
| 网络结构 | Linear(5→16) → GELU → Linear(16→16) → GELU → Linear(16→1) |
| 训练数据 | 仅 train / valid split |
| 数据增强 | 以真实值为锚，随机叠加预测偏差（prob=0.3），提升鲁棒性 |
| 早停 | 以 valid MSE 选最优权重 |

## 8. 当前实验表现（测试区间快照）

> 指标来自 `reports/0030_btc_backtest_candlestick/metrics.txt`，高波动 anchor 可视化窗口。

| 指标 | 固定 lead=5 | 混合 lead 均值 | **MLP 融合** |
|------|-------------|----------------|--------------|
| MAE (USDT) | 679.4 | 468.9 | **330.4** |
| 方向准确率 | 67.2% | — | **56.8%** |

**解读**：

- MLP 融合显著 **降低价格幅度误差**（MAE 从 679 → 330），预测曲线更贴近真实 K 线。
- 方向准确率仍有限（约 57%），说明模型对“涨跌方向”的判别能力仍需继续优化。
- VQ 码本仍存在 **少数码主导** 的现象（norm_entropy 偏低），形态离散化多样性是已知短板。

## 11. 已知限制与后续方向

1. **VQ 码本坍缩**：16 码中仅少数活跃，形态表达力受限 → 可减码数或加强 spread/entropy 约束。  
2. **多步预测滞后**：逐步 lead 预测仍偏“跟随历史” → 已加 anti-lag 损失与 horizon-head 选项，可继续调参。  
3. **方向准确率不足 60%**：当前优化重心在幅度拟合；可加大 direction-weight / 逐步方向序列头权重。  
4. **多任务市场状态模型**：代码库已具备 `MarketStateHead`（方向/收益/波动/风险四头），处于实验阶段（report 0049/0050），尚未替代当前主路径。
