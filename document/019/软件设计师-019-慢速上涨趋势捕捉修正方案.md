# 慢速上涨趋势捕捉修正方案

## 1. 文档说明

| 项 | 内容 |
|----|------|
| 角色 | 软件设计师 |
| 流水号 | 019 |
| 主题 | 修复缓慢上升长段开多机会浪费严重的问题 |
| 依据 | `document/016/软件设计师-016-趋势输出模块设计与集成方案.md`、`document/018/软件设计师-018-长趋势收益捕捉修正方案.md` |
| 当前配置参考 | `prod/v0.0.0/config/trading_rule.json` |
| 目标读者 | 程序员、模型训练师、回测师 |

本文启动 019 项目。目标是补齐当前交易系统对“慢速上涨长趋势”的识别、开多、持有与升级能力。019 不推翻 015/018 的崩跌与长趋势持仓设计，而是新增低波动上涨趋势状态机，解决系统长期偏短线、偏下跌补救、慢涨吃不到的问题。

---

## 2. 问题描述

当前系统已经能处理：

```text
短线模型开仓；
模型失效下的 crash short；
部分趋势持仓延长；
best-point 观察信号。
```

但对缓慢上涨长段仍浪费严重，表现为：

```text
慢涨区间内长期 FLAT；
或开多后很快因 max_hold_bars / reverse / pred_cum_ret_5 转弱退出；
趋势模块可能识别为 UP，但交易规则没有将其转化为 LONG_TREND；
best-point 标签偏短，容易提示短线 exit，而非长持有。
```

核心判断：

```text
慢速上涨不是 surge，也不是短线强多；
它是低波动、低斜率、持续性强的趋势状态。
```

因此不能继续只调：

```text
open_edge_threshold
p_up threshold
pred_cum_ret_5 threshold
```

而应新增：

```text
SLOW_UP_TREND
```

---

## 3. 金融定义

慢速上涨趋势定义：

```text
价格在较长窗口中缓慢抬升；
回撤较浅；
大多数 K 线维持在中短期均线上方；
没有明显加速暴涨；
风险概率不高；
短期模型信号可能不强，但价格结构持续改善。
```

它与普通强上涨不同：

| 类型 | 特征 | 交易含义 |
|------|------|----------|
| SURGE | 短时间大幅拉升、波动扩张 | 追涨风险高，需事件模式 |
| NORMAL_UP_TREND | 方向和强度都明确 | 可趋势持仓 |
| SLOW_UP_TREND | 斜率不大但持续性强 | 小仓位建仓、耐心持有 |
| RANGE | 均线缠绕、无方向 | 不交易或短线 |

019 重点是 `SLOW_UP_TREND`。

class SlowTrendContext:

## 5. 慢涨判定规则

### 5.1 均线结构

默认：

```text
ema_fast = 12
ema_mid = 24
ema_slow = 72
```

慢涨基础条件：

```text
close > ema_fast
ema_fast > ema_mid
ema_mid >= ema_slow 或 ema_mid 正在上穿 ema_slow
```

允许：

```text
ema_mid 与 ema_slow 距离不大；
因为慢涨初期长期均线可能滞后。
```

### 5.2 斜率

慢涨不是爆发，使用长窗口斜率：

```text
slope_24_atr = (close[t] - close[t-24]) / atr[t]
slope_48_atr = (close[t] - close[t-48]) / atr[t]
```

判定：

```text
slope_24_atr >= 1.0
或 slope_48_atr >= 1.8
```

强慢涨：

```text
slope_24_atr >= 1.8
且 slope_48_atr >= 2.5
```

### 5.3 持续性

```text
persistence_above_ema_fast = 过去 24 根 close > ema_fast 的比例
persistence_above_ema_mid = 过去 24 根 close > ema_mid 的比例
```

慢涨要求：

```text
persistence_above_ema_fast >= 0.62
persistence_above_ema_mid >= 0.55
```

稳定慢涨：

```text
persistence_above_ema_fast >= 0.75
persistence_above_ema_mid >= 0.67
```

### 5.4 回撤不破坏结构

定义：

```text
pullback_depth_atr = max(0, ema_fast - close) / atr
max_drawdown_24_atr = (rolling_high_24 - close) / atr
```

慢涨允许小回撤：

```text
close 可短暂低于 ema_fast；
但不能连续跌破 ema_mid；
max_drawdown_24_atr 不应过大。
```

默认：

```text
pullback_depth_atr <= 0.8
max_drawdown_24_atr <= 3.0
```

### 5.5 排除横盘

慢涨不能等同横盘。若：

```text
slope_48_atr < 1.0
且 rolling_high_48 / rolling_low_48 振幅很小
```

则判定为：

```text
RANGE
```

不允许开多。

## 6. 慢涨打分

```text
close > ema_fast                              +1
ema_fast > ema_mid                            +1
ema_mid >= ema_slow 或 ema_mid 上行           +1
slope_24_atr >= 1.0                           +1
slope_48_atr >= 1.8                           +1
persistence_above_ema_fast >= 0.62            +1
persistence_above_ema_mid >= 0.55             +1
max_drawdown_24_atr <= 3.0                    +1
p_risk <= 0.45                                +1
```

判定：

```text
score >= 5 -> SLOW_UP_TREND
score >= 7 -> STABLE_SLOW_UP_TREND
```

注意：

```text
p_flat 高不直接否决慢涨；
慢涨常被短期模型识别成 flat。
```

但若：

```text
p_flat >= 0.50
且 slope_48_atr < 1.8
```

仍视为横盘，不开仓。

## 7. 新增交易行为

### 7.1 慢涨观察

空仓时，若：

```text
is_slow_uptrend == true
但 0062e OPEN_LONG_SIGNAL 未触发
```

先输出：

```text
WATCH_SLOW_UPTREND
```

观察不下单，只记录：

```text
slow_up_score
missed_slow_uptrend_bar
```

### 7.2 慢涨试探多

当满足：

```text
position.side == FLAT
is_stable_slow_uptrend == true
p_risk <= slow_up_risk_max
drawdown 未触发账户防守
非 crash/downtrend
```

允许：

```text
OPEN_LONG_SLOW_TREND
```

不要求：

```text
edge >= open_edge_threshold
p_up >= open_prob_threshold
pred_cum_ret_5 很强
```

但要求模型没有强反对：

```text
NOT (
  pred_cum_ret_5 < -0.10
  or edge <= -0.04
  or p_risk >= risk_exit_threshold
)
```

仓位：

```text
position_ratio = 0.04
stable_slow_uptrend 可到 0.06
max_slow_up_position_ratio = 0.08
```

### 7.3 慢涨仓升级为趋势仓

若慢涨多头持仓后：

```text
profit_atr >= 0.8
slow_uptrend 仍成立
trend_signal.direction == UP 或 slow_context.is_stable_slow_uptrend
p_risk < risk_exit_threshold
```

动作：

```text
UPGRADE_SLOW_LONG_TO_TREND
hold_mode = TREND
lifecycle = TREND
```

### 7.4 慢涨趋势继续持有

只要：

```text
is_slow_uptrend == true
max_drawdown_24_atr <= slow_up_exit_drawdown_atr
p_risk < risk_exit_threshold
```

则：

```text
HOLD_SLOW_UPTREND
```

不因单根：

```text
pred_cum_ret_5 <= 0
edge 小幅转负
p_flat 偏高
```

直接退出。

## 8. 慢涨退出规则

### 8.1 慢涨破坏

退出投票：

```text
close < ema_mid 连续 2 根                  +2
slope_24_atr < 0                           +1
ret_6_atr <= -1.2                          +1
max_drawdown_24_atr > 3.5                  +1
p_risk >= risk_exit_threshold              +2
0062e pred_cum_ret_5 < -0.10 连续 2 根      +1
```

触发：

```text
exit_votes >= 3
```

动作：

```text
CLOSE_SLOW_UPTREND_BROKEN
```

### 8.2 慢涨止盈

慢涨不应使用过紧 trailing。

分层：

```text
profit_atr < 1.5:
  保持正常止损；

1.5 <= profit_atr < 3.0:
  止损抬到接近保本；

profit_atr >= 3.0:
  只在 slow_uptrend 破坏后退出，不因普通 exit 信号直接平仓；

profit_atr >= 5.0:
  可减仓 30%，保留 runner。
```

新增动作：

```text
REDUCE_SLOW_UP_PROFIT_LOCK
HOLD_SLOW_UP_RUNNER
```

## 9. 与现有模块的关系

### 9.1 与 0062e

0062e 仍是主模型。

慢涨规则只在：

```text
0062e 没有强反对；
价格结构持续向上；
风险没有恶化；
```

时补充开多和持有。

### 9.2 与 016 TrendSignalProvider

016 负责通用趋势方向和阶段。

019 负责：

```text
低斜率慢涨；
允许 p_flat 较高；
不要求 acceleration；
更重视 persistence。
```

### 9.3 与 017 best-point

017 best-point 模型第一版不用于慢涨退出。

可用方式：

```text
p_hold_long 高 -> 增强慢涨持有信心；
p_exit_long 高 -> 只作为退出投票的一票；
p_long_entry_zone 高 -> 可降低 WATCH -> OPEN 的等待时间。
```

禁止：

```text
p_exit_long 单独平掉慢涨趋势仓。
```

### 9.4 与 015/018 crash

慢涨规则不得在以下状态触发：

```text
is_crash == true
trend_direction == DOWN 且 confirmed
account defensive mode
```

```text

```

过滤条件：

```text
direction == long
holding_bars >= 8
raw_return > 0
net_roi >= 0.10
max adverse excursion 不超过该笔收益的 50%
```

标签：

```text
slow_long_entry_zone

slow_long_exit_zone
```

用途：

```text
第一阶段不训练新模型；
先用标签评估 SlowUptrendDetector 的 recall/precision。
```

## 14. 实施顺序

### 阶段一：只观察，不交易

实现：

```text
SlowUptrendDetector
日志字段
missed_slow_uptrend_bars 指标
```

验收：

```text
慢涨段能被标出来；
横盘段误报不过高。
```

### 阶段二：小仓位慢涨开多

实现：

```text
OPEN_LONG_SLOW_TREND
WATCH_SLOW_UPTREND
```

仓位不超过 4%~6%。

### 阶段三：慢涨趋势升级与持有

实现：

```text
UPGRADE_SLOW_LONG_TO_TREND
HOLD_SLOW_UPTREND
CLOSE_SLOW_UPTREND_BROKEN
```

重点观察平均持仓是否提升。

### 阶段四：runner 与分层止盈

实现：

```text
REDUCE_SLOW_UP_PROFIT_LOCK
HOLD_SLOW_UP_RUNNER
```

## 15. 不建议做的事

不建议：

```text
直接降低普通 OPEN_LONG_SIGNAL 阈值；
取消 p_flat 过滤但不加慢涨状态；
让 best-point exit 单独平慢涨仓；
慢涨一识别就满仓；
在 downtrend/crash 中触发慢涨开多；
用短线最佳点标签训练慢涨模型。
```

原因：

```text
这些做法要么增加噪声交易，
要么重回短线系统，
要么与 crash/downtrend 防守逻辑冲突。
```

---

## 16. 一句话结论

019 的核心是：

```text
为低波动、低斜率、持续性上涨单独建模；
用 SLOW_UP_TREND 补齐当前系统只擅长短线和下跌补救的短板；
小仓位进入，趋势确认后升级，趋势破坏后退出。
```
