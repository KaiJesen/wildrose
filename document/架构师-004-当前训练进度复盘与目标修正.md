# 当前训练进度复盘与模型训练目标修正

## 1. 文档目标

本文档基于当前已完成的 `0051` 至 `0058` 训练结果，判断模型训练进度、训练师需要重点关注的问题，并修正此前《架构师-002-模型指标训练指导》和《架构师-003-理想模型指标目标指导》中的阶段性训练目标。

本文只讨论模型训练、指标判断与训练目标，不涉及交易执行、仓位、下单或策略规则。

## 2. 当前训练进度判断

最新可用报告已推进到：

```text
reports/0058_market_state_usable
```

`0058` 测试集关键指标：

```text
direction_acc        = 31.22%
direction_macro_f1   = 0.308
cum_direction_acc    = 60.81%
return_ic            = 0.054
return_mae           = 0.03569
volatility_mae       = 0.08810
risk_f1              = 0.530
risk_positive_true   = 23.24%
risk_positive_pred   = 46.08%
```

`0058` 相比 `0050` 的进展：

```text
cum_direction_acc    : 55.41% -> 60.81%
direction_macro_f1   : 0.269  -> 0.308
return_ic            : 0.073  -> 0.054
volatility_mae       : 0.085  -> 0.088
risk_f1              : 0.455  -> 0.530
```

结论：

1. `0058` 已达到此前定义的“可用模型”门槛。
2. 当前训练目标应从“冲过可用线”转为“稳定可用并向成熟模型靠拢”。
3. 不应继续单纯追求更高 `cum_direction_acc`，否则可能牺牲方向类别均衡与收益/波动率质量。
4. 当前仍不建议扩大模型主干，应优先修正指标偏科、类别分布偏差和选模准则。

## 3. 是否需要修正之前的训练目标指导

需要修正，但不是推翻。

### 3.1 保留的判断

此前文档中的以下判断仍然成立：

- 多任务市场状态模型方向正确。
- 当前不应优先扩大模型规模。
- 训练必须保持真实数据、严格时间切分和 train-only 阈值估计。
- `loss` 不能作为单独选模依据。
- 需要同时看 direction、return、volatility、risk 四类指标。

### 3.2 需要修正的内容

此前 `架构师-002` 主要目标是：

```text
cum_direction_acc    >= 56%
direction_macro_f1   >= 0.30
risk_f1              >= 0.48
return_ic            > 0
volatility_mae       <= 0.10
```

该目标已被 `0058` 满足。因此后续训练不应继续把这些作为主攻目标，而应改为更高一级的“稳定可用目标”。

此前 `架构师-003` 中的成熟目标仍可作为中期目标，但需要拆成两个阶段：

1. 稳定可用阶段：先确认 `0058` 的提升不是偶然。
2. 成熟过渡阶段：再提高 `direction_macro_f1`、`return_ic` 与分布健康度。

## 4. 当前训练师必须关注的问题

### 4.1 `0058` 虽然可用，但存在方向类别偏置

`0058` 的 direction 类别分布：

```text
true down = 38.78%, pred down = 18.92%
true flat = 25.68%, pred flat = 50.81%
true up   = 35.54%, pred up   = 30.27%
```

问题：

- 模型明显高估 flat 类。
- down 类预测不足。
- `cum_direction_acc` 很高，但逐步三分类质量并不充分。

训练师需要重点关注：

- direction confusion matrix。
- down 类 recall。
- flat 类 precision。
- `direction_macro_f1` 是否继续提升。

### 4.2 risk 任务从低召回转为偏过度预测

`0058` 的 risk 分布：

```text
risk_positive_true = 23.24%
risk_positive_pred = 46.08%
```

问题：

- `risk_f1=0.530` 已明显改善。
- 但预测风险正样本比例接近真实值的 2 倍。
- 后续如果继续提高 risk 权重，可能造成 risk 全面偏正。

训练师需要关注：

- risk precision / recall 拆分指标。
- risk positive rate ratio。
- risk confusion matrix。
- risk head 是否过度敏感。

### 4.3 `return_ic` 仍未达到成熟目标

`0058`：

```text
return_ic = 0.054
```

虽然为正，但低于此前成熟目标 `0.08`，也低于 `0050` 的 `0.073`。

问题：

- 当前 `cum_direction_acc` 和 `risk_f1` 提升，可能部分牺牲了收益排序能力。
- per-horizon return IC 不均衡：

```text
h1 = -0.024
h2 = -0.049
h3 = -0.037
h4 =  0.068
h5 =  0.120
```

训练师需要关注：

- 是否模型只在后两个 horizon 有收益排序能力。
- 是否需要 horizon-level 选模或 horizon-specific loss weight。
- 不要让分类任务完全压制 return head。

### 4.4 `volatility_mae` 没有继续改善

`0058`：

```text
volatility_mae = 0.088
```

该指标仍满足可用门槛，但未达到成熟目标 `<= 0.075`。因此下一阶段不应继续降低 `volatility_weight`，否则波动率头可能继续退化。

## 6. 新训练目标

### 6.1 下一轮最低目标

下一轮建议命名为：

```text
0059_market_state_balanced_mature
```

最低目标：

```text
cum_direction_acc    >= 58%
direction_macro_f1   >= 0.33
return_ic            >= 0.05
volatility_mae       <= 0.085
risk_f1              >= 0.52
```

分布约束：

```text
risk_positive_rate_pred / risk_positive_rate_true <= 1.7
direction_pred_flat <= 42%
direction_pred_down >= 25%
```

说明：

- `cum_direction_acc` 可以从 `60.8%` 回落到 `58%`，但必须换来更均衡的方向分类与更稳定的 return IC。
- `direction_macro_f1` 必须从 `0.308` 提升到至少 `0.33`。
- `risk_f1` 不要求继续大幅提升，但风险正样本预测比例必须收敛。

### 6.2 成熟过渡目标

如果 `0059` 达标，则后续 `0060` 的目标为：

```text
cum_direction_acc    >= 59%
direction_macro_f1   >= 0.35
return_ic            >= 0.07
volatility_mae       <= 0.080
risk_f1              >= 0.53
```

分布约束：

```text
risk_positive_rate_pred / risk_positive_rate_true in [0.8, 1.6]
direction_pred_down / direction_true_down >= 0.70
direction_pred_flat / direction_true_flat <= 1.60
```

### 6.3 暂缓追求的目标

以下目标暂时不作为下一轮硬目标：

```text
cum_direction_acc >= 63%
return_ic >= 0.10
risk_f1 >= 0.58
```

原因：

- 当前单轮 `cum_direction_acc` 已较高，但类别均衡不足。
- 过早追求更高 risk F1 可能进一步造成 risk 正类过度预测。
- return IC 需要先恢复到 `0.07` 附近，再讨论 `0.10`。

## 7. 选模分数修正

此前 score 仍偏向 `cum_direction_acc`，导致 `0058` 这类“累计方向很强但类别偏 flat”的模型容易被选中。

建议 `0059` 使用新的 balanced score：

```text
score_0059 =
    0.25 * cum_direction_acc
  + 0.25 * direction_macro_f1
  + 0.20 * clipped_return_ic
  + 0.15 * risk_f1
  + 0.10 * volatility_quality
  + 0.05 * distribution_health
```

其中：

```text
clipped_return_ic = clip(return_ic / 0.10, -1.0, 1.0)
volatility_quality = 1 - clip(volatility_mae / 0.12, 0.0, 1.0)
```

`distribution_health` 建议定义为：

```text
distribution_health =
  1
  - abs(risk_positive_rate_pred / risk_positive_rate_true - 1) * 0.3
  - abs(direction_pred_flat - direction_true_flat) * 1.0
  - max(0, direction_true_down - direction_pred_down) * 0.8
```

如果暂时不实现完整公式，至少加入硬过滤：

```text
reject if risk_positive_rate_pred / risk_positive_rate_true > 1.8
reject if direction_pred_flat > 45%
reject if return_ic <= 0
```

## 8. 建议训练配置

### 8.1 不建议继续增加 risk 权重

`0058` 已出现 risk 正类预测偏高，因此不建议下一轮继续提高 `risk_weight`。

建议：

```text
return_weight     = 0.35
direction_weight  = 0.45
volatility_weight = 0.12
risk_weight       = 0.08
```

相比上一阶段：

- 略提高 return，恢复收益排序能力。
- 保持 direction 为主任务。
- 略提高 volatility，避免波动率头退化。
- 略降低 risk，控制过度预测。

### 8.2 标签阈值不要继续降低

当前方向/risk 标签已足够密集。下一轮不建议继续降低阈值。

建议保持：

```text
direction_threshold_quantile = 0.25
risk_threshold_quantile      = 0.70
```

如需对比，只允许做单独实验：

```text
direction_threshold_quantile = 0.30
risk_threshold_quantile      = 0.70
```

目的：

- 观察 flat 类是否因方向阈值过低/过高产生偏置。
- 不要同时改 risk 阈值，避免无法归因。

### 8.3 类别权重需要重新校准

如果当前使用 class weights 或 focal loss，下一轮重点不是简单加大，而是检查：

- down 类是否权重不足。
- flat 类是否被过度奖励。
- risk 正类是否权重过高。

建议方向：

```text
direction:
  down_weight: 适度上调
  flat_weight: 适度下调
  up_weight: 保持或小幅调整

risk:
  positive_weight: 下调或限制上限
```

具体数值必须由 train split 类别分布估计，不能手工针对 test 调参。

### 8.4 训练轮数

建议：

```text
epochs = 40 - 60
```

但选模必须使用 balanced score，不再使用旧 score。

如果 valid 上出现：

```text
return_ic 连续 5 个评估点为负
或 risk_positive_pred > 50%
或 direction_pred_flat > 50%
```

则该训练应提前停止并标记为偏科，不必跑满。

## 10. 是否需要修改模型结构

当前不建议修改主干结构。

理由：

- `0058` 已达到可用模型门槛。
- 当前问题主要是任务平衡和类别分布，不是模型完全无法学习。
- 多轮结果显示不同配置能分别提升 `risk_f1`、`direction_macro_f1` 和 `cum_direction_acc`，说明训练策略仍有优化空间。

只有当以下情况连续出现，才进入结构改造：

```text
连续 2 轮 balanced 训练后：
direction_macro_f1 < 0.33
或 return_ic <= 0
或 risk_positive_pred 无法控制在 1.8 倍以内
```

可考虑的结构改造：

- direction/risk 专用轻量 decoder。
- horizon-aware direction head。
- risk head 单独校准层。
- CPC encoder 与 MarketStateHead 的同口径对比。

## 11. 新目标结论

此前训练目标“达到可用模型”已经由 `0058` 基本完成。新的训练目标应修正为：

```text
保持 cum_direction_acc >= 58%
提升 direction_macro_f1 到 >= 0.33
保持 return_ic >= 0.05 并向 0.07 回升
保持 risk_f1 >= 0.52 但控制 risk 正类过度预测
将 volatility_mae 控制在 <= 0.085
```

训练师当前最需要关注的不是继续拉高单一指标，而是让模型从“可用但偏科”进入“稳定、均衡、可解释”的成熟过渡阶段。
