# 0059 训练复盘与 0060 训练目标指导

## 1. 文档目标

本文档基于 `reports/0059_market_state_balanced_mature`、`reports/0059b_market_state_balanced_mature`、`reports/0059c_market_state_balanced_mature` 三轮训练报告，对照《架构师-004-当前训练进度复盘与目标修正》判断目标完成情况，并制定下一步 `0060` 训练目标。

本文只讨论模型训练、指标和选模目标，不涉及交易执行、仓位、下单或策略规则。

## 2. 004 号文档目标回顾

`架构师-004` 对 `0059` 的最低目标是：

```text
cum_direction_acc    >= 58%
direction_macro_f1   >= 0.33
return_ic            >= 0.05
volatility_mae       <= 0.085
risk_f1              >= 0.52
```

分布约束：

```text
risk_positive_rate_pred / risk_positive_rate_true <= 1.7
direction_pred_flat <= 42%
direction_pred_down >= 25%
```

阶段定位：

```text
稳定可用模型，并向成熟模型过渡
```

## 4. 0059 完成情况结论

`0059` 系列没有完整达成 `架构师-004` 的 0059 目标。

逐项判断：

```text
cum_direction_acc >= 58%     : 仅 0059b 达成，但综合质量差
direction_macro_f1 >= 0.33   : 0059 / 0059c 达成
return_ic >= 0.05            : 三组均未达成
volatility_mae <= 0.085      : 0059 / 0059c 达成
risk_f1 >= 0.52              : 0059 / 0059c 达成
分布约束                      : 0059 / 0059c 达成，0059b 方向分布偏 down
```

核心结论：

1. `0059c` 解决了 `0058` 的 flat 过度预测问题。
2. `0059c` 也缓解了 risk 正类过度预测问题。
3. 但 `0059c` 没能继承 `0058` 的累计方向优势。
4. `return_ic` 成为当前最稳定的短板，三组均低于 `0.05`。
5. 下一轮不应再大幅调整 risk 或 direction 阈值，应优先恢复 `cum_direction_acc` 与 `return_ic`。

## 5. 当前训练师需要关注的内容

### 5.1 关注 `cum_direction_acc` 与 `direction_macro_f1` 的拉扯

`0058`：

```text
cum_direction_acc  = 60.81%
direction_macro_f1 = 0.308
```

`0059c`：

```text
cum_direction_acc  = 56.08%
direction_macro_f1 = 0.341
```

说明：

- `0059c` 明显改善了逐步三分类质量。
- 但累计方向优势被削弱。

训练师下一步要做的是恢复累计方向，而不是回退到 `0058` 的 flat 偏置。

### 5.2 关注 `return_ic` 未达标

三组 `0059`：

```text
0059  return_ic = 0.033
0059b return_ic = 0.048
0059c return_ic = 0.038
```

均未达到 `0.05`。这说明当前权重和选模方式仍没有充分保护收益排序任务。

需要重点检查：

- valid 选模时是否过度偏向分布健康和风险任务。
- return head 是否被 direction/risk 分类头压制。
- per-horizon return IC 是否仍主要集中在 h4/h5。

### 5.3 关注 risk 不要再过度增强

`0059c`：

```text
risk_f1 = 0.542
risk_positive_pred = 29.73%
risk_positive_true = 23.24%
```

这是比 `0058` 更健康的风险预测分布。因此 `0060` 不应继续提高 risk 权重，也不应降低 risk 阈值。

### 5.4 关注 0059c 的训练时长

`0059c` 使用 `epochs=25`，比 `0059`/`0059b` 更短，但综合更均衡。说明长训未必带来成熟，可能会使模型向某些任务漂移。

下一轮需要增加早停和硬过滤，而不是简单拉长训练。

```text

恢复 cum_direction_acc，
提升 return_ic。
```

cum_direction_acc    >= 58.0%
direction_macro_f1   >= 0.34
return_ic            >= 0.05
volatility_mae       <= 0.070

risk_positive_rate_pred / risk_positive_rate_true in [0.9, 1.5]

- `risk_f1 >= 0.53` 是继承 `0059c` 风险识别能力的底线。

## 7. 0060 训练策略建议

### 7.1 继承 0059c，而不是 0058 或 0059b

推荐以 `0059c` 为基线，因为它的分布最健康：

```text
direction_pred_down  = 37.84%
direction_pred_flat  = 31.76%
direction_pred_up    = 30.41%
risk_positive_pred   = 29.73%
```

不建议继承 `0058`：

- flat 预测过高。
- risk 正类预测过高。

不建议继承 `0059b`：

- return/volatility 明显退化。
- direction 上偏 down，risk 不达标。

### 7.2 权重微调

`0059c` 的配置：

```text
return/direction/volatility/risk = 0.35/0.45/0.12/0.08
cum_direction_weight = 0.035
```

建议 `0060`：

```text
return_weight       = 0.38
direction_weight    = 0.42
volatility_weight   = 0.12
risk_weight         = 0.08
cum_direction_weight= 0.045
```

动机：

- 小幅提高 return，保护 `return_ic`。
- 小幅提高 cumulative direction，恢复 `cum_direction_acc`。
- 不再提高 risk，避免正类预测比例再次上升。
- 保持 volatility，保留 `0059c` 的波动率改善。

### 7.3 阈值保持不变

建议保持：

```text
direction_threshold_quantile = 0.25
risk_threshold_quantile      = 0.70
```

原因：

- `0059c` 的类别分布已经明显健康。
- 当前短板不再是标签密度，而是累计方向和收益排序。

如需单独对比，只允许一个副实验：

```text
direction_threshold_quantile = 0.30
risk_threshold_quantile      = 0.70
```

该副实验只用于判断 flat/up/down 边界，不作为主线。

### 7.4 选模准则修正

`0060` 不能只使用 balanced score。需要加入 hard gates：

```text
hard_gates:
  cum_direction_acc >= 56.5% on valid
  return_ic >= 0.02 on valid
  direction_macro_f1 >= 0.33 on valid
  risk_positive_rate_pred / true <= 1.6 on valid
```

测试验收仍按 `0060` 最低目标执行。

建议 score：

```text
score_0060 =
    0.25 * cum_direction_acc
  + 0.22 * direction_macro_f1
  + 0.23 * clipped_return_ic
  + 0.12 * risk_f1
  + 0.10 * volatility_quality
  + 0.08 * distribution_health
```

相比 `0059`：

- 提高 `return_ic` 权重。
- 保留分布健康，但不让分布健康压过核心预测质量。
- 适当提高 cumulative direction 的硬门槛。

### 7.5 训练轮数和早停

建议：

```text
epochs = 35 - 45
```

不建议直接跑满 60 轮。`0059c` 表明短训更均衡，长训可能向某些任务漂移。

早停条件：

```text
valid return_ic 连续 5 次 <= 0
或 valid cum_direction_acc 连续 5 次 < 53%
或 risk_positive_pred / true > 1.8
或 direction_pred_up < 20%
```

## 9. 是否需要修改架构师-004 目标

需要小幅修正。

`架构师-004` 对 `0059` 的目标方向正确，但经过 `0059` 系列验证后，应将下一步目标从：

```text
单纯 balanced_mature
```

修正为：

```text
cum_direction + return_ic recovery under healthy distribution
```

也就是：在健康分布约束下恢复累计方向和收益排序，而不是继续单纯改善分类分布或风险识别。

## 10. 下一步结论

`0059` 系列的训练价值在于确认：

1. `0059c` 解决了 `0058` 的分布偏置。
2. `0059c` 保留了较好的 risk 和 volatility 指标。
3. 当前新瓶颈是 `cum_direction_acc` 和 `return_ic`。

因此 `0060` 的训练目标应明确为：

```text
以 0059c 为基线，
保持 direction_macro_f1 >= 0.34，
保持 risk_f1 >= 0.53，
保持 volatility_mae <= 0.070，
将 cum_direction_acc 恢复到 >= 58%，
将 return_ic 提升到 >= 0.05。
```

如果 `0060` 仍无法同时恢复 `cum_direction_acc` 与 `return_ic`，下一步再考虑 horizon-aware return/cum direction 选模或 direction-return 双目标解耦，而不是继续调整 risk 任务。
