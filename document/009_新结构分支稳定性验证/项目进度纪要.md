## 3. 阶段二裁定（项目经理）

### 3.1 稳定性结论

**裁定：009 阶段一目标达成，新结构分支认定为「有条件稳定」。**

含义：

- 可以对外说：0062e 路线在相邻随机种子上 **站得住**
- 不能说：已全面超越 0058，或已达成熟/理想模型

### 3.2 基线策略（书面决定）

采用 **双基线并存**，职责如下：

| 基线 | 代表实验 | 使用场景 | git 标签 |
|------|----------|----------|----------|
| **主基线（usable）** | 0058 | 需要 step 收益排序、usable 全项目验收、对外默认可用版本 | `v4.0-market-state-usable` |
| **稳定分支候选** | 0062e / 0063_s42 | 累计收益排序、累计方向判断、多任务均衡且不坍缩的研究/集成分支 | `v4.1-cum-return-branch` → 建议增补 **`v4.2-stable-cum-branch`** |

**不替换主基线**：0058 仍为全项目默认 release；0062e 路线 **不得**在未经过项目经理裁定前取代 0058。

**分支候选 checkpoint 推荐**（技术侧一致意见）：

```text
首选：0062e / 0063_s42（等价可复现）
备选：0063_s43（分类与累计方向略优）
```

### 3.3 版本策略

| 动作 | 决定 |
|------|------|
| 维持 `v4.0-market-state-usable` | ✅ |
| 维持 `v4.1-cum-return-branch` | ✅ |
| 新增 `v4.2-stable-cum-branch` | **建议打标**（指向 0063 稳定性验证完成后的 HEAD，与 0062e/0063 报告同批提交） |
| 将 0062e 升格为唯一主版本 | ❌ |

### 3.4 阶段四（0064）资源裁定

**批准进入阶段四**，但仅限：

```text
收益指标恢复探索（0064 系列）
```

约束（继承架构师初审）：

- 不扩大模型主干、不新增复杂结构
- 不以牺牲 0063 已验证的稳定性为代价追 step 收益指标
- 须由架构师下达 0064 任务书后，训练师方可开跑

### 3.5 对外口径（团队统一说法）

1. **0058**：当前 usable **主基线**，step 收益排序更可靠。  
2. **0062e/0063**：新结构 **稳定分支候选**，累计收益与累计方向能力更强，已多 seed 验证。  
3. 两版并存是 **有意设计**，不是「新旧混乱」；选用哪版取决于下游场景（见 `项目经理-009-双轨验收与基线说明.md`）。
