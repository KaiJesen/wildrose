## 2. 为什么有两条基线

008–009 阶段结论：项目同时需要两种能力，**无法由单一 checkpoint 同时最优**。

| 能力侧重 | 当前更合适的选择 |
|----------|------------------|
| 逐步收益排序（usable 传统强项） | **0058** 主基线 |
| 累计收益排序 + 累计方向 + 防坍缩多任务 | **0062e / 0063** 稳定分支候选 |

因此采用 **双基线并存**，不是过渡混乱，而是明确分工。

---

## 3. 两条验收轨（A / B）

训练与报告通过 `target_stage` 自动映射验收轨，产出中带 `acceptance_track` 字段。

| 验收轨 | `target_stage` 典型值 | 对应文档阶段 | 当前代表 |
|--------|----------------------|--------------|----------|
| **A 轨 · usable 主基线** | `usable` | 冲击/维持 usable 全项目验收 | 0058 |
| **A 轨延伸** | `balanced_mature`、`cum_return_recovery` | 成熟过渡、专项恢复 | 0059c、0060 等 |
| **B 轨 · 新结构分支** | `return_direction_branch` | 0061 结构 + 0062 修复及之后 | 0062e、0063 |

**规则**：

- 跑 usable 线实验 → 读 A 轨结论，与 0058 对照。
- 跑新结构分支实验 → 读 B 轨结论，与 0062e/0063 对照。
- **禁止**用 A 轨门槛评判 B 轨实验，反之亦然（避免无效内卷）。

## 4. 选用哪版模型（决策树）

```text
下游需要什么？
│
├─ 对外默认可用 / step 收益排序优先
│     → 0058（v4.0-market-state-usable）
│
├─ 累计收益排序 / 累计方向 / 研究 cum 分支
│     → 0062e 或 0063_s42（v4.1 / 建议 v4.2-stable-cum-branch）
│
└─ 不确定
      → 默认 0058；需 cum 能力时再显式切换分支候选
```

**集成原则**：代码与配置里 **写清楚** 使用的 checkpoint 与 `target_stage`，避免隐式混用。

## 5. 训练与报告流程（阶段三固化）

### 5.1 发起训练

```bash
# B 轨：新结构分支（默认配置已对齐 0062e）
python examples/train_market_state_model.py \
  --target-stage return_direction_branch \
  --seed <SEED> \
  --report-dir reports/<run_id> \
  --checkpoint-dir checkpoints/<run_id>

# A 轨：usable 主基线线
python examples/train_market_state_model.py \
  --target-stage usable \
  ...
```

### 5.2 查看报告

每轮 `reports/<run_id>/` 含：

- `REPORT.md`：含 **验收轨道**、`target_stage`、`decision`
- `metrics.json`：含 `acceptance_track`、`target_stage`、`decision`

### 5.3 多 seed 汇总（009 起）

```bash
python examples/summarize_multi_seed.py \
  reports/0063_multi_seed_s42_market_state_stability \
  reports/0063_multi_seed_s43_market_state_stability \
  reports/0063_multi_seed_s44_market_state_stability \
  -o document/009_新结构分支稳定性验证/0063_multi_seed_stability/metrics_summary.json
```

## 6. 版本标签与含义

| 标签 | 含义 |
|------|------|
| `v3.1-high-vol-fusion` | 主预测线（K 线 + MLP 融合），非市场状态线 |
| `v4.0-market-state-usable` | 市场状态 **usable 主基线**（0058） |
| `v4.1-cum-return-branch` | 新结构分支 **首次候选**（0062e，008 conditional） |
| `v4.2-stable-cum-branch` | 新结构分支 **多 seed 验证后**（009 建议） |
