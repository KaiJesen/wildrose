# 市场状态模型：结构与输入输出说明

> 项目：Wildrose  
> 模型路线：0062e / 0063 稳定分支（累计收益增强型）  
> 决策频率：1 小时 K 线  
> 更新：2026-06  

---

## 一、文档目的

本文说明当前用于预判比特币永续合约（以 BTCUSDT 为例）短期走势的**市场状态模型**：

- 模型由哪些部分组成  
- 输入是什么、输出是什么  
- 数据如何从 K 线流入、又如何变成预测结果  

本文面向需要查阅架构的读者；文中若出现英文缩写，均在括号中给出中文注释。

## 二、模型总体作用

模型在每根 **1 小时 K 线收盘后** 执行一次推断：

- **回看**：过去约 **128 根** K 线（约 5 天多）  
- **前瞻**：未来约 **5 根** K 线（约 5 小时）  

它同时回答四个问题：

1. 未来每一步大概涨还是跌（收益）  
2. 未来每一步更偏向跌、横盘还是涨（方向）  
3. 未来波动有多大（波动率）  
4. 是否处于高风险环境（风险状态）  

模型输出的是**统计意义上的预判**，供后续交易规则使用；**不等于**对下一根 K 线收盘价的精确承诺。

## 三、输入（Input）

### 3.1 原始数据来源

原始数据为交易所 K 线，每条记录包含：

| 字段 | 含义 |
|------|------|
| 开盘价（Open） | 该小时第一笔成交价 |
| 最高价（High） | 该小时最高成交价 |
| 最低价（Low） | 该小时最低成交价 |
| 收盘价（Close） | 该小时最后一笔成交价 |
| 成交量（Volume） | 该小时成交数量 |

### 3.2 特征工程：每根 K 线变成 17 个数字

程序不会直接把原始五价一量丢进网络，而是先转为 **17 维特征向量**（每个数字对应一个含义）：

**形态类（5 个）**

| 序号 | 特征名 | 含义 |
|------|--------|------|
| 1 | 对数收益率（log return） | 相对上一根收盘价的涨跌幅度 |
| 2 | 实体比例（body ratio） | 实体占整根 K 线高低区间的比例 |
| 3 | 上影线比例（upper wick） | 上影线占整根区间的比例 |
| 4 | 下影线比例（lower wick） | 下影线占整根区间的比例 |
| 5 | 对数成交量（log volume） | 成交量的对数变换，减轻极端值影响 |

**趋势类（12 个）**

在 **20、60、120** 根 K 线三个时间窗口上，各计算 4 项（共 12 项）：

- 斜率（slope）：价格线性趋势方向与 steepness  
- 残差标准差（residual std）：趋势拟合后的抖动程度  
- 决定系数（R²）：趋势拟合好坏  
- 趋势强度（strength）：斜率相对残差的比值  

**因果原则**：在时刻 t 计算特征时，**只能使用 t 及以前** 的历史 K 线，禁止使用未来数据，以保证回测与实盘一致。

### 3.3 标准化

每个特征在时间上做 **滚动标准化（trailing z-score）**：

- 默认窗口：120 根 K 线  
- 仅使用当前及过去数据计算均值与标准差  
- 训练集统计量不泄漏到未来  

### 3.4 送入模型的张量形状

一次推断时，模型接收：

| 名称 | 形状 | 说明 |
|------|------|------|
| 上下文 K 线（ctx_bars） | 批次 × 128 × 17 | 128 为上下文长度（context bars） |
| 有效长度（ctx_lengths） | 批次 | 实际有效 K 线根数，通常为 128 |

## 四、输出（Output）

模型启用 **市场状态预测头（Market State Head）** 时，一次前向计算返回以下结果（训练时还有对应标签用于计算损失）：

### 4.1 收益预测（return_pred）

- **形状**：未来 5 步，每步 1 个数值  
- **含义**：未来第 1～5 根 K 线的对数收益率预判（经标准化尺度；可反标准化还原）  
- **说明**：当前模型在 **逐步** 收益排序上较弱；**累计** 5 步收益相关能力相对更强  

### 4.2 累计收益预测（cum_return_pred）

- **形状**：每个样本 1 个数值  
- **含义**：未来 5 根 K 线 **累计** 对数收益  
- **产生方式**：由 **按步长感知的收益头（Horizon Return Head）** 专门输出  
- **用途**：交易规则中判断「接下来一段」整体偏多还是偏空  

### 4.3 方向预测（direction_logits）

- **形状**：未来 5 步 × 3 类  
- **三类**：跌（0）/ 横盘（1）/ 涨（2）  
- **含义**：每一步的三分类 logits（未归一化分数）；经 softmax 后可得概率  
- **派生量**：上涨概率减下跌概率，常作为 **edge** 用于规则  

### 4.4 累计方向（cum_direction_logit）

- **形状**：每个样本 1 个数值  
- **含义**：未来 5 步 **整体** 涨还是跌的二分类 logit  

### 4.5 波动预测（volatility_pred）

- **形状**：未来 5 步，每步 1 个数值  
- **含义**：未来已实现波动强度的预判  

### 4.6 风险预测（risk_logits）

- **形状**：未来 5 步 × 2 类  
- **含义**：低/高波动风险状态；高概率时常用于 **禁止开仓或强制减仓**  

### 4.7 训练用辅助输出（推断时可忽略）

- 向量量化损失（VQ loss）、困惑度（perplexity）  
- 切分点 logits（break logits）  
- 形态代号（segment codes）

## 五、模型内部结构

主类名：**KlinePatternPredictor**（K 线形态预测器）。  
整体可理解为 **四条流水线**：

```
原始 K 线特征 [128 × 17]
        ↓
① 自动切分 + 向量量化编码（AutoSegmentVQEncoder）
        ↓
形态 token 序列 [最多 16 段 × 模型维度]
        ↓
② 段位置嵌入 + 因果 Transformer 主干
        ↓
③ 段级注意力汇聚 + 原始上下文旁路
        ↓
④ 市场状态多任务预测头
        ↓
收益 / 方向 / 波动 / 风险
```

---

### 5.1 第一层：自动切分与向量量化（AutoSegmentVQEncoder）

**作用**：把 128 根 K 线自动切成若干 **形态片段**，每段压缩成一个 token。

**步骤 A — 切分（BarSequenceSegmentingMHA）**

- 将每根 K 线投影到模型维度（默认 128）  
- 用 **一层因果多头自注意力（Causal Multi-Head Attention）** 扫描序列  
- 预测在哪些 bar 之后插入「结构断点」（类似章节标题）  
- 约束：每段最短 4 根、最长 32 根 K 线；最多 16 段  

**步骤 B — 段内编码（SegmentMHAEncoder + SegmentCNNEncoder）**

- **多头注意力段编码器**：段内用 summary token 汇聚整段信息  
- **卷积神经网络段编码器（可选）**：用 3/5/9 根局部卷积核捕捉短周期形态  
- 两路结果融合后进入 **向量量化（Vector Quantization，VQ）**  

**步骤 C — 向量量化（VQ）**

- 将连续段向量映射到约 **16 个离散形态代号**  
- 类似把无穷多种走法归到有限「形态词典」，便于后续序列建模  

**输出**：形态 token 序列，形状约为 **批次 × 段数 × 128**。

---

### 5.2 第二层：因果 Transformer 主干（CausalTransformer）

**作用**：在 **段级 token 序列** 上建模时间依赖。

- 层数：默认 2 层  
- **因果掩码（causal mask）**：第 i 段只能看见第 1～i 段，不能看见未来段  
- 段序列开头加 **可学习段位置嵌入（positional embedding）**：表示「第几段」，最多 16 段  

**注意**：此处位置是 **段序号**，不是原始 K 线的绝对 bar 序号；段长可变，故不等同于真实日历距离。

---

### 5.3 第三层：汇聚与旁路（Pooling + Raw Context）

**段级注意力汇聚（SegmentAttentionPool）**

- 可学习 query 对全部段 token 做交叉注意力（cross-attention）  
- 得到单个 **汇总向量（pooled vector）**，代表整段历史的综合理解  

**原始上下文旁路（raw_context_features）**

- 直接从最近 K 线提取摘要：多窗口累计收益、波动、末根形态、趋势一致性等（约 17～29 维）  
- 经线性投影后与汇总向量 **相加**  
- **目的**：避免「切段 + 量化」丢失细粒度近期信息  

---

### 5.4 第四层：市场状态预测头（MarketStateHead）

在汇总向量（及必要时段 token）上并行输出多任务结果：

| 子模块 | 输出 |
|--------|------|
| 收益头（return head）或按步长感知收益头（Horizon Return Head） | 逐步收益 + 累计收益 |
| 方向状态头（Direction State Head） | 逐步三分类 + 累计方向 |
| 波动头（volatility head） | 逐步波动 |
| 风险头（risk head） | 逐步二分类风险 |

## 六、默认结构与规模参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 上下文长度（context_bars） | 128 | 输入 K 线根数 |
| 预测步长（pred_horizon） | 5 | 输出未来步数 |
| 模型维度（d_model） | 128 | 内部向量宽度 |

| 单段最大长度（max_seg_len） | 32 | 每段最多 K 线数 |

## 七、模型如何形成「预判」（机制说明）

1. **历史形态分段**：模型学习在波动转折处切开走势，形成少量「章节」。  
2. **形态归类**：每章压缩成离散代号，便于比较「以前是否见过类似结构」。  
3. **顺序阅读**：因果 Transformer 按时间顺序读这些代号，推断「当前结构之后常出现什么」。  
4. **多任务输出**：同时给出收益、方向、波动、风险，供规则层过滤与下单。  

**能力边界（项目实测）**：

| 能力 | 水平 | 说明 |
|------|------|------|
| 累计收益排序相关（cum_return_ic） | 较强（约 0.10～0.13） | 适合判断未来数小时整体强弱 |
| 累计方向准确率 | 中等偏上（约 60% 量级） | 优于随机猜测 |
| 逐步收益排序相关（return_ic） | 较弱 | 不适合精确「下一根」择时 |
| 低误差（mean absolute error） | 可能误导 | 数值平滑不等于排序能力好 |

## 八、从模型到交易

模型只产生 **信号**；是否开仓、杠杆、止损由 **规则层**（如 v012 规则）决定：

- 方向 edge、累计收益符号、风险概率 → 开仓过滤  
- 最大杠杆、单笔风险预算、最大回撤停止 → 风控  

回测目录：`backtest/`  
训练报告目录：`reports/`

## 十、结构总览图

```
币安等来源 1 小时 K 线（开高低收量）
              │
              ▼
    每根 K 线 → 17 维特征（因果 + 滚动标准化）
              │
              ▼
    过去 128 根 ──→ AutoSegmentVQEncoder
              │      ├─ 因果切分（找结构断点）
              │      ├─ 段内编码（注意力 + 卷积）
              │      └─ 向量量化 → 形态 token
              ▼
    + 段位置嵌入 → CausalTransformer（2 层，因果）
              ▼
    SegmentAttentionPool + 原始上下文旁路
              ▼
    MarketStateHead
      ├─ 逐步 / 累计收益
      ├─ 逐步 / 累计方向
      ├─ 波动
      └─ 风险
              ▼
    交易规则（阈值、止损、杠杆）→ 回测 / 仿真
```
