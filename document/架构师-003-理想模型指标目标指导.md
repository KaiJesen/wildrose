# BTC 合约预测模型理想指标目标指导

## 1. 文档目标

本文档面向模型训练师，定义当前 BTC 多任务市场状态模型在理想状态下应达到的指标画像，并给出从当前状态逐步逼近该目标的训练路径。

本文只讨论模型预测能力、训练指标和验证标准，不涉及交易执行、资金管理、仓位、开平仓或策略规则。

## 2. 理想状态的定义

理想状态不是指模型能够高准确率预测每一根 K 线涨跌。BTC 合约短周期公开数据噪声很高，若模型在 out-of-sample 上声称长期达到 `70%+` 的方向准确率，反而应优先怀疑数据泄漏、评估污染或样本选择偏差。

本文定义的理想模型是：

```text
在严格时间切分、无泄漏、真实公开数据条件下，
模型能稳定输出未来市场状态的概率化预测，
并在方向、收益排序、波动率和风险识别四类任务上同时优于基线。
```

理想模型应具备四个特征：

1. 方向判断有弱而稳定的优势。
2. 收益预测具备正 IC，且 valid/test 同方向。
3. 波动率预测显著优于 naive baseline。
4. 风险状态识别不坍缩，能识别少数高风险样本。

## 3. 当前基线

当前正式实验 `0050_market_state_formal` 的测试集指标：

```text
direction_acc        = 34.59%
direction_macro_f1   = 0.269
cum_direction_acc    = 55.41%
return_ic            = 0.073
return_mae           = 0.02598
volatility_mae       = 0.08525
risk_f1              = 0.455
```

当前模型的状态可以概括为：

- `return_ic` 已转正，收益排序任务已有初步信号。
- `return_mae` 与 `volatility_mae` 相比 smoke 明显改善。
- `direction_macro_f1` 和 `risk_f1` 仍弱，说明分类任务没有被充分学好。
- 当前不是结构失败，而是训练目标、类别平衡、标签阈值和选模准则仍需优化。

## 4. 理想指标分级

训练目标分为三个层级：可用、成熟、理想。训练师不得跳过“可用”直接追求“理想”。

### 4.1 可用模型

可用模型用于确认当前架构路线有效。

```text
cum_direction_acc    >= 56%
direction_macro_f1   >= 0.30
return_ic            >  0
volatility_mae       <= 0.10
risk_f1              >= 0.48
```

要求：

- 5 项至少满足 4 项。
- `return_ic <= 0` 时不得判定为可用。
- `direction_macro_f1 < 0.27` 时不得判定为可用。

### 4.2 成熟模型

成熟模型用于作为后续研究主线。

```text
cum_direction_acc    >= 58%
direction_acc        >= 37%
direction_macro_f1   >= 0.34
return_ic            >= 0.08
return_mae           <= 0.024
volatility_mae       <= 0.075
risk_f1              >= 0.52
```

额外要求：

- valid 与 test 的 `return_ic` 同号。
- valid 与 test 的 `cum_direction_acc` 差距不超过 `4%`。
- `risk_positive_rate_pred` 不能接近 `0` 或 `1`。
- 三分类 direction 的任一类别 F1 不得长期为 `0`。

### 4.3 理想模型

理想模型是当前公开数据和小型模型路线下的长期目标，不是单轮训练目标。

```text
cum_direction_acc    = 60% - 63%
direction_acc        = 38% - 42%
direction_macro_f1   = 0.38 - 0.45
return_ic            = 0.10 - 0.15
return_mae           <= 0.022
volatility_mae       <= 0.060
risk_f1              = 0.58 - 0.65
```

稳定性要求：

```text
multi_seed_std(cum_direction_acc) <= 2.0%
multi_seed_std(return_ic)         <= 0.04
valid_test_gap(cum_direction_acc) <= 3.0%
valid_test_gap(direction_macro_f1)<= 0.05
valid_test_gap(risk_f1)           <= 0.07
```

如果某次实验超过理想上限很多，例如 `cum_direction_acc > 68%` 或 `return_ic > 0.25`，训练师必须优先排查泄漏，而不是直接接受结果。

## 5. 理想指标组合画像

理想模型不是单项指标极高，而是多项指标互相支持。

### 5.1 方向任务画像

理想状态：

- `cum_direction_acc` 稳定在 `60% - 63%`。
- `direction_macro_f1` 达到 `0.38 - 0.45`。
- down / flat / up 三类均有有效召回。
- 每个 horizon 的方向准确率不应极端分化。

诊断目标：

```text
direction_acc_h1 >= 38%
direction_acc_h2 >= 37%
direction_acc_h3 >= 36%
direction_acc_h4 >= 35%
direction_acc_h5 >= 35%
```

如果只有 `h1` 好、后续 horizon 失效，说明模型更像短期反应器，不是稳定市场状态模型。

### 5.2 收益任务画像

理想状态：

- `return_ic` 在 valid/test 均为正。
- 多次 seed 下平均 `return_ic >= 0.10`。
- 预测收益的标准差不能塌缩到接近 `0`。
- `return_mae` 相对当前 `0050` 至少再下降 `10% - 15%`。

诊断目标：

```text
return_ic_h1...hH 多数为正
cum_return_ic >= step_return_ic 的稳定水平
pred_return_std / true_return_std 不低于 0.25
```

如果 `return_ic` 为正但 `pred_return_std` 极低，模型可能只学到均值附近的排序噪声。

### 5.3 波动率任务画像

理想状态：

- `volatility_mae <= 0.060`。
- 高波动区间误差不能显著劣于普通区间。
- 预测波动率应与真实 realized volatility 呈正相关。

建议新增指标：

```text
volatility_corr
volatility_mae_high_vol
volatility_mae_low_vol
volatility_direction_acc
```

其中 `volatility_direction_acc` 表示模型能否判断未来波动率相对当前水平上升或下降。

### 5.4 风险任务画像

理想状态：

- `risk_f1 = 0.58 - 0.65`。
- risk 正样本召回率不能过低。
- 预测正样本比例接近真实正样本比例，不出现全负或全正。

诊断目标：

```text
risk_precision >= 0.50
risk_recall    >= 0.50
risk_f1        >= 0.58
risk_positive_rate_pred / risk_positive_rate_true in [0.6, 1.6]
```

如果 `risk_f1` 提升但 recall 很低，说明模型只在极少数明显样本上预测风险，不足以称为理想风险识别。

## 6. 理想模型的一票否决项

以下任一情况出现，即使单项指标达到理想区间，也不得判定为理想模型：

1. 阈值、scaler、class weights 在 valid/test 上重新拟合。
2. 只在单一 seed、单一切分上达标。
3. 类别分布显示 direction 或 risk 预测坍缩。
4. valid 指标好、test 指标明显反向。
5. 只提升 `return_ic`，但 `direction_macro_f1` 和 `risk_f1` 恶化。
6. 未报告 baseline 对比。
7. 未报告 per-horizon 指标。

## 7. 达标路线图

### 阶段 1：从当前到可用

目标：

```text
cum_direction_acc >= 56%
direction_macro_f1 >= 0.30
risk_f1 >= 0.48
return_ic > 0
```

动作：

- 使用 `direction_threshold_quantile=0.25`。
- 使用 `risk_threshold_quantile=0.70`。
- 使用分类偏重 loss：

```text
return_weight     = 0.30
direction_weight  = 0.50
volatility_weight = 0.10
risk_weight       = 0.10
```

- 补充 class distribution 与 confusion matrix。
- 保持模型规模不变。

不得做：

- 不得扩大 `d_model`。
- 不得同时更换主干和标签。
- 不得只根据 test 选模型。

### 阶段 2：从可用到成熟

目标：

```text
cum_direction_acc >= 58%
direction_macro_f1 >= 0.34
return_ic >= 0.08
risk_f1 >= 0.52
volatility_mae <= 0.075
```

动作：

- 对 direction/risk 引入 train class weights。
- 对 risk 尝试 focal loss。
- 增加 per-horizon loss 诊断。
- 进行 3 seed 复跑，观察指标方差。
- 若风险头仍停滞，再考虑 direction/risk 专用轻量 decoder。

训练师应保留以下对照：

```text
0051: 阈值 + 权重
0052: 0051 + class weights
0053: 0052 + risk focal loss
```

### 阶段 3：从成熟到理想

目标：

```text
cum_direction_acc >= 60%
direction_macro_f1 >= 0.38
return_ic >= 0.10
risk_f1 >= 0.58
volatility_mae <= 0.060
```

动作：

- 使用 walk-forward validation，而不是单次 train/valid/test。
- 扩展训练数据到更长时间窗，例如 `2y - 4y`。
- 引入多标的预训练，但 BTC 仍作为独立 test。
- 对比 CPC encoder、VQ segment encoder 和普通 causal transformer。
- 增加校准指标，例如 ECE、Brier score。

允许考虑结构改造：

- direction/risk 专用 decoder。
- horizon-aware multi-task head。
- CPC 预训练 encoder 接入 `MarketStateHead`。
- 多周期输入，例如 `1h + 4h` 的 causal 对齐特征。

## 8. 理想状态选模分数

可用阶段使用偏保守分数：

```text
score_v1 = 0.45 * cum_direction_acc
         + 0.25 * direction_macro_f1
         + 0.15 * max(return_ic, -0.05)
         + 0.10 * risk_f1
         - 0.05 * volatility_mae
```

成熟阶段建议升级为：

```text
score_v2 = 0.30 * cum_direction_acc
         + 0.20 * direction_macro_f1
         + 0.20 * normalized_return_ic
         + 0.15 * risk_f1
         + 0.10 * volatility_quality
         + 0.05 * stability_score
```

其中：

```text
normalized_return_ic = clip(return_ic / 0.15, -1, 1)
volatility_quality   = 1 - clip(volatility_mae / 0.12, 0, 1)
stability_score      = 1 - normalized_multi_seed_std
```

理想阶段不得只用单次 valid score 选模，必须使用：

```text
final_score = 0.60 * walk_forward_mean_score
            + 0.25 * test_score
            + 0.15 * stability_score
```

## 10. 训练师执行原则

1. 先达成可用，再追求成熟，最后才追求理想。
2. 每轮只改变一个主要因素，保证结果可归因。
3. 方向、收益、波动率、风险四类指标必须同时看。
4. 任何异常高指标先查泄漏。
5. 不把 `loss` 下降等同于模型变好。
6. 不用单一 test 表现决定模型。
7. 每次训练必须保留 baseline 对比。

## 11. 最终目标

当前项目的长期理想目标可以表述为：

```text
在真实 BTCUSDT 公开 1h 数据上，
多任务市场状态模型经过 walk-forward 与多 seed 验证后，
稳定达到：

cum_direction_acc    >= 60%
direction_macro_f1   >= 0.38
return_ic            >= 0.10
volatility_mae       <= 0.060
risk_f1              >= 0.58

且无泄漏、无预测坍缩、valid/test 指标一致。
```

达到该目标后，模型可被视为成熟的市场状态预测模型；在此之前，训练工作的重心应继续放在标签质量、类别平衡、稳定性验证和多任务权重控制上。
