# 软件设计师_003_市场状态模型最终训练建议[训练建议]

## 1. 结论（最终建议）

基于架构师指导与当前 `0049_market_state_smoke`、`0050_market_state_formal` 结果，最终建议为：

1. **本轮不改主干结构，继续训练与调参**（保持 `KlinePatternPredictor + MarketStateHead` 路线）。
2. 优先修复 `direction/risk` 弱监督问题（阈值与损失权重），暂不扩大模型容量。
3. 使用“分阶段训练 + 指标分层验收”替代一次性长训，确保每轮可解释、可回滚、可复现。

原因：`return_ic` 已从负值转正（`-0.026 -> 0.073`），`return_mae` 与 `volatility_mae` 明显改善，说明当前多任务建模方向正确；短板集中在 `direction_macro_f1` 与 `risk_f1`，属于监督密度与任务权重分配问题，优先通过训练策略解决更稳健。

---

## 2. 一票否决项（先检查，再看分数）

出现以下任一情况，本轮结果直接判定无效，不进入选模：

1. 在 valid/test 上重新拟合阈值、scaler 或 class weight。
2. `direction_threshold` / `risk_vol_threshold` 不是仅由 train split 估计。
3. 出现 NaN、loss 爆炸、预测坍缩为单一类别且未解释。
4. 只报 test 最优，不给 best valid 及选模依据。

---

## 3. 当前模型状态诊断（用于后续训练约束）

来自 `reports/0049_market_state_smoke/metrics.txt` 与 `reports/0050_market_state_formal/metrics.txt`：

- `direction_acc`: `34.6% -> 34.6%`（无提升）
- `direction_macro_f1`: `0.283 -> 0.269`（下降）
- `cum_direction_acc`: `54.7% -> 55.4%`（小幅提升）
- `return_ic`: `-0.026 -> 0.073`（显著改善）
- `return_mae`: `0.040250 -> 0.025979`（显著改善）
- `volatility_mae`: `0.150485 -> 0.085249`（显著改善）
- `risk_f1`: `0.453 -> 0.455`（几乎持平）

解释：

1. 共享编码器已学习到收益/波动信息，但分类头（direction/risk）仍受类别分布与损失竞争影响。
2. 当前阈值（`direction_threshold=0.00120399`，`risk_vol_threshold=0.00492658`）导致 `flat`/低风险样本偏多，削弱分类梯度质量。
3. 因核心回归任务已改善，不应回退到结构重构，而应先提高分类任务监督信噪比。

---

## 4. 最终训练策略（执行版）

## 4.1 固定不变项（避免引入额外变量）

- 数据：`binance_vision / BTCUSDT / 1h / 365d`（只用真实数据）
- 架构：`d_model=128, n_heads=4, trunk_layers=2, pred_horizon=5`
- 时间切分：严格顺序切分，不随机打散
- 阈值估计：仅用 train split 拟合，valid/test 只应用

## 4.2 必调项（第一优先级）

1. 标签阈值：
   - `direction_threshold_quantile: 0.35 -> 0.25`
   - `risk_threshold_quantile: 0.80 -> 0.70`
2. 损失权重：
   - `return_weight=0.30`
   - `direction_weight=0.50`
   - `volatility_weight=0.10`
   - `risk_weight=0.10`
3. 训练时长：
   - `epochs=60`
   - 保留早停或 best-checkpoint 机制，防止后段震荡回退

## 4.3 次优先级增强（仅在第一优先级完成后启用）

1. 为 `direction`、`risk` 的 CE 加入 class weight（按 train 标签统计）。
2. 若 `direction_macro_f1` 仍无提升，再尝试 focal loss（仅作用于分类头）。
3. 维持学习率策略不变，避免和阈值调整产生耦合干扰。

---

## 5. 选模与验收标准（最终版）

## 5.1 选模分数（valid）

建议统一为：

`score = 0.45*cum_direction_acc + 0.25*direction_macro_f1 + 0.15*max(return_ic, -0.05) + 0.10*risk_f1 - 0.05*volatility_mae`

说明：

- 主目标仍是方向一致性与分类均衡；
- `return_ic` 做下限截断，降低少量负值 epoch 的波动冲击；
- `risk_f1` 明确进入主分数，避免风险头长期“失声”；
- `volatility_mae` 作为惩罚项抑制偏科。

## 5.2 最低验收门槛（0051）

1. `cum_direction_acc >= 56.0%`
2. `direction_macro_f1 >= 0.30`
3. `risk_f1 >= 0.48`
4. `return_ic > 0`（valid/test 同号）
5. `volatility_mae <= 0.10`

合格条件：5 项至少满足 4 项；若 `return_ic <= 0` 或 `direction_macro_f1 < 0.27`，即使其余项达标也不作为主线默认模型。

## 6. 必须补充的诊断指标（写入 metrics.json）

1. `direction_class_distribution_true`
2. `direction_class_distribution_pred`
3. `risk_positive_rate_true`
4. `risk_positive_rate_pred`
5. `direction_confusion_matrix`
6. `risk_confusion_matrix`
7. `direction_acc_h1...hH`
8. `return_ic_h1...hH`
9. `pred_return_std`
10. `pred_volatility_std`

目的：定位 `direction_macro_f1` 下降原因（类别混淆 vs 边界设置）与 `risk_f1` 停滞原因（正样本稀疏 vs 预测坍缩）。

---

## 7. 建议实验排期（减少试错成本）

## 阶段 A（阈值+权重）：1 轮

- 目标：验证分类监督密度提升是否立即带来 `direction_macro_f1` 抬升。
- 配置：使用 4.2 全量参数。

## 阶段 B（类别重加权，0052）：1 轮

- 条件：A 阶段 `direction_macro_f1 < 0.30`。
- 目标：缓解长尾类别下的 CE 偏置。

## 阶段 C（风险头专项，0053）：1 轮

- 条件：B 阶段仍未触发验收门槛。
- 目标：进一步提升风险识别召回，验证 `risk_f1` 是否可突破 `0.48`。
- 配置：`risk_weight=0.20`，risk 分类启用 focal loss，其余参数固定。

说明：A/B/C 每阶段只改变一个因素，便于定位收益来源，符合“强基线约束”。

---

## 8. 触发“需要改模型”的条件（避免过早重构）

仅在以下条件同时满足时，才建议进入结构改造：

1. 已完成 A/B/C 三阶段训练；
2. `direction_macro_f1` 连续两轮 < `0.30`；
3. `risk_f1` 连续两轮 < `0.48`；
4. `return_ic` 保持正值但分类任务长期停滞。

若触发，再考虑：

- direction/risk 分支解耦（共享 trunk + 专用轻量 decoder）；
- 或将 CPC 预训练编码器接入同一 `MarketStateHead` 做同口径对比。

## 9. 建议命令（0051 基准执行）

```bash

  --source binance_vision --symbol BTCUSDT --interval 1h --days 365 \
  --epochs 60 --batch-size 64 --device cpu \
  --direction-threshold-quantile 0.25 \
  --risk-threshold-quantile 0.70 \
  --return-weight 0.30 \
  --direction-weight 0.50 \
  --volatility-weight 0.10 \
  --risk-weight 0.10 \
  --checkpoint-dir checkpoints/0051_market_state_tuned \
  --report-dir reports/0051_market_state_tuned
```
