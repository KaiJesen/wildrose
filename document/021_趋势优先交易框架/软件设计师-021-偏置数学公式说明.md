| 项 | 内容 |
|----|------|
| 角色 | 软件设计师 |

**核心原则**：`bias > 1` 的语义必须在每个接入点有唯一公式；**偏置不得直接放宽硬风险边界**（`max_position_ratio`、熔断、最坏损失上限）。

---

## 2. 符号约定

| 符号 | 含义 |
|------|------|
| `eps` | 小正数，建议 `1e-6`，防止除零 |
| `base_*` | 配置中的基础阈值（来自 `TradingRuleConfig`） |
| `side` | `long` 或 `short` |

`open_bias_side`：`bias > 1` 表示**放宽**开仓（降低 edge/prob 门槛）；`bias < 1` 表示收紧；`bias == 0` 等价于禁止开仓。

`risk_tolerance_bias_side`：**独立**于 `open_bias`；仅放大/缩小风险类阈值的容忍度，不降低 edge 门槛。

---

## 3. 开仓阈值修正

当前 `RuleEngine` 常见判定为“越大越好”或“越小越好”两类，必须分开写。

### 3.1 Edge / 概率 / 累计收益（越大越好）

```text
effective_edge_threshold_side =
    base_edge_threshold_side / max(open_bias_side, eps)

effective_prob_threshold_side =
    base_prob_threshold_side / max(open_bias_side, eps)

effective_cum_ret_threshold_side =
    base_cum_ret_threshold_side / max(open_bias_side, eps)
```

**直觉**：`open_bias_long = 1.2` → 有效门槛变为原来的 `1/1.2 ≈ 0.83`，更容易满足。

### 3.2 风险概率（越小越好，`p_risk <= threshold`）

```text
effective_risk_max_side =
    base_risk_max_side * max(risk_tolerance_bias_side, eps)
```

**注意**：顺势放宽开仓时，应主要调 `open_bias`，**不要**同步把 `risk_tolerance_bias` 拉高，否则 20x 杠杆下会双重放宽。

### 3.3 开仓 gate（布尔）

```text
may_open_long =
    trend_bias.allow_open_long
    AND open_bias_long > 0
    AND risk_budget.allow_open_long
    AND decision_scope in {open_only, open_size, full}
    AND NOT (legacy block 与 bias 重复生效 — 见迁移开关)
```

## 4. 仓位计算（PositionSizer）

```text
signal_strength_scale = f(signal)   # 现有逻辑，不变

raw_target_ratio =
    base_position_ratio
    * signal_strength_scale
    * size_bias_side

target_position_ratio =
    min(
        raw_target_ratio,
        risk_budget.remaining_position_ratio,
        cfg.max_position_ratio
    )
```

`size_bias_side`：

| 场景 | 典型范围 |
|------|----------|
| 高一致性顺势 | `1.2 ~ 1.5` |
| 普通顺势 | `1.0 ~ 1.2` |
| 轻度逆向 | `0.4 ~ 0.7` |
| 中度逆向 | `0.2 ~ 0.5` |
| 强逆势 | `0.0`（或极小试探仓，由配置 `min_probe_ratio`） |

**硬边界**：无论 `size_bias` 多大，`target_position_ratio <= max_position_ratio`（20%）恒成立。

## 5. RiskBudget 对象（动作级权限）

`RiskManager` 每 bar 输出 `RiskBudget`，与 `TrendBiasContext` **分离**。Builder **不**计算 RiskBudget。

```python
@dataclass(frozen=True)
class RiskBudget:
    current_position_ratio: float
    max_position_ratio: float
    current_margin_ratio: float
    remaining_position_ratio: float
    worst_case_loss_ratio: float
    remaining_loss_budget_ratio: float

    allow_open_long: bool
    allow_open_short: bool
    allow_add_long: bool
    allow_add_short: bool
    allow_reverse: bool

    open_reject_reason_long: str
    open_reject_reason_short: str
    add_reject_reason_long: str
    add_reject_reason_short: str
```

Phase 0 契约必须能回答：**能不能新开？能不能加仓？能不能反手？还能开多大？被拒绝原因是什么？**

### 5.1 新开仓许可

```text
may_open_long =
    trend_bias.allow_open_long
    AND open_bias_long > 0
    AND risk_budget.allow_open_long
    AND decision_scope in {open_only, open_size, full}

may_open_short =
    trend_bias.allow_open_short
    AND open_bias_short > 0
    AND risk_budget.allow_open_short
    AND decision_scope in {open_only, open_size, full}
```

### 5.2 加仓许可

```text
may_add_long =
    trend_bias.allow_add_long
    AND risk_budget.allow_add_long
    AND risk_budget.remaining_position_ratio > 0
    AND decision_scope in {open_size, full}
```

空头 `may_add_short` 对称。

### 5.3 反手许可

```text
may_reverse =
    risk_budget.allow_reverse
    AND trend_bias.allow_open_{target_side}   # 目标方向
    AND decision_scope in {open_only, open_size, full}
```

反手与空仓新开、同向加仓的风险判断分开估算；拒绝时写入 `open_reject_reason_*` 或专用 `reverse_reject_reason`。

### 5.4 加仓后目标仓位

```text
add_target_ratio =
    min(
        current_position_ratio + add_increment * size_bias_side,
        risk_budget.remaining_position_ratio + current_position_ratio,
        max_position_ratio
    )
```

加仓前必须重算 `worst_case_loss_ratio`；若超过保证金最坏损失上限 → `allow_add_{side}=false`，`add_reject_reason_{side}=TREND_ADD_REJECTED_WORST_CASE`。

## 6. 持仓时长（hold_bias）

```text
effective_min_hold_bars_side =
    round(base_min_hold_bars_side * hold_bias_side)
```

- `hold_bias > 1`：延长最短持仓
- `hold_bias < 1`：缩短（逆向仓、衰竭段）

逆向仓额外约束（与 `counter_level` 联动）：

| `counter_level` | `hold_bias` 上限建议 |
|-----------------|----------------------|
| `LIGHT` | `≤ 0.8` |
| `MEDIUM` | `≤ 0.6` |
| `HARD_BLOCK` | 不应新开；已持则 `≤ 0.5` |

---

## 7. 退出投票（exit_bias）

```text
effective_exit_votes_required =
    ceil(base_exit_votes * max(exit_bias_side, eps))
```

- `exit_bias > 1`：需要**更多**票才退出（顺势 runner）。例：`base=3, exit_bias=1.5` → `ceil(4.5)=5`
- `exit_bias < 1`：更快退出。例：`base=3, exit_bias=0.7` → `ceil(2.1)=3`（略快）或 `0.5` → `2`

`best_point_exit` 贡献固定 `±1` 票，不乘 `exit_bias`（P6 边界）。

### 7.1 强制退出（不受 exit_bias 放宽）

趋势侧与风险侧分开：

```text
trend_bias.force_exit_{side}   # P1 趋势强退建议（LEG_END、crash 极端等）
risk_event.force_exit          # P0 风险强退命令（熔断、最坏损失），最终优先级最高

if risk_event.force_exit OR trend_bias.force_exit_{side}:
    → CLOSE（覆盖任何 hold_bias / exit_bias）
```

## 8. 同一 bar 动作优先级总表

| 优先级 | 动作类型 | 示例 | 与 bias 关系 |
|--------|----------|------|--------------|
| **P0** | 风险强制退出 / 熔断 | `CLOSE_RISK`、`force_exit_*` | 覆盖一切偏置 |
| **P1** | 趋势破坏强退出 / 反向腿确认 | `CLOSE_TREND_BREAK`、`LEG_END` | 覆盖 hold/exit 放宽 |
| **P2** | 减仓保护利润 / exhaustion | `REDUCE_TREND_EXHAUSTION` | 先于加仓 |
| **P3** | 加仓 / upgrade | `ADD_TREND_CONTINUATION`、`UPGRADE_TO_TREND` | 须过 P0/P1/P2 |
| **P4** | 新开仓 | `OPEN_LONG` / `OPEN_SHORT` | 须 `trend_bias.allow_open_*` + `risk_budget.allow_open_*` |
| **P5** | 继续持有 | `HOLD_RUNNER`、`HOLD_TREND_PULLBACK` | 默认 |
| **P6** | 观望 | `HOLD_NO_ENTRY` | 默认 |

规则：

```text
同一 bar：P0/P1 可覆盖任何趋势偏置
P3 加仓必须晚于 P0/P1/P2 检查
P4 新开仓不得绕过 trend_bias.allow_open_side 与 risk_budget.allow_open_side
P3 加仓不得绕过 trend_bias.allow_add_side 与 risk_budget.allow_add_side
RuleEngineActionResolver 在消费阶段合并 TrendBias + RiskBudget + RiskEvent，而非 TrendBiasBuilder 内嵌 P0
```

## 9. alignment_score 计算公式（冻结）

对多头视角（空头对称）：

```text
score_macro = +1 if macro==UP else (-1 if macro==DOWN else 0)
score_leg   = +2 if leg==UP   else (-2 if leg==DOWN   else 0)
score_micro = +1 if micro==UP else (-1 if micro==DOWN else 0)

raw_long = score_macro + score_leg + score_micro

alignment_score_long = clamp_compress(raw_long)
  >= +3  → +2
  +1~+2  → +1
  0      → 0
  -1~-2  → -1
  <= -3  → -2
```

`counter_level_long` 映射：

| `alignment_score_long` | `counter_level_long` |
|------------------------|----------------------|
| `+2`, `+1` | `NONE` |
| `0` | `LIGHT` |
| `-1` | `MEDIUM` |
| `-2` | `HARD_BLOCK` |

修正：EXHAUSTION 可将 `HARD_BLOCK` 降为 `MEDIUM`；crash 强确认可将空头 `MEDIUM` 升为 `NONE`（见冲突裁决表）。

## 10. 偏置与硬风险边界

以下边界 **任何 bias 不得突破**：

1. `max_position_ratio`（20%）
2. `max_margin_loss` / 最坏损失估计
3. 日回撤熔断、连亏冷却（`risk_event`，P0）
4. `risk_event.force_exit`（P0，高于 `trend_bias.force_exit_*`）

### 10.1 `HARD_BLOCK` 强不变量

当 `counter_level_{side} == HARD_BLOCK` 时，**必须同时**成立（禁止“或”语义）：

```text
allow_open_{side}  = false
open_bias_{side}   = 0
size_bias_{side}   = 0
allow_add_{side}   = false
```

若配置允许极小逆势试探仓，须独立开关，不得混在普通 bias 中：

```text
allow_hard_counter_probe = true
hard_counter_probe_ratio <= 0.01
hard_counter_probe_max_hold_bars <= 2
```

此时 `counter_level` 可标为 `LIGHT` + 专用 reason `HARD_COUNTER_PROBE_ONLY`，而非 `HARD_BLOCK`。
