## 2. 回测版本矩阵

| 版本 ID | `decision_scope` | `disable_legacy_trend_rules` | 说明 |
|---------|------------------|------------------------------|------|
| `v020_baseline` | — | — | 无 trend_bias，现行系统 |
| `v021_observe_bias_only` | `observe` | `false` | Phase A：生成并记录 bias，不参与决策 |

## 3. 分阶段通过标准

### Phase 0：契约冻结（无回测）

- [ ] `TrendBiasContext` 字段与本文档族一致
- [ ] `TrendBiasConfig` + `decision_scope` 入配置
- [ ] 冲突裁决表、偏置公式文档评审通过
- **未通过不得改 `rules.py` 主流程**

### Phase A：`v021_observe_bias_only`

| 类别 | 指标 | 标准 |
|------|------|------|
| 稳定性 | bias 字段非空率 | ≥ 99% bar |
| 稳定性 | `reason_codes` 非空率 | ≥ 95% bar |
| 可解释 | 关键场景有 reason | 6 月初下跌、慢涨长段、震荡段人工 spot check 通过 |
| 回归 | `trade_count` vs v020 | 差异 = 0（未参与决策） |
| 回归 | 收益 / 回撤 vs v020 | 差异 = 0 |

### Phase B：`v021_open_bias`

| 类别 | 指标 | 标准 |
|------|------|------|
| 架构 | `legacy_trend_direct_block_count` | 相对 v020 下降（旧 block 已关闭） |
| 逆势 | `hard_counter_open_count` | 接近 0 或仅试探仓 |
| 稳定 | `trade_count` | 相对 v020 变化 < ±15%（异常需归因） |
| 可解释 | `bias_reason_codes_coverage` | 每个非 HOLD 开仓动作 ≥ 1 条 bias reason |
| 回归 | `max_drawdown` | 不得显著劣化（> 10% 相对恶化需评审） |

### Phase C：`v021_open_size_bias`

| 类别 | 指标 | 标准 |
|------|------|------|
| 风险 | `max_position_ratio_observed` | **永远 ≤ 20%** |
| 风险 | `max_margin_loss_ratio_observed` | ≤ 配置上限 |
| 风险 | `trend_add_candidate_count` | 可统计 |
| 风险 | `trend_add_risk_evaluated_count` | 若 candidate>0，须等于 candidate |
| 风险 | `trend_add_rejected_by_risk_count` | 可统计；可为 0，须有 reason 分布 |
| 风险 | `trend_add_allowed_count` | 可统计 |
| 趋势 | `avg_trend_hold_bars` | ≥ v020 基线 × 0.95 |
| 回归 | `max_drawdown` | 不得显著劣化 |

### Phase D：`v021_full_bias`

| 类别 | 指标 | 标准 |
|------|------|------|
| 架构 | `legacy_trend_direct_read_count` | 相对 v020 **显著下降**（目标 −80%） |
| 趋势 | `confirmed_leg_coverage_ratio` | ≥ v020 基线 |
| 持仓 | `avg_trend_hold_bars` | ≥ v020 基线 |
| 逆势 | `hard_counter_open_count` | 接近 0 |
| 可解释 | `bias_reason_codes_coverage` | 每个非 HOLD 动作有 bias reason |
| 稳定 | `trade_count` | 不因 bias 重复生效暴增（±20% 内） |
| 回归 | `max_drawdown` | 不得显著劣化 |

## 4. 架构验收 gates 汇总

| 类别 | 指标 | 建议阈值 / 说明 |
|------|------|-----------------|
| 架构 | `legacy_trend_direct_read_count` | Phase D 显著下降 |
| 可解释 | `bias_reason_codes_coverage` | 每个非 HOLD 动作必须有 bias reason |
| 趋势覆盖 | `confirmed_leg_coverage_ratio` | 不低于 v020 基线 |
| 持仓 | `avg_trend_hold_bars` | 不低于 v020 基线 |
| 逆势控制 | `hard_counter_open_count` | 接近 0 或仅试探仓 |
| 风险 | `max_position_ratio_observed` | 永远 ≤ 20% |
| 风险 | `trend_add_candidate_count` | 可统计 |

| 风险 | `trend_add_rejected_by_risk_count` | 可统计；可为 0，须有 reason 分布 |
| 回归 | `max_drawdown` | 不得显著劣化 |
| 稳定 | `trade_count` | 不应因重复生效暴增 |

## 5. 迁移期单一决策所有权

| Phase | bias 参与范围 | 旧趋势规则 |
|-------|---------------|------------|
| A | 无（仅日志） | 完全保留 |
| B | `allow_open_*`、`open_bias_*` | block 关闭或降为诊断 |
| C | + `size_bias_*`、RiskBudget | slow/crash 直接开仓动作改为 bias |
| D | + add/hold/exit/force_exit | RuleEngine 不直接消费原始趋势字段 |

配置：

```json
{
  "trend_bias": {
    "enabled": true,
    "decision_scope": "open_only",
    "disable_legacy_trend_rules": true
  }
}
```

**没有 `disable_legacy_trend_rules`，禁止进入 Phase B+ 的收益对比验收。**

## 7. 关键场景人工验收清单

| 场景 | 期望 bias 行为 |
|------|----------------|
| 6 月初持续下跌样本 | crash P1 优先；多头 block；空头 bias 上调；**窗口以回测报告标注的起止时间为准** |
| 慢涨长段 | slow_up 提高 long hold/size；非独立分支开仓 |
| 震荡无确认腿 | `leg_direction=NONE`；逆向 LIGHT，小仓 |
| leg DOWN + micro PULLBACK 做空 | 非 HARD_BLOCK，中等门槛 |
| 强共振逆势做多 | `counter_level_long=HARD_BLOCK`，`hard_counter_open_count` 不增 |
