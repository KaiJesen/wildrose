## 2. 背景：为什么逐 bar 打分不够

016/019 已把趋势从“下跌过滤器”扩展为方向、强度、阶段输出，但仍存在结构性局限：

```text
1. 每根 K 独立打分，缺少“这段趋势从哪开始、已经走了多久”的显式记忆；
2. SLOW_UP / FAST_UP / CRASH / RANGE 靠阈值竞争，容易在边界来回切换；
3. 交易规则仍把 trend_signal 当瞬时开关，而非区间上下文；
4. 回测标签也是逐 bar 的，无法衡量“是否吃满一整段趋势腿”。
```

典型失败模式：

| 现象 | 根因 |
|------|------|
| 6 月初长跌未吃满 | 识别到 DOWN，但无“下跌腿”概念，crash 仓被短规则平掉 |
| 慢涨长期 FLAT | 单 bar score 弱，未形成“慢涨腿”持续上下文 |
| 持仓 2–3 根就退 | 退出看瞬时 phase/score，不看腿内位置 |
| 趋势升级 count = 0 | 缺少“已进入 CONFIRMED_LEG”这一离散事件 |

020 不推翻 016/018/019，而是在其之上增加**区间层**。

## 3. 金融定义：趋势 = 连续区间序列

### 3.1 基本定义

```text
趋势区间（Trend Leg）= 从结构拐点 A 到结构拐点 B 的连续价格运动，
在该区间内主导方向一致，且回撤未破坏区间结构。
```

市场由多层区间嵌套组成：

```text
L0 Regime（大环境）     ：数周～数月，牛/熊/震荡宏观状态
L1 Trend Leg（趋势腿）  ：数小时～数天，可交易的主趋势段
L2 Sub-leg（子腿）      ：数根～数十根 K，腿内的推进/回撤/整理
L3 Trade Lifecycle      ：单笔持仓在腿内的 PROBATION → TREND → RUNNER
```

### 3.2 与现有模块的对应关系

| 020 层级 | 现有模块 | 关系 |
|----------|----------|------|
| L0 Regime | 暂无 | 020 新增，粗粒度背景 |
| L1 Trend Leg | `TrendSignal` + `SlowTrend` + `Crash` 的区间化 | 020 统合输出 |
| L2 Sub-leg | `TrendPhase` 的区间化 | 020 在腿内细分 |
| L3 Lifecycle | `engine.py` 中 `PositionLifecycle` | 018 已有，020 为其提供腿上下文 |

## 5. 分段方法

第一版推荐 **方案 A：Swing 结构分段**（可因果、可解释、实现成本低）。**方案 B：变更点检测** 作为第二版增强或与 A 混合。**§12** 描述用 Teacher 工具标注、Student Transformer 在线推断的路线。

---

### 5.1 方案 A：Swing 结构分段（推荐 v1）

#### 5.1.1 三步流水线

方案 A 不是“看到一个拐点就切一段趋势”，而是 **由细到粗** 的三步：

```text
Step 1  因果 Swing 拐点检测      →  结构转折点
Step 2  相邻拐点 → Micro Leg     →  最小结构段（较碎）
Step 3  同向合并 + 质量过滤      →  Macro Trend Leg（可交易大趋势）
```

#### 5.1.2 Step 1：因果 Swing 拐点

在 bar `i` 上，用**只看过去 + 固定确认延迟**的规则：

```text
swing_high 在 bar i 被确认，当：
  1. high[i] 是 [i-L, i] 内最高（左侧 L 根）
  2. 从 i 到当前 bar t，high[i] 仍是 [i, t] 内最高
  3. 当 t = i + R 时确认（右侧 R 根延迟，保证因果）

swing_low 对称，用 low 和窗口内最低。
```

拐点序列交替：`… swing_low → swing_high → swing_low → …`

#### 5.1.3 Step 2：Micro Leg

相邻两个 swing 拐点之间形成 **Micro Leg**，计算：

| 字段 | 含义 |
|------|------|
| `direction` | 低点→高点 = UP，高点→低点 = DOWN |
| `leg_bars` | 拐点间 K 线数 |
| `leg_return_atr` | (终点 close - 起点 close) / 起点 ATR |
| `pullback_inside` | 段内最大逆向回撤 / ATR |

此阶段段数多、粒度细，**不可直接用于交易**。

#### 5.1.4 Step 3：合并为 Macro Trend Leg

从第一个 Micro Leg 开始向后扫描：

```text
若当前 Micro Leg 与正在构建的 Macro Leg：
  - 方向相同，且
  - 中间回撤 < merge_pullback_atr × ATR
→ 合并进同一条 Macro Leg（更新 end、累计 return、duration）

若：
  - 方向相反，或
  - 回撤 ≥ merge_pullback_atr × ATR
→ 闭合当前 Macro Leg，开启新 Macro Leg
```

合并后得到 **L1 Trend Leg**，再按 §6.1 分类为 `SLOW_UP_LEG`、`FAST_DOWN_LEG` 等。

#### 5.1.5 双层 Swing（排除小趋势的关键）

单靠增大 L/R 不够；推荐 **两个尺度同时工作**：

```text
Small Swing (L=3, R=3)   → 腿内 Sub-leg 阶段（IMPULSE / PULLBACK），不切断 Macro Leg
Large Swing (L=8, R=8)   → Macro Trend Leg 边界（大趋势起止）
```

规则：

```text
新 Macro Leg 开始：仅当 Large Swing 出现反向确认拐点；
Macro Leg 结束：    Large Swing 反向拐点 + 回撤/效率条件；
Small Swing 反向：  只更新 sub_phase = PULLBACK，不 end Macro Leg。
```

这样 3~5 根的小反弹不会结束 6 月初长达数十根的大跌腿；慢涨中的小回调被吞并在同一条 `SLOW_UP_LEG` 内。

#### 5.1.6 七层过滤：排除小趋势、保留大趋势

| 层级 | 手段 | 作用 |
|------|------|------|
| 1 结构尺度 | 增大 Large Swing 的 L/R | 忽略毛刺拐点 |
| 2 同向合并 | `merge_pullback_atr` | 腿内小回撤不切段 |
| 3 最短时长 | `min_leg_bars` | 过滤数根 K 的噪声段 |
| 4 幅度门槛 | `min_move_atr` | 净移动不足标 RANGE |
| 5 效率门槛 | `leg_efficiency` | 震荡来回标 RANGE |
| 6 双层 Swing | Large 定边界、Small 定子阶段 | 大趋势不被小波动切断 |
| 7 闭合降级 | 不达标标 TRANSITION/RANGE | 不当成可交易趋势 |

**小趋势表现与对策：**

| 小趋势表现 | 用哪层过滤 |
|------------|------------|
| 几根 K 的毛刺高低点 | Large Swing（L/R 大） |
| 上涨中 2~3 根小回调 | `merge_pullback_atr` 同向合并 |
| 震荡里来回几根阳线 | `leg_efficiency` + `RANGE_LEG` |
| 幅度很小的波动 | `min_leg_return_atr` / `min_move_atr` |
| 时间很短的结构段 | `min_leg_bars` |
| 腿内小反向 | Small Swing 只标 PULLBACK，不 end Macro Leg |

**过滤参数建议（1h BTC）：**

| 参数 | 微观 Small | 宏观 Large | 说明 |
|------|------------|------------|------|
| `swing_left_bars` | 3 | 6~8 | 越大越钝 |
| `swing_right_bars` | 3 | 6~8 | 因果确认延迟 |
| `merge_pullback_atr` | — | 1.5~2.5 | 慢涨可取 2.0+ |
| `min_leg_bars` | — | 6~24 | 只要大趋势用 12~24 |
| `min_move_atr` | — | 1.0~1.5 | 至少 1~1.5 ATR 净移动 |
| `min_efficiency` | — | 0.25~0.30 | 低于此倾向 RANGE |

**调参方向：**

| 现象 | 调参 |
|------|------|
| 小趋势太多、腿太碎 | 增大 Large L/R；提高 `min_leg_bars`、`min_move_atr` |
| 大趋势被切成多段 | 增大 `merge_pullback_atr`；边界仅用 Large Swing |
| 慢涨识别不到 | 略降 `min_move_atr`，但提高 `min_leg_bars` 与 efficiency |
| 急跌反应太慢 | Large R 略减，或 `crash.py` 单独触发 `CRASH_LEG` |

#### 5.1.7 方案 A 完整在线算法

```text
初始化 macro_leg = None

每来一根 bar t：
  1. 更新 Small Swing、Large Swing（因果确认）
  2. 若 Large Swing 新拐点确认：
       a. 若有 macro_leg：尝试闭合
       b. 闭合时检查 duration、return_atr、efficiency
       c. 不达标 → 标 RANGE 或与邻腿合并
       d. 达标   → 输出 TrendLeg，分类 leg_type
       e. 从新区间开始 macro_leg = LEG_FORMING
  3. 若无 Large 拐点但方向延续：
       用 Small Swing 更新 sub_phase（PULLBACK / IMPULSE）
  4. 同向 Micro 段若回撤 < merge_pullback_atr：扩展 macro_leg，不切断
```

#### 5.1.8 方案 A 默认可回测参数组

```text
Small: L=3, R=3
Large: L=8, R=8
merge_pullback_atr = 2.0
min_leg_bars = 12
min_move_atr = 1.2
min_efficiency = 0.30
```

### 5.2 方案 B：变更点检测（v2 可选）

方案 B 在价格或收益序列上找 **统计性质突变** 的位置，输出**区间边界候选**；正式 Trend Leg 闭合仍需 Swing 或效率规则二次确认（见 §5.4）。

#### 5.2.1 区间边界的含义

```text
区间边界 = Trend Leg 的开始 bar 与结束 bar。
边界处：方向、波动、斜率、效率等在短窗口内显著变化。
```

#### 5.2.2 观测序列（严格因果）

| 序列 | 公式要点 | 适合检测 |
|------|----------|----------|
| ATR 归一化收益 | `r_t = (close_t - close_{t-1}) / ATR_{t-1}` | 方向/幅度突变 |
| 滚动斜率 | `slope_t = (close_t - close_{t-W}) / (W * ATR_t)` | 慢涨↔震荡 |
| 滚动波动 | `vol_t = std(r_{t-W:t})` | crash/surge |
| 趋势效率 | `eff_t = \|sum(r)\| / sum(\|r\|)` 在窗口 W 内 | 趋势↔盘整 |

推荐多通道联合：`x_t = [r_t, slope_24, vol_12, eff_24]`，对各通道变点投票或联合 BOCPD。

#### 5.2.3 检测算法

| 算法 | 在线/因果 | 适用场景 |
|------|-----------|----------|
| **BOCPD**（推荐） | 是 | 在线变点概率；急跌、波动突变 |
| **CUSUM** | 是 | 轻量；缓慢漂移（慢涨↔盘整） |
| **PELT** | 离线 | 整段历史最优变点；仅用于 Teacher 标注或滚动窗口近似 |

**BOCPD 要点（1h）：**

```text
观测：每个 run 内 x_t ~ Normal(μ, σ) 或 Student-t；
超参 hazard λ：期望 run 长度 ≈ 1/λ（如 λ=1/48 → 平均腿 ~48 根）；
阈值 τ：变点概率 > 0.3~0.5 为候选；
冷却期：确认边界后 min_leg_bars 内不再报新边界。
```

**CUSUM 要点：** 检测累积偏差超阈，作 BOCPD 辅助通道。

**PELT 在线用法：** 仅用 `[t-L, t]` 滚动窗口，只采纳最靠近 `t` 的变点；或离线标注后训练“距下一变点 bar 数”的回归头。

#### 5.2.4 两阶段边界确认

```text
阶段 1 — 变点候选（敏感）
  BOCPD 在 x_t 上输出 changepoint_prob[t]
  若 prob > τ_cp → cp_candidate[t] = True

阶段 2 — 结构确认（稳健）
  在 cp_candidate 附近 R 根内，须满足之一才定为正式边界：
    - 因果 swing high/low 被确认；
    - 或 leg_efficiency < eff_min 且反向 slope 持续 M 根；
    - 或回撤 > leg_end_pullback_atr × ATR。
```

#### 5.2.5 边界闭合判定（伪代码）

```python
cp_score = bocpd_changepoint_prob(x_t)
swing_break = confirmed_swing_against_leg()
eff_collapse = leg_efficiency(window) < 0.20
pullback_break = adverse_excursion > leg_end_pullback_atr * ATR

boundary_signal = (
    (cp_score > 0.45 and swing_break)
    or (cp_score > 0.60)
    or (eff_collapse and pullback_break)
)

if boundary_signal and leg_duration >= min_leg_bars:
    close_leg_at(t - swing_right_bars)
    start_new_leg_at(t)
```

新腿开始不等于立即重仓：`LEG_FORMING` → 小仓或观望；`LEG_CONFIRMED` → 允许趋势升级。

#### 5.2.6 方案 B 参数建议（1h）

| 参数 | 建议初值 | 作用 |
|------|----------|------|
| `bocpd_hazard` | 1/36 ~ 1/72 | 预期腿长 |
| `cp_threshold` | 0.45 | 变点候选 |
| `confirm_swing_right` | 3 | 因果确认 |
| `min_leg_bars` | 6~12 | 防碎裂 |
| `cooldown_bars` | 6 | 边界后静默 |
| `eff_min` | 0.20 | 趋势↔盘整 |

### 5.3 与模型切分的关系

`transformer_kit` 中已有 `AutoSegmentVQEncoder` 做形态 token 切分，那是 **特征工程层** 切分，与交易层 Trend Leg 不同：

```text
VQ segment：服务模型 embedding，粒度细、无金融语义；
Trend Leg  ：服务持仓与风控，粒度粗、有方向与阶段语义。
```

两者不合并；训练侧可用 Trend Leg 标签监督 Student 模型（见 §12）。

### 5.4 方案 A/B 混合策略（推荐生产）

| 环节 | 方案 A | 方案 B |
|------|--------|--------|
| 边界主信号 | Large Swing 拐点 | 统计变点 |
| 滞后 | R 根确认 | 可调 hazard，往往更早 |
| 震荡误切 | 较少 | 较多，需冷却期 |
| 急跌/崩跌 | 可能偏晚 | 更早发现 vol/μ 突变 |

**混合规则：**

```text
边界开始：Large Swing 确认为主；BOCPD 提前进入 LEG_FORMING；
边界结束：BOCPD 与 Large Swing 谁先到用谁，但须满足 min_leg_bars + 效率过滤；
腿内子阶段：仅 Small Swing，不用变点切段。
```

## 6. 区间分类体系

### 6.1 L1 Trend Leg 类型（8 类）

```python
class TrendLegType(str, Enum):
  SLOW_UP_LEG = "SLOW_UP_LEG"       # 低斜率、持续抬升、浅回撤
  FAST_UP_LEG = "FAST_UP_LEG"       # 高斜率、波动扩张、加速上行
  SLOW_DOWN_LEG = "SLOW_DOWN_LEG"   # 阴跌、缓坡下行
  FAST_DOWN_LEG = "FAST_DOWN_LEG"   # 急跌、大阴线主导
  CRASH_LEG = "CRASH_LEG"           # 模型失效下的崩跌腿（对接 crash.py）
  SURGE_LEG = "SURGE_LEG"           # 短时暴涨（可做多或仅观察）
  RANGE_LEG = "RANGE_LEG"           # 均线缠绕、无方向效率
  TRANSITION_LEG = "TRANSITION_LEG" # 腿间过渡，方向未确认
```

分类用 **腿级指标**（整段聚合，非单 bar）：

| 指标 | 公式要点 | 用途 |
|------|----------|------|
| `leg_return_atr` | (close_end - close_start) / ATR_start | 方向与幅度 |
| `leg_duration_bars` | end - start + 1 | 区分慢/快 |
| `leg_slope_atr` | leg_return_atr / leg_duration_bars | 斜率 |
| `leg_efficiency` | \|leg_return\| / sum(\|bar_return\|) | 趋势效率 |
| `leg_pullback_ratio` | max_adverse / \|leg_return\| | 回撤深度 |
| `leg_vol_expansion` | mean(ATR_leg) / mean(ATR_prev) | 识别 crash/surge |
| `leg_hh_hl_score` | 结构分（higher-high/low 占比） | 方向确认 |

分类规则（示意，具体阈值进 `TrendSegmentConfig`）：

```text
CRASH_LEG      : leg_vol_expansion >= 1.8 且 leg_return_atr <= -3 且 is_model_blind
FAST_DOWN_LEG  : leg_return_atr <= -2.5 且 leg_slope_atr <= -0.08
SLOW_DOWN_LEG  : leg_return_atr <= -1.0 且 leg_efficiency >= 0.35 且 leg_duration >= 12
FAST_UP_LEG    : leg_return_atr >= 2.5 且 leg_slope_atr >= 0.08
SLOW_UP_LEG    : leg_return_atr >= 0.8 且 leg_efficiency >= 0.30 且 leg_duration >= 24
RANGE_LEG      : |leg_return_atr| < 0.8 且 leg_efficiency < 0.25
TRANSITION_LEG : 其余或腿长度 < min_leg_bars
```

闭合 Macro Leg 时若不达标则降级：

```text
duration < 12 且 |return_atr| < 1.5  → TRANSITION / RANGE
efficiency < 0.25                     → RANGE
仍很短                               → 与相邻同向腿再合并
```

`SlowUptrendDetector` 命中 → 强制或加权 `SLOW_UP_LEG`；`CrashRegimeDetector` 命中 → `CRASH_LEG`。

### 6.2 L2 Sub-leg 阶段（腿内）

```python
class SubLegPhase(str, Enum):
  IMPULSE = "IMPULSE"           # 主推进
  PULLBACK = "PULLBACK"         # 腿内回撤
  BASE = "BASE"                 # 横盘整理
  BREAKOUT = "BREAKOUT"         # 整理后突破
  EXHAUSTION = "EXHAUSTION"     # 推进减弱
  LEG_END = "LEG_END"           # 腿即将/已经结束
```

判定：在当前 L1 腿内，用 **Small Swing** 或局部 ret/ATR 划分 sub-leg；**不开启新 L1 腿**。

### 6.3 L0 Regime（背景）

```python
class MarketRegime(str, Enum):
  BULL = "BULL"
  BEAR = "BEAR"
  NEUTRAL = "NEUTRAL"
  HIGH_VOL = "HIGH_VOL"
```

用 72/168 bar EMA 斜率 + 长窗口 realized vol 分位数。Regime **不直接触发开仓**，只调制仓位上限与腿类型先验。

### 7.1 TrendLeg（单段腿）

```python
class TrendLeg:
```

### 7.2 SegmentContext（引擎每 bar 输出）

```python
@dataclass
class SegmentContext:
    regime: MarketRegime
    active_leg: TrendLeg | None
    previous_leg: TrendLeg | None
```

### 7.3 与 TrendSignal 的关系

```text
每 bar 计算顺序：
  1. TrendSignalProvider.compute()   → 微观特征
  2. SlowUptrend / Crash             → 特化腿候选
  3. TrendSegmentEngine.update()     → 更新 L1/L2，输出 SegmentContext
  4. （可选）TrendLegClassifier    → 输出腿类型概率，与规则融合
```

## 8. 状态机

### 8.1 Trend Leg 状态机

```text
                    ┌─────────────────┐
                    │  NO_LEG / RANGE │
                    └────────┬────────┘
                             │ 同向 swing 累积 >= min_leg_bars
                             ▼
                    ┌─────────────────┐
              ┌────│  LEG_FORMING    │────┐
              │    └────────┬────────┘    │
              │ 结构破坏     │ 结构确认     │ 超时未确认
              ▼             ▼             ▼
       ┌──────────┐  ┌──────────────┐  ┌──────────┐
       │ NO_LEG   │  │ LEG_CONFIRMED│  │ NO_LEG   │
       └──────────┘  └──────┬───────┘  └──────────┘
                            │ 反向 swing / 回撤超阈 / 效率崩塌 / 变点候选
                            ▼
                     ┌──────────────┐
                     │  LEG_CLOSING │
                     └──────┬───────┘
                            │ 新腿 forming 或回 NO_LEG
                            ▼
                     ┌──────────────┐
                     │  NO_LEG /    │
                     │ 新 LEG_FORMING│
                     └──────────────┘
```

关键：**只有进入 `LEG_CONFIRMED` 才允许趋势型加仓/升级**；`LEG_FORMING` 只允许 probe 或小仓。方案 B 的变点可提前触发 `LEG_FORMING`，但不单独触发重仓。

### 8.2 持仓与腿的绑定

`PositionState` 扩展字段：

```python
entry_leg_id: int | None
entry_leg_type: TrendLegType | None
held_through_sub_phases: list[SubLegPhase]
```

规则：

```text
开仓时记录 entry_leg_id；
只要 active_leg.leg_id 不变且未 LEG_CLOSING → 趋势退出投票权重降低；
腿切换为反向类型 → 触发 CLOSE_TREND_BROKEN 或减仓；
同向新腿（合并）→ 不强制平仓，更新 entry_leg_id。
```

### 9.1 开仓：腿类型优先于单 bar 信号

### 9.2 持仓：子阶段驱动退出

| Sub-leg | 多头行为 | 空头行为 |
|---------|----------|----------|
| `IMPULSE` | `HOLD_TREND` | `HOLD_TREND` |

| `EXHAUSTION` | `REDUCE_TREND_EXHAUSTION` | 同上 |
| `LEG_END` | `CLOSE_TREND_BROKEN` 或 runner 减仓 | 同上 |

### 9.3 升级条件（解决 trend_upgrade_count = 0）

```text
  AND bars_since_leg_start >= upgrade_min_bars（建议 4~8）
```

```python
def update(self, bar_idx, ohlc, atr, trend_signal, slow_ctx, crash_ctx) -> SegmentContext:
    sh_s, sl_s = self.swing_small.on_bar(bar_idx, high, low)
    sh_l, sl_l = self.swing_large.on_bar(bar_idx, high, low)

    return self._build_context()
```

## 12. Teacher-Student：标注工具 + 分类 Transformer

### 12.1 方案结论

**可行，且推荐作为 020 的 Phase E/F。** 本质是：

```text
Teacher（trend_label_tool，离线）
  - 可用 hindsight Large Swing / PELT 得到高质量腿边界
  - 输出逐 bar 因果对齐标签
  - 不参与实盘

Student（TrendLegClassifier，在线）
  - 输入：过去 T 根 K 线 + 因果特征（可含 0062e 输出）
  - 输出：趋势种类概率 + 辅助任务
  - 推理时无全局未来信息
```

与 017 best-point 原则相同：**标签可 hindsight，交易/推理必须因果。** Student 定位为 **辅助信号**（填充 `SegmentContext` 概率字段），不替代风控与 0062e 主开仓逻辑。

### 12.2 成立前提

| 条件 | 说明 |
|------|------|
| 标签与特征时间对齐 | bar `t` 的标签表示“在 t 仅看 ≤t 数据时老师的结论”，不能把腿结束后才确定的类型原样标在腿中间每一根 |
| 训练特征因果 | 学生输入不含未来 swing、未来腿统计量 |
| 延迟一致 | 显式 `is_leg_confirmed` 任务，区分 FORMING / CONFIRMED |
| walk-forward 评估 | 按时间切分，禁止打乱 bar |

### 12.3 Teacher：`tools/trend_label_tool.py`

**输入：** OHLCV parquet/csv，可选 0062e 预测列（仅作特征，不作标签）。

**内部：** hindsight Large Swing 切腿 → 腿级分类 → **回溯**为逐 bar 因果标签。

**输出 schema（每 bar 一行）：**

| 列名 | 类型 | 说明 |
|------|------|------|
| `timestamp` | datetime | bar 时间 |
| `trend_leg_type` | int/str | 8 类 `TrendLegType` |
| `sub_phase` | int/str | 6 类 `SubLegPhase` |
| `leg_progress` | float | 腿内进度 0~1 |
| `is_leg_confirmed` | int | 0=FORMING，1=CONFIRMED |
| `bars_since_leg_start` | int | 腿龄 |
| `leg_id` | int | 所属腿 ID（复盘用） |
| `teacher_confidence` | float | 老师置信度（可选） |

**因果回溯规则（对腿 [s, e]、类型 Y）：**

```text
对 t ∈ [s, e]：
  若 t < s + R_large（Large Swing 未确认）
    → trend_leg_type = TRANSITION，is_leg_confirmed = 0
  若 t ≥ s + R_large 且 t ≤ e - M_end
    → trend_leg_type = Y，is_leg_confirmed = 1
  若 t 接近 e（效率崩塌 / 反向 swing 前 M_end 根）
    → sub_phase = EXHAUSTION 或 LEG_END
leg_progress[t] = (t - s) / max(e - s, 1)
```

Teacher 可用比在线更优的参数（更大窗口、离线 PELT 辅助边界），但 **输出必须按上表因果化**。

### 12.4 Student：`TrendLegClassifier`

**结构：**

```text
输入：[B, T, F]  过去 128~512 根 1h 特征（OHLCV 派生 + 可选 0062e）
骨干：CausalTransformer（与 MarketStateModel 同源）
输出头：
  - trend_leg_type_head  → 8 类 softmax
  - sub_phase_head         → 6 类 softmax
  - leg_progress_head      → sigmoid
  - leg_confirmed_head     → binary
```

**损失：**

```text
L = CE(leg_type) + 0.5*CE(sub_phase) + 0.3*Huber(progress) + 0.2*BCE(confirmed)
+ class_weights 或 focal（抑制 RANGE 主导）
```

**禁止：** 把 Teacher 的离散规则特征（如“未来 swing 是否确认”）直接喂给学生；只喂因果特征，用 Teacher 标签监督。

### 12.5 在线融合

```text
规则 TrendSegmentEngine  → SegmentContext（硬边界、可解释）
Student 概率            → p_slow_up_leg, p_fast_down_leg, ...
融合示例：
  leg_type = argmax(student_prob) 仅当 student_conf > 0.6 且与规则不冲突
  否则以规则为准
  RuleEngine 仍保留 BLOCK / 风控硬闸
```

| 维度 | 仅规则在线 | 规则 + Student |
|------|------------|----------------|
| 可解释性 | 高 | 中 |
| 边界抖动 | 硬切换 | 概率平滑 + 迟滞 |
| 维护成本 | 调参 | 重标注 + 重训 |
| 风险 | 规则盲区 | 分布外失效 |

### 12.6 风险与对策

| 风险 | 对策 |
|------|------|
| 标签噪声 | 固定 Teacher 参数版本；只保留高 `teacher_confidence` 样本 |
| 类别不平衡 | focal / class weight；按腿过采样稀有类 |
| 标签泄露 | 强制 `is_leg_confirmed` 与 FORMING 段标签 |
| 与 0062e 重叠 | 任务不同：0062e 短周期 edge，Student 中周期结构 |
| 虚高准确率 | 看 macro-F1、confirmed 段 F1、与老师 kappa、leg_coverage |

### 12.7 Student 验收指标

| 指标 | 说明 |
|------|------|
| `macro_f1_leg_type` | 8 类宏平均 F1 |
| `f1_confirmed_only` | 仅 `is_leg_confirmed=1` 的 bar |
| `kappa_vs_teacher` | 与老师一致性 |
| `walk_forward_stable` | 多折时间外推 F1 波动 |
| 交易侧 | `leg_coverage_ratio`、`missed_slow_up_legs` 较纯规则改善 |

## 14. 风险与约束

```text
1. swing 确认延迟：swing_right_bars 带来滞后，用 LEG_FORMING 小仓；Student 不得偷跑未来；
2. 腿合并过激进：efficiency 过滤；震荡误并为趋势时调高 min_efficiency；
3. 腿切过碎：Large Swing + min_leg_bars + merge_pullback_atr；
4. 方案 B  alone 易碎：变点须二次 Swing/效率确认；
5. Teacher 标签禁止直接用于实盘：仅 Student 或因果 TrendSegmentEngine；
6. 020 不替代 0062e：短周期 edge vs 中周期持仓尺度。
```
