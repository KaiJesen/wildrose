# BTC 合约交易规则设计（基于当前模型输出）

## 1. 目标与边界

- 目标：将当前 `market_state` 模型输出转成可执行的 BTCUSDT 永续合约规则。
- 使用周期：`1h` 决策（每根 K 线收盘后决策，下一根开盘执行）。
- 账户目标：先追求回撤可控和稳定性，其次再优化收益。
- 适用模型特性（来自 `0065a`）：累计方向能力优于逐步收益能力，`cum_return_ic` 可用，`step return_ic` 偏弱。
- 结论：本规则是“中低频趋势+风控优先”框架，不做高频抢点。

> 说明：本文档是策略工程设计，不构成投资建议。实盘前需先纸面与仿真验证。

---

## 2. 输入信号标准化

将模型输出统一为以下字段（若字段名不同，做一层映射）：

- `p_up`：上涨概率（方向类 c2）
- `p_down`：下跌概率（方向类 c0）
- `p_flat`：震荡概率（方向类 c1）
- `p_risk`：高风险概率（风险头）
- `pred_ret_1...pred_ret_5`：未来 1~5 根预测收益（或可换算收益）
- `pred_cum_ret_5`：未来 5 根累计收益预测（若无则用 `pred_ret_1...5` 加权近似）
- `ts`：信号时间戳

派生变量：

- `edge = p_up - p_down`
- `conf = abs(edge)`
- `cum_edge = pred_cum_ret_5`
- `risk_ok = (p_risk <= 0.38)`（初始阈值，可周更）

---

## 3. 交易状态机

只允许三个持仓状态：`LONG` / `SHORT` / `FLAT`。

### 3.1 开仓条件

**做多开仓（`FLAT -> LONG`）**

- `edge >= 0.08`
- `p_up >= 0.42`
- `p_flat <= 0.34`
- `cum_edge > 0`
- `risk_ok == true`

**做空开仓（`FLAT -> SHORT`）**

- `edge <= -0.08`
- `p_down >= 0.42`
- `p_flat <= 0.34`
- `cum_edge < 0`
- `risk_ok == true`

### 3.2 持仓继续条件

- LONG 持仓：`edge > -0.03` 且 `cum_edge >= 0`
- SHORT 持仓：`edge < 0.03` 且 `cum_edge <= 0`
- 若不满足，先减仓到 50%；连续 2 根不满足则平仓。

### 3.3 平仓条件（任一触发）

- 方向反转：LONG 时 `edge <= -0.05`；SHORT 时 `edge >= 0.05`
- 风险升高：`p_risk >= 0.48`
- 时间止盈止损窗结束：持仓超过 6 根 1h K（覆盖 5 步预测周期后再给 1 根缓冲）
- 触发硬止损 / 止盈（见第 5 节）

---

## 4. 仓位与杠杆规则

### 4.1 单笔风险预算

- 每笔最大风险：账户净值的 `0.40%`。
- 单方向总暴露上限：账户净值名义仓位 `1.6x`。
- 最大杠杆：`3x`（即使交易所允许更高也不提升）。

### 4.2 动态仓位公式

定义：

- `conf_norm = min(conf / 0.20, 1.0)`
- `risk_penalty = clip((0.48 - p_risk) / 0.20, 0.0, 1.0)`
- `pos_scale = 0.35 + 0.65 * conf_norm * risk_penalty`

执行仓位：

- `target_notional = base_notional * pos_scale`
- `base_notional` 由“0.40% 风险预算 + 止损距离”反推得出。

解释：

- 信号强、风险低时接近满额；
- 信号弱或风险高时自动降仓；
- 保证不同波动环境下风险可比。

---

## 5. 风控与交易保护

### 5.1 价格风控（硬规则）

- 初始止损：`1.2 * ATR(14)`
- 分批止盈：`1.0 * ATR` 减仓 40%，`2.0 * ATR` 再减仓 40%，余仓追踪止盈
- 追踪止盈：从浮盈峰值回撤 `0.8 * ATR` 触发

### 5.2 组合风控（账户级）

- 日内最大回撤：`-2.0%`，触发后当天停止开新仓
- 周内最大回撤：`-5.0%`，触发后进入观察模式，仅允许 30% 试探仓位
- 连续亏损 3 笔：冷却 `12` 根 1h K（沿用当前回测脚本默认思路）

### 5.3 市场事件过滤

- 高波动新闻窗口（CPI/FOMC/非农等）前后各 1 根 1h K 不开新仓
- 资金费率绝对值过高（建议阈值 `|funding| > 0.05% / 8h`）时仅半仓交易
- 交易所异常（延迟、断连、滑点突增）立即切换 `FLAT`

## 6. 执行流程（可直接程序化）

1. 每根 1h 收盘，拉取最新模型输出与行情数据。
2. 计算 `edge/conf/cum_edge` 与风控变量（ATR、资金费率、当日回撤）。
3. 先做“账户级风控检查”（若失败直接不交易）。
4. 按状态机判断：开仓 / 持有 / 减仓 / 平仓。
5. 计算目标仓位并下单（限价优先，超时未成交再转市价小滑点保护）。
6. 记录执行日志，更新绩效看板。

---

## 7. 参数校准与验收节奏

### 7.1 周度校准（滚动）

- 训练外最近 `180` 天为开发窗，最近 `30` 天为验证窗。
- 每周重估阈值：`edge` 开仓阈值、`p_risk` 阈值、`p_flat` 过滤阈值。
- 仅当验证窗通过才更新实盘参数，否则沿用旧参数。

### 7.2 验收门槛（建议）

- 验证窗 Sharpe >= `1.0`
- 胜率 >= `48%`
- Profit Factor >= `1.15`
- 最大回撤 >= `-12%`（即不低于 -12%）
- 手续费后净收益 > Buy&Hold（同窗口）

---

## 8. 监控与复盘模板

每笔交易至少记录：

- 信号：`p_up/p_down/p_flat/p_risk/edge/cum_edge`
- 执行：方向、仓位、杠杆、成交价、滑点、手续费
- 结果：持仓时长、PnL、退出原因（反转/止损/止盈/时间窗/风控）
- 环境：当时 ATR、资金费率、市场事件标记

每周复盘四件事：

1. 亏损主要来自“错方向”还是“仓位过大”。
2. `p_risk` 阈值是否过松（若大额亏损多出现在高 `p_risk`，则收紧）。
3. `p_flat` 过滤是否有效（震荡期交易次数与净值关系）。
4. 信号衰减速度（第 1~6 根持仓收益分布是否仍匹配）。

---

## 9. 先行试运行建议（两周）

- 第 1 周：仅纸面交易，完整跑流程与日志，不下真实订单。
- 第 2 周：小仓位实盘（最大 0.25x 名义），验证执行质量与滑点。
- 两周后若满足：
  - 执行偏差（计划价 vs 成交价）可控；
  - 无风控违规；
  - 绩效不劣于纸面回放；
  则放大到正式参数的 50%，再观察 2 周。
