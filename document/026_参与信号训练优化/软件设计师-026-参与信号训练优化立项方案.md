## 2. 立项背景

### 2.1 024/025 已证明什么

| 结论 | 证据 |
|------|------|
| 参与信号 **可学** | 024 constrained valid `participation_auc ≈ 0.572` |
| TEQ 消费路径 **有效** | B0 teq=3，teq_pnl +0.80%，return 9.01% |
| 规则层 coverage 上限 | B0 coverage 26.7%，距探索线 28% 差 1.3pp |
| 多通道规则扩展 **失败** | A3a return 7.25%（coverage 不变）；A3b return 6.81%（coverage 30% 但 −2.2pp） |
| Pareto 前沿在 c1_pw20 上 **不可突破** | 024 w_part 膝点与 025 A3b 同构：coverage↑ return↓ |

### 2.2 025 结案后的根因链（架构师 §6.1）

```text
c1_pw20 的 participate_score 在慢涨/标准通道无区分力
  → ideal_participate 为严格 binary hindsight，正样本极少
  → 参与头仅读全局池化向量，未访问 bar 级序列上下文
  → 腿内参与需要 12~48 bar 视野，但多视野头在 constrained 训练下被 cum_IC≤5% 约束压制
  → 参与特征与短期 cum return 特征结构性冲突
```

### 2.3 无约束训练的对照（方案矩阵 §2.1）

| 训练模式 | valid part_auc | 全链路 return | 说明 |
|----------|----------------|---------------|------|
| constrained（024 B0） | **0.572** | **9.01%** | legacy 稳定，参与弱 |
| 无约束 0065a-1 | 0.611~0.633 | −8% 量级 | legacy 头漂移，不可投产 |

**设计师裁定**：026 必须在 **可控 legacy 退化**（≤10% cum_IC）前提下提升 `participation_auc`，禁止回到无约束全量重训路径。

### 2.4 跨机复现基线（Phase 0 硬门禁）

| 资产 | 路径 |

## 4. 目标与范围

### 4.1 总目标（分层验收）

**026 成功定义**：

```text
主目标 = 探索线双门 PASS（test return ≥8.84%，coverage ≥28%）
       + valid participation_auc ≥0.62
       + cum_return_ic 相对 0062e 退化 ≤10%（硬门禁，见 §5.9）
       + multi-seed（≥3）稳定性抽查；

Phase 4 投产线（return ≥9.31%，coverage ≥35%）为条件评估，非本版必达；
```

## 5. 方案设计

### 5.1 推荐路径：F1 组合（架构师方案矩阵 §6）

```text
Phase 0: C3（ParticipationAttn cross-attn）           ← 最高性价比单点
Phase 1: C1（腿上下文特征）+ D1（采样增强）          ← 按 §4.1.1 分级必做/可选
Phase 2: A1（三阶标签）或 B1（渐进解冻）            ← 按 §4.1.2 裁定
Phase 3: TEQ wp 重调 + B0 vs 026-best 全链路 A/B   ← 仅模型轨达标后
```

备选激进路径 **F2**（A2 + B3 + C2）仅当 Phase 0~1 后 `part_auc < 0.60` 且架构师书面批准时启动。

### 5.2 WP0 — 参与头 cross-attn（C3，P0）

**问题**：当前 `participation_logit_*` 为 `Linear→GELU→Linear`，输入仅全局池化向量 `pooled [B,D]`，丢失 bar 级序列信息。

**改动**：`transformer_kit/pattern_model.py`

```text
ParticipationAttn（参考 HorizonReturnHead L231-264，简化版）：
  query = learned_param [1,1,D] + pooled.unsqueeze(1)   # 或 concat 变体
  context = MultiheadAttention(query, trunk_tokens [B,S,D], mask=~seg_mask)
  participation_logit = Linear(LayerNorm(context).squeeze(1)) → [B]

训练：冻结 trunk，仅训 Q/K/V 投影 + 输出 MLP
```

| 对比项 | HorizonReturnHead | ParticipationAttn（C3） |
|--------|-------------------|-------------------------|
| query shape | `[1, horizon, D]` | `[1, 1, D]` |
| output | step + cum | 单个 logit `[B]` |
| mask | `~seg_mask` | 同 |
| trainable | Q/K/V + MLP | 同（trunk 冻结） |

**验收（valid）**：`participation_auc ≥ 0.60`（Phase 0 门）；晋级路径见 §4.1.1。

### 5.3 WP1 — 腿上下文特征（C1，P0）

**改动**：`segment_dataset.py` / `pattern_model.py` bar 特征维 +3~4：

| 特征 | 说明 |
|------|------|
| `leg_progress_ratio` | [0,1]，当前 bar 在腿内位置 |
| `bars_since_leg_start` | `log(1+bars)` 归一化 |
| `leg_type_embedding` | 可学习 4 维，concat 至 bar embedding |

**版本 hash 冻结（评审 H4 同类）**：label parquet 元数据与 checkpoint 须写入并比对：

```text
segment_module_version
trend_signal_config_hash
```

训练与推理 `TrendSegmentEngine` 不一致时 **拒绝加载或告警**。

### 5.4 WP2 — 采样增强（D1，P0）

**改动**：`training.py` DataLoader 权重

```text
ideal_participate tier=2（或 ideal=1）:  weight = 10.0
ideal=0, confirmed leg, flat:            weight = 2.0   # hard negative
其他:                                    weight = 0.5
每 batch 正样本占比目标 ≥ 20%（当前 < 5%）
```

### 5.5 WP3 — 三阶有序标签（A1，P1）

**改动**：`trading_system/leg_participation_labels.py`、`training.py`

```text
long tier 2（强参与）: ROI>0 ∧ MAE≤1.5 ATR ∧ progress≤0.35   # 等同当前 ideal=1
long tier 1（弱参与）: ROI>0 ∧ MAE≤2.5 ATR ∧ progress≤0.50
tier 0: 其他

short tier 2: ROI<0 ∧ MAE≤1.5 ATR（MAE 用 high[s] 对称计算，继承 024 §6.2.3）
short tier 1: ROI<0 ∧ MAE≤2.5 ATR
short 正样本 leg_type: FAST_DOWN_LEG only（继承 024 H2）
```

**损失**：固定使用 **CORAL**（有序回归）；不在 valid/test 间切换损失类型。

### 5.6 WP4 — 渐进解冻（B1，P1 备选）

```text
Stage 1（~1 周）：冻结 trunk，训练 participation + horizon 头至 part_auc 收敛
Stage 2（~1 周）：解冻 trunk 最后 2 层 + layer_norm，lr=1e-5
                  L = L_part + L_leg_dir + 0.1·L_legacy，cum_IC≤10%
                  逐 epoch 监控 cum_IC；退化 >8% 写入 REPORT 高亮
```

选型见 §4.1.2；**A1 优先于 B1**。

### 5.7 WP6 — 全链路验收（Phase 3）

**实验臂**（冻结 `configs/trading_rule_v023_phase1c_0062e.json` 规则主体）：

| 臂 | 模型 | 规则 | 说明 |
|----|------|------|------|
| A0 | 0062e | phase1c | 023 冻结参照 |
| B0 | c1_pw20 | 024 TEQ wp=0.35 | 025 研究基线 |
| M1 | 026 Phase0（C3） | 同 B0 | 最小改动验证 |
| M2 | 026 Phase1（C3+C1+D1） | 同 B0 | 主候选 |
| M3 | 026 Phase2（+A1 或 +B1） | 同 B0 + valid TEQ 重调 | 终态候选 |

**TEQ 重调（评审 H4）**：

```text
必须基于新 checkpoint 的 valid participate_score 重新拟合 calibrator；
禁止直接复用 backtest/v024_constrained/teq_edge_calibration.json。
仅 valid 网格 w_part；test 一次性报告。
```

**通道条件复测（评审 B2，非自动）**：

须 **同时** 满足以下全部条件，且 **架构师书面批准** 后方可执行 **1 个** 对照臂：

```text
1. Phase 3 探索 return ≥ 8.84%；
2. coverage ∈ [27%, 28%) — 差目标 <1pp；
3. 架构师书面批准（不可脚本自动触发）；
4. 仅 1 臂：B0 + std_trend_gate 或 B0 + slow_up_gate（二选一），禁止全矩阵调参。
```

### 5.8 训练约束修订（相对 024）

| 参数 | 024 | 026 |
|------|-----|-----|
| `cum_ic_min_ratio` | 0.95（≤5% 退化） | **0.90**（≤10% 退化） |
| `participation_weight` | 2.0（c1_pw20） | 初值 2.0；见 §5.9 配方 |

### 5.9 cum_IC 退化硬门禁与 C3 训练配方（评审 §3.3、H3）

**cum_IC 监控（Phase 0~2 每个 checkpoint）**：

```text
退化 ≤ 8%：正常推进；
退化 ∈ (8%, 10%]：训练可继续，REPORT 高亮警告；
退化 > 10%：立即回退上一步权重，不得进入下一阶段；
若退化 > 10% 且归因 C3 单独训练 → 增加 0.05·L_legacy 蒸馏项后重训。
```

|------|------|------|

| `epochs` | 15 | early-stop patience 6 |
| `participation_weight` | 2.0 | 配合 D1 可上调 |
| `optimizer` | AdamW lr=3e-4 | 仅 ParticipationAttn 参数组 |
| `early_stop_metric` | `0.6·part_auc + 0.4·cum_ic_ratio` | composite；part_auc 下限 0.58 |

## 6. 验收与里程碑

## 10. 立项结论（设计师）

**批准 026 立项（有条件）**，以架构师 **F1 组合** 为主路径，并吸收 v1.1 修订：

2. **C3 晋级分级**：快速晋级门槛 **0.65**（非 0.62）；[0.62,0.65) 必须 C1+D1；
3. **双轨验收**：模型轨达标前不进全链路；
4. **cum_IC 10% 硬门** + Phase 0 训练配方冻结；
5. **通道复测**须架构师书面批准，单臂对照；
6. **TEQ 校准**必须基于新 checkpoint 重拟合。

预计工期：**1~3 周**（取决于 C3 是否达 0.65 快速晋级）。
