## 2. 立项背景

### 2.1 024/025 已证明什么

| 结论 | 证据 |
|------|------|
| 参与信号 **可学** | 024 constrained valid `participation_auc ≈ 0.572` |
| TEQ 消费路径 **有效** | B0 teq=3，teq_pnl +0.80%，return 9.01% |
| 规则层 coverage 上限 | B0 coverage 26.7%，距探索线 28% 差 1.3pp |
| 多通道规则扩展 **失败** | A3a return 7.25%（coverage 不变）；A3b return 6.81%（coverage 30% 但 −2.2pp） |
| Pareto 前沿在 c1_pw20 上 **不可突破** | 024 w_part 膝点与 025 A3b 同构：coverage↑ return↓ |

### 2.2 025 结案后的根因链（架构师 §6.1）

```text
c1_pw20 的 participate_score 在慢涨/标准通道无区分力
  → ideal_participate 为严格 binary hindsight，正样本极少
  → 参与头仅读全局池化向量，未访问 bar 级序列上下文
  → 腿内参与需要 12~48 bar 视野，但多视野头在 constrained 训练下被 cum_IC≤5% 约束压制
  → 参与特征与短期 cum return 特征结构性冲突
```

### 2.3 无约束训练的对照（方案矩阵 §2.1）

| 训练模式 | valid part_auc | 全链路 return | 说明 |
|----------|----------------|---------------|------|
| constrained（024 B0） | **0.572** | **9.01%** | legacy 稳定，参与弱 |
| 无约束 0065a-1 | 0.611~0.633 | −8% 量级 | legacy 头漂移，不可投产 |

**设计师裁定**：026 必须在 **可控 legacy 退化**（≤10% cum_IC）前提下提升 `participation_auc`，禁止回到无约束全量重训路径。

### 2.4 跨机复现基线（Phase 0 硬门禁）

| 资产 | 路径 |
|------|------|
| B0 固化包 | `prod/v1.1.1/` |
| 冒烟验收 | `bash prod/v1.1.1/scripts/verify_phase0.sh` |
| 025 Phase 0 | `python examples/run_v025_phase0.py --skip-train` |

**未过 B0 复现门不得启动 026 训练实验。**

## 4. 目标与范围

### 4.1 总目标（分层验收）

**026 成功定义**：

```text
主目标 = 探索线双门 PASS（test return ≥8.84%，coverage ≥28%）
       + valid participation_auc ≥0.62
       + cum_return_ic 相对 0062e 退化 ≤10%
       + multi-seed（≥3）稳定性抽查；

Phase 4 投产线（return ≥9.31%，coverage ≥35%）为条件评估，非本版必达；
未达投产线时维持 v1.1.0 prod，通过模型可进入研究基线（v1.1.2+）迭代。
```

| 阶段 | 最低要求 | 备注 |
|------|----------|------|
| **Phase 0** | B0 复现 PASS + **C3** cross-attn 参与头 valid `part_auc ≥ 0.60` | **硬门禁**；3 天内出信号 |
| **Phase 1** | **C1** 腿上下文特征 + **D1** 采样增强；`part_auc ≥ 0.62` | 冻结 trunk，仅训参与相关参数 |
| **Phase 2** | **A1** 三阶有序标签 **或** **B1** 渐进解冻（二选一，由 Phase 1 结果裁定） | cum_IC 退化 ≤10% |
| **Phase 3** | 全链路 A/B：0062e+A0 / B0 / 026-best + TEQ 重调参 | 探索双门一次性 test 验收 |
| **Phase 4** | 投产线评估 + 项目经理书面批准 | 大概率需后续迭代 |

**探索线（test，一次性验收，继承 024/025）**：

| 指标 | 门槛 |
|------|------|
| `total_return` | ≥ **8.84%** |
| `leg_count_coverage_ratio` | ≥ **28%** |

**模型轨（valid）**：

| 指标 | 门槛 | 说明 |

## 5. 方案设计

### 5.1 推荐路径：F1 组合（架构师方案矩阵 §6）

```text
Phase 0: C3（参与头 cross-attn）                    ← 最高性价比单点
Phase 1: C1（腿上下文特征）+ D1（采样增强）         ← 低成本叠加
Phase 2: A1（三阶标签）或 B1（渐进解冻）           ← 按 Phase 1 part_auc 裁定
Phase 3: TEQ wp 重调 + B0 vs 026-best 全链路 A/B
```

备选激进路径 **F2**（A2 + B3 + C2）仅当 Phase 0~1 后 `part_auc < 0.60` 且架构师书面批准时启动。

### 5.2 WP0 — 参与头 cross-attn（C3，P0）

**问题**：当前 `participation_logit_*` 为 `Linear→GELU→Linear`，输入仅全局池化向量 `pooled [B,D]`，丢失 bar 级序列信息。

**改动**：`transformer_kit/pattern_model.py`

```text
learned_query [1,1,D]
context = CrossAttention(query, trunk_tokens [B,S,D])   # trunk 可冻结
participation_logit = Linear(context.squeeze) → [B]

实现参考现有 HorizonReturnHead（~30 行参与头版本）
训练：冻结 trunk，仅训 Q/K/V 投影 + 输出层 + participation bias
```

**验收（valid）**：`participation_auc ≥ 0.60`（Phase 0 门）；≥0.62 则晋级 Phase 3 快速路径。

### 5.3 WP1 — 腿上下文特征（C1，P0）

**改动**：`segment_dataset.py` / `pattern_model.py` bar 特征维 +3~4：

| 特征 | 说明 |
|------|------|
| `leg_progress_ratio` | [0,1]，当前 bar 在腿内位置 |
| `bars_since_leg_start` | `log(1+bars)` 归一化 |
| `leg_type_embedding` | 可学习 4 维，concat 至 bar embedding |

**与 024 标签对齐**：特征在推理时可由 `TrendSegmentEngine` 在线计算，训练时与 `build_leg_participation_labels.py` 一致。

### 5.4 WP2 — 采样增强（D1，P0）

**改动**：`training.py` DataLoader 权重

```text
ideal_participate = 1（或 tier=2）:  weight = 10.0
ideal = 0, confirmed leg, flat:       weight = 2.0   # hard negative
其他:                                 weight = 0.5
每 batch 正样本占比目标 ≥ 20%（当前 < 5%）
```

### 5.5 WP3 — 三阶有序标签（A1，P1）

**改动**：`trading_system/leg_participation_labels.py`、`training.py`

```text
tier 2（强参与）: ROI>0 ∧ MAE≤1.5 ATR ∧ progress≤0.35     # 等同当前 ideal=1
tier 1（弱参与）: ROI>0 ∧ MAE≤2.5 ATR ∧ progress≤0.50
tier 0（不参与）: 其他

损失：ordered BCE 或 CORAL；正样本量预期 ×2~3
同时 cum_IC 约束放宽至 ≤10%（架构师方向 2）
```

**short 方向**：仍仅 `FAST_DOWN_LEG` 正样本（继承 024 H2，与 TEQ 消费对齐）。

### 5.6 WP4 — 渐进解冻（B1，P1 备选）

```text
Stage 1（~1 周）：冻结 trunk，训练 participation + horizon 头至 part_auc 收敛
Stage 2（~1 周）：解冻 trunk 最后 2 层 + layer_norm，lr=1e-5
                  L = L_part + L_leg_dir + 0.1·L_legacy，cum_IC≤10%
```

当 A1 与 B1 冲突时：**优先 A1**（标签改动风险低于 trunk 解冻）。

### 5.7 WP6 — 全链路验收（Phase 3）

**实验臂**（冻结 `configs/trading_rule_v023_phase1c_0062e.json` 规则主体）：

| 臂 | 模型 | 规则 | 说明 |
|----|------|------|------|
| A0 | 0062e | phase1c | 023 冻结参照 |
| B0 | c1_pw20 | 024 TEQ wp=0.35 | 025 研究基线 |
| M1 | 026 Phase0（C3） | 同 B0 | 最小改动验证 |
| M2 | 026 Phase1（C3+C1+D1） | 同 B0 | 主候选 |
| M3 | 026 Phase2（+A1 或 +B1） | 同 B0 + valid TEQ 重调 | 终态候选 |

**TEQ 重调**：仅 **valid** 网格 `w_part`（继承 024 sweep 范围）；test 一次性报告。

**通道复测（条件触发）**：仅当 `part_auc ≥ 0.62` 且探索 return 达标但 coverage 差 <1pp 时，启用 025 已实现的 `std_trend_gate` / `slow_up_gate` 做 **对照臂**，不重开 025 主路径。

### 5.8 训练约束修订（相对 024）

| 参数 | 024 | 026 |
|------|-----|-----|
| `cum_ic_min_ratio` | 0.95（≤5% 退化） | **0.90**（≤10% 退化） |
| `participation_weight` | 2.0（c1_pw20） | 初值 2.0；可配合 D1 采样 |
| `early_stop_metric` | composite | composite + **part_auc 下限门** |
| `freeze_legacy_market_state_heads` | true（constrained） | Phase 0~1 **true**；Phase 2 B1 按阶段解冻 |

## 8. 风险与退场

| 风险 | 等级 | 缓解 |
|------|------|------|
| C3 无效（part_auc < 0.58） | 中 | 并行启动 A1；架构师裁定 F2 |
| trunk 解冻后 legacy 崩溃 | 高 | B1 仅解冻顶层；cum_IC 硬门；A1 优先于 B1 |
| part_auc 升但全链路 coverage 仍 <28% | 中 | 条件启用 025 通道 + E1 通道校准 |
| TEQ 重调再现 024 膝点 | 高 | 接受 Pareto 报告；不降低探索 return 门 |
| 正样本仍不足 | 中 | A1 三阶 + D1；必要时 A2 连续分数 |

**退场条件**：

```text
Phase 0+1 后 valid part_auc < 0.58 且 A1 无效 → 026 暂停，架构师评估 F2 或 027 新数据窗口；
Phase 3 探索双门 FAIL → 026 结案，维持 v1.1.0 prod + v1.1.1 研究基线，知识资产入库。
```

## 10. 立项结论（设计师）

**建议批准 026 立项**，以架构师 **F1 组合** 为主路径：

1. **Phase 0 必做 C3**（参与头 cross-attn），3 天内验证训练侧是否可突破 part_auc 0.60；
2. **Phase 1 叠加 C1 + D1**，目标 part_auc ≥ 0.62；
3. **Phase 2 按结果选 A1 或 B1**，cum_IC 放宽至 10%；
4. **Phase 3 全链路探索双门** 为唯一 test 成功定义；
5. **规则层零改动** 为默认路径，除非 Phase 3 coverage 差 <1pp 且 return 已达标。

预计工期：**1~3 周**（取决于 Phase 0 C3 是否触发快速晋级）。
