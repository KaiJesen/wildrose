## 2. 总体结论

v1.1 方案已经完成实质性修订，023 从“参与率提升方向”推进到“具备工程入口条件”的状态。

上一轮评审的三项阻塞问题整体已关闭：

1. **收益 / 回撤门槛已统一**：`max_drawdown` 明确采用负数口径，推荐配置要求 `total_return >= v022`，`v022 × 0.95` 被降级为单 PR 探索容忍线。
2. **覆盖率指标已拆成三层**：`leg_count_coverage_ratio`、`leg_bar_coverage_ratio`、`leg_pnl_capture_ratio` 均已定义，并增加 `effective_leg_covered` 防刷分条件。
3. **趋势资格开仓已纳入统一链路**：新增 `TrendEntryQualifier` / `TrendEntryQualification`，并要求经 `trend_bias + risk_budget` 后才允许 `OPEN_*_TREND_QUALIFIED`。

高优先级建议也基本吸收：

- Participation Floor 默认关闭，仅作为 A3-floor 实验项；
- 022 前置 gate、配置 hash、模块版本记录已写入；
- `RiskManager` 职责回归硬风险边界；
- Phase 1 拆为 slow_up、crash upgrade、trend-qualified 三个 PR；
- 慢涨假突破、再入场、加仓均增加归因指标。

因此复审结论为：

## 3. 初审阻塞项复审

### B1：收益 / 回撤硬门槛不一致

**复审状态：已关闭。**

v1.1 在 §5.1 与 §13 中明确：

| 层级 | 要求 |
|------|------|
| Phase 1 探索线 | `total_return >= v022 × 0.95`，仅 PR 内部分支诊断 |
| 推荐 config 硬门槛 | `total_return >= v022（9.31%）` |
| 回撤口径 | `max_drawdown` 为负数，`>= -2.0%` 表示不超过 2% 回撤 |
| 软目标 | `total_return >= 12%` 或 `excess_return` 相对 v022 提升 20% |

这消除了“收益退化仍通过最终验收”的风险。

**复审意见：通过。**

---

### B2：核心参与度指标公式未冻结

**复审状态：基本关闭，实施前需补方向一致性澄清。**

v1.1 已新增 §5.3，定义：

- `effective_leg_covered`
- `leg_count_coverage_ratio`
- `leg_bar_coverage_ratio`
- `leg_pnl_capture_ratio`
- `slow_up_watch_to_open_ratio`
- `slow_up_false_entry_count`
- 分通道 PnL

这解决了初审中“单一 `leg_coverage_ratio` 容易被 probe 刷分”的主要问题。

但当前公式中：

```text
overlap_bars = |{bar : position_side != FLAT} ∩ leg_bars|
```

仍缺少 **持仓方向与腿方向一致** 的要求。若系统在 `FAST_UP` 腿中持有空仓，或在 `FAST_DOWN` 腿中持有多仓，按当前公式也可能被计入 coverage。这会污染“趋势参与度”指标。

**实施前澄清建议：**

将 coverage 拆为：

```text
aligned_overlap_bars
counter_overlap_bars
```

并规定：

```text
leg_count_coverage_ratio / leg_bar_coverage_ratio
只统计 aligned_overlap_bars；
counter_overlap_bars 单独进入 counter_leg_participation_count；
若 counter_overlap_bars > 0，须与 HARD_BLOCK / probe 规则联动解释。
```

`leg_pnl_capture_ratio` 还应明确 denominator 采用正值机会收益，避免下跌腿分母为负：

```text
ideal_leg_pnl = abs(close_end - close_start) / close_start
```

或等价的同向理论收益。

**复审意见：不阻塞 Phase 0，但必须在 `eval_participation.py` 实现前写入公式。**

---

### B3：趋势资格开仓绕开统一偏置 / 风险预算链路

**复审状态：已关闭。**

v1.1 在 §6.3.3 中引入：

```text
TrendEntryQualifier
TrendEntryQualification
RuleEngine 读取 trend_bias + TrendEntryQualification + risk_budget
```

并明确禁止绕过：

```text
counter_level_* == HARD_BLOCK
CHOP_HARD

```

这符合 021/022 的架构边界：趋势资格是一个资格对象，不是 `rules.py` 重新直读多模块后的特判。

**复审意见：通过。**

## 5. 业务可行性复审

023 的业务目标在 v1.1 中更稳健：

- 最终推荐配置不得低于 v022 收益；
- 回撤上限明确；
- 覆盖率不再是单一指标；
- 慢涨、趋势资格、crash upgrade、再入场、加仓均要求分通道 PnL；
- Participation Floor 不进入默认推荐配置。

这避免了两个常见误区：

```text
为了提高交易次数而牺牲收益；
为了提高覆盖率而制造无效小仓。
```

**业务复审结论：通过。**

## 8. 实施前澄清项

以下 3 项不构成立项阻塞，但建议在 Phase 0 PR 或实现注释中明确：

### C1：coverage 必须区分 aligned / counter

将 §5.3 中 `overlap_bars` 改为：

```text
aligned_overlap_bars =
  position_side 与 leg_direction 同向的 overlap bars

counter_overlap_bars =
  position_side 与 leg_direction 反向的 overlap bars
```

最终 coverage 只统计 aligned overlap。

### C2：`leg_pnl_capture_ratio` 分母必须为正值机会收益

避免下跌腿或亏损腿导致 ratio 符号反转。建议：

```text
ideal_leg_pnl = abs(close_end - close_start) / close_start
```

并在分母过小时跳过或单列 `small_move_leg_count`。

### C3：响应文档中的章节索引需校准

`软件设计师-023-评审意见响应与修订说明.md` 中少量索引与 v1.1 实际章节不一致，例如：

- B3 写 §6.2.3，实际为 §6.3.3；
- H1 写 §6.4.2，实际为 §6.5.2；
- H3 写 §6.3.1，实际为 §6.4.1。

这不影响方案内容，但会影响后续审计与定位，建议顺手修正。
