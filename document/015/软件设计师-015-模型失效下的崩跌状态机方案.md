---

## 2. 当前版本结论

014b 已经修复了两个问题：

```text
1. 关闭 014a probe short 泛滥，收益从 014a+tuned 的 +2.81% 提升到 +8.35%；
2. 6/17 标准空头通过 trend_hold 持有 14 bars，避免固定 8 bars 提前退出。
```

但 014b 仍有明显短板：

```text
6 月初大段下跌仍未赚到；
sentinel_short_count = 0；
missed_downtrend_bars = 490；
short_coverage_downtrend_ratio = 2.4%。
```

---

## 3. 6 月初失败原因

6 月 1 日至 6 月 5 日附近，价格已经出现强下跌结构：

```text
trend_is_downtrend = true
trend_is_strong_downtrend = true（多处）
ret_6_atr 多次显著为负
close < ema_fast < ema_slow
```

但模型输出并不支持空头：

```text
edge 多数为正；
p_down 不高；
pred_cum_ret_5 多数为正；
p_flat 接近过滤上限；
p_risk 偏高但未进入强制退出。
```

因此：

```text
标准 OPEN_SHORT_SIGNAL 不触发；
014b sentinel 因 edge/pred_cum_ret 条件过严也不触发；
系统保持 FLAT，错过 6 月初主要下跌。
```

架构判断：

```text
这不是继续调低 open_edge_threshold 能解决的问题；
这是模型在特定崩跌 regime 下发生方向失效。
```

015 应把该场景显式定义为：

```text
MODEL_BLIND_CRASH
```

---

## 4. 015 总体方案

015 不再沿用 014a 的普通 probe short，也不依赖 0062e 给空头确认。

新方案：

```text
新增 CrashRegimeDetector；
当价格满足崩跌状态，且模型没有给出可靠反向保护时，
触发独立的 CRASH_SHORT 防守空头。
```

关键区别：

```text
014a: 看到普通下跌结构就频繁小空，导致亏损。
014b: 模型确认空头后持得更久，但模型失效时不开空。
015: 只有进入崩跌 regime 时，绕过模型方向判断，开一次防守空头。
```

## 5. 新增模块：`CrashRegimeDetector`

### 5.1 文件

新增：

```text
trading_system/crash.py
```

### 5.2 输出对象

```python
@dataclass
class CrashContext:
    is_crash: bool
    is_model_blind_crash: bool
    crash_score: float
    drawdown_24h: float
    ret_6_atr: float
    ret_12_atr: float
    range_expansion: float
    consecutive_down_bars: int
    lower_low_break: bool
    model_disagrees: bool
    reason_codes: list[str]
```

### 5.3 崩跌状态定义

`is_crash` 需要满足价格行为，不依赖模型方向。

建议投票规则：

```text
ret_6_atr <= -3.0
或 ret_12_atr <= -5.0
或 24h drawdown <= -6%
或 close < rolling_low_24
或 consecutive_down_bars >= 4 且 range_expansion >= 1.5
```

第一版建议：

```text
crash_votes >= 2 -> is_crash = true
crash_votes >= 3 -> strong_crash = true
```

其中：

```text
drawdown_24h = close / rolling_high_24 - 1
ret_6_atr = (close[t] - close[t-6]) / atr[t]
ret_12_atr = (close[t] - close[t-12]) / atr[t]
range_expansion = mean(true_range[t-3:t]) / mean(true_range[t-24:t])
lower_low_break = close < min(low[t-24:t])
```

全部使用当前及历史已完成 K 线，不允许使用未来数据。

## 6. 模型失效定义

`is_model_blind_crash` 表示：

```text
价格已进入崩跌，
但模型没有给出标准空头信号。
```

触发：

```text
is_crash == true
position.side == FLAT
standard_open_short == false
```

同时要求模型没有强反向保护：

```text
NOT (
  p_up >= 0.38
  and edge >= 0.06
  and pred_cum_ret_5 > 0.25
  and p_risk < 0.40
)
```

解释：

```text
如果模型极强看多且低风险，可以先不做 crash short；
但 6 月初这种 p_up 不强、p_risk 偏高、价格持续崩跌的情况，应允许防守空头。
```

---

## 7. 新行为：`OPEN_SHORT_CRASH`

### 7.1 触发条件

全部满足：

```text
position.side == FLAT
crash_context.is_model_blind_crash == true
p_risk <= crash_short_risk_max
p_flat <= crash_short_flat_max
not daily_open_block
not account_circuit_breaker
not cooldown
```

建议默认：

```text
crash_short_risk_max = 0.48
crash_short_flat_max = 0.42
```

注意：

```text
OPEN_SHORT_CRASH 不要求 edge < 0；
不要求 pred_cum_ret_5 < 0；
不要求 p_down >= open_prob_threshold。
```

原因：

```text
该行为专门处理模型方向失效。
```

### 7.2 仓位

`OPEN_SHORT_CRASH` 不得使用普通空头仓位。

建议：

```text
初始仓位 4%
strong_crash 可到 6%
最大 8%
```

若后续模型转为空头确认，再升级：

```text
UPGRADE_CRASH_TO_MODEL_SHORT
```

升级后使用标准 `PositionSizer`，但总仓位仍不超过 20%。

## 8. 单 regime 限频

014a 失败的核心是反复 probe。015 必须避免重蹈覆辙。

定义：

```text
crash_regime_id
crash_regime_active
crash_short_used
crash_short_cooldown_until
```

规则：

```text
同一 crash regime 只允许 1 次 OPEN_SHORT_CRASH；
若止损退出，不允许在同一 regime 再开；
只有价格重新站回 ema_fast 且 crash_votes 连续 6 根低于 2，才结束 regime；
新 regime 才允许再次 crash short。
```

## 9. 持仓管理

### 9.1 crash short 初始持仓

进入后：

```text
position.entry_was_crash = true
position.hold_mode = CRASH
position.crash_bars = 0
```

### 9.2 快速失败退出

如果开仓后很快失败，必须退出：

```text
current_price > entry_price + 1.2 * atr_at_entry
或 crash_votes < 2 连续 2 根
```

动作：

```text
CLOSE_CRASH_FAILED
```

### 9.3 趋势继续则延长

若：

```text
crash_votes >= 2
且 close < ema_fast
```

则允许持有：

```text
crash_max_hold_bars = 18
strong_crash_max_hold_bars = 30
```

### 9.4 模型确认后升级

crash short 持仓中，若出现标准空头条件：

```text
edge <= -open_edge_threshold
p_down >= open_prob_threshold
pred_cum_ret_5 < 0
p_flat <= open_flat_max
p_risk <= risk_open_max
```

动作：

```text
UPGRADE_CRASH_TO_MODEL_SHORT
hold_mode = TREND
```

### 9.5 利润保护

若：

```text
peak_profit_atr >= 2.0
且 profit_atr 回撤 >= 1.0
```

退出：

```text
CLOSE_CRASH_TRAIL
```

## 13. 回测验收

必须对照：

```text
v014 tuned
v014a tuned
v014b trend_hold tuned
v015 crash_regime
```

硬性验收：

```text
position_limit_violations == 0
risk_rule_violations == 0
max_margin_loss_ratio_observed < 1.00
trade_count <= v014b_trade_count + 3
crash_short_count <= 3
same_regime_reentry_count == 0
```

6 月初专项：

```text
6/1~6/5 期间必须至少触发一次 MODEL_BLIND_CRASH；
若触发 OPEN_SHORT_CRASH，不能在同一 crash regime 反复开仓；
该段净收益应为正，或亏损不超过 -1.0% ROE；
若未开仓，报告必须列出未触发的具体 vote 缺口。
```

整体目标：

```text
total_return >= v014b 的 80%
max_drawdown 不超过 v014b + 2pp
profit_factor 不显著低于 v014b
```

优先级：

```text
先修复 6 月初行为；
再看整体收益；
禁止为了单段收益导致交易次数暴增。
```

## 14. 不建议做的事

不建议：

```text
重新打开 014a 普通 probe short；
继续降低 p_down/open_edge 阈值；
取消 p_risk 上限；
同一 crash regime 多次重开；
用 hindsight 最优交易直接做标签；
把 crash short 仓位直接提高到普通模型仓位。
```

原因：

```text
014a 已证明高频试探会拖垮收益；
015 只解决模型失效下的极端崩跌，不承担日常空头识别。
```

---

## 15. 结论

015 的核心改动是：

```text
承认模型在 6 月初崩跌 regime 中方向失效；
新增独立 CrashRegimeDetector；
只在 MODEL_BLIND_CRASH 中开一次小仓位防守空头；
若模型随后确认空头，则升级为 014b trend_hold；
若没有确认或崩跌失败，则快速退出。
```

一句话：

```text
014b 解决“空头拿不住”；
015 解决“模型失效时根本不开空”。
```
