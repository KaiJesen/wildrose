## 3. 问题症结

### 3.1 horizon-specific return loss 破坏了累计收益分支

0063 的核心优势：

```text
cum_return_ic mean = 0.127
cum_direction_from_return_acc mean = 60.6%
```

0064a/c/d 加入 horizon-specific step return 强化后：

```text
cum_return_ic 降至 0.026 / 0.005 / -0.002
cum_direction_from_return_acc 降至 54.1% / 52.0% / 51.4%
```

判断：

```text
step return 强化和 cum return 分支存在强冲突。
当前不是权重大小问题，而是目标方向冲突。
```

### 3.2 step return_ic 没有被有效恢复

0064 三组：

```text
return_ic = 0.010 / 0.004 / 0.009
```

均没有接近最低目标 `0.020`。即便明显牺牲 `cum_return_ic`，也没有换来 step `return_ic`。

判断：

```text
当前 OHLCV + 1h + 5 horizon 标签设置下，step-level return 排序信号很弱；
通过简单 horizon loss 加权无法提取。
```

### 3.3 direction flat 类反复坍缩

0064a/c/d 均出现：

```text
direction_pred_flat = 0%
```

说明：

- 提高 step return 监督后，方向分类边界被破坏；
- 模型在 down/up 之间二分化，丢失 flat 状态；
- 这会使 `direction_macro_f1` 和市场状态解释性下降。

判断：

```text
0064 不是可调小偏差，而是破坏了 0063 的稳定分布结构。
```

### 3.4 低 loss / 低 MAE 是误导信号

0064 的 `return_mae`、`volatility_mae` 很低，但 IC 和方向指标变差。

例如 0064a：

```text
return_mae = 0.011
volatility_mae = 0.018
return_ic = 0.010
cum_return_ic = 0.026
```

判断：

```text
模型可能在输出更平滑、更保守的数值，降低 MAE，
但没有提升排序能力，甚至破坏累计收益排序。
```

后续不能用 MAE 下降判断收益预测变好。

## 4. 架构师裁定

### 4.1 是否继续 0064 当前路线

不继续。

明确停止：

```text
停止 horizon-specific return loss 加权路线。
停止 0064a/c/d 同类变体。
不再通过加大 h1/h2/h3 权重追 step return_ic。
```

### 4.2 是否回滚新结构

不回滚。

保留：

```text
0062e / 0063 新结构稳定分支
HorizonReturnHead
cum_return_pred
cum_return_ic 作为核心指标
```

原因：

```text
0063 已证明新结构对累计收益排序有效且稳定；
0064 失败的是 step return 恢复训练策略，不是新结构分支本身。
```

### 4.3 是否替换 0058 主基线

不替换。

继续双基线：

```text
0058 = usable 主基线
0062e / 0063 = cum-return 新结构稳定分支候选
```

### 4.4 是否继续追 step return_ic

暂缓。

step return_ic 当前不是短期主线目标。除非后续引入新特征、新标签定义或更系统的诊断，否则不建议继续投入训练资源追 `return_ic >= 0.02`。

## 9. 后续若要重新研究 step return_ic，需要什么前置条件

只有满足以下任一条件，才重新打开 step return_ic 研究：

1. 引入新的公开特征，例如资金费率、持仓量、基差等；
2. 改标签定义，例如从逐 bar return 改为分段/区间收益；
3. 改任务目标，例如只预测 h4/h5 或累计路径形态；
4. 做跨时间 walk-forward 后发现某些 regime 下 step return_ic 稳定存在；
5. 项目经理明确批准新的研究阶段。

否则当前默认结论为：

```text
step return_ic 不是当前模型线短期可恢复目标。
```
