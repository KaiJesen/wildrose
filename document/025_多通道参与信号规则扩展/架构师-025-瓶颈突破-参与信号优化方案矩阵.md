## 2. 核心诊断

### 2.1 症状

```
c1_pw20 约束训练 (cum_IC ≤5% 退化约束):
  valid part_auc: 0.572 （仅过 0.55 门，未达 0.58 强分类器）
  TEQ 通道:   teq=3 参与 ✅（Pareto 膝点 return 9.01% / coverage 26.7%）
  慢涨通道:   A3a 9 组 valid 零候选；test 最优 return 7.25% (−1.76pp vs B0)
  标准混合:   A3b 6 组 valid 全部 −3.41% return；test 最优 return 6.81% (−2.20pp)

无约束训练 (cum_IC 不设限):
  valid part_auc: 0.611 → 0.633
  legacy 头漂移: A1 return −8.34% → −8.12% （无 TEQ/有 TEQ）
```

### 2.2 根因：双重绑定

```text
根因 1：表示瓶颈
  冻结的 0062e trunk 被优化为 5-bar cum return + direction，
  无法编码腿级别的长期上下文特征。
  part_auc ≤0.572 说明 participate_logit 几乎无法超过随机猜测的置信区间。

根因 2：标签信号不足
  ideal_participate 是严格的 binary hindsight：
    ROI > 0 ∧ MAE ≤ 1.5 ATR ∧ progress ≤ 0.35
  正样本率极低。Binary BCE 对所有负样本等价处理，
  无法区分 "接近理想但 MAE 略大" 和 "在 CHOP 中不可能参与"。

根因 3：cum_IC 约束过紧
  95% baseline 等价于 ≤5% 退化。
  约束训练下 part_auc 被天花板压制，
  无约束训练下 part_auc 可提升 +0.06～0.08，但付出 −16pp return 代价。
  这表明存在结构性的多任务冲突：腿参与特征 ≠ 短期 cum return 特征。
```

### 2.3 评估框架

以下方案按两个维度评估：

| 维度 | 含义 |
|------|------|
| **预期 Δpart_auc** | 相对当前 0.572 的提升幅度（0 = 无变化，+ = 提升） |
| **工程成本** | L（< 3 天）、M（1~2 周）、H（3~4 周） |
| **回归风险** | 003 方向/return 头退化的概率 |

## 3. 方案矩阵

### 类别 A：标签升级（最大杠杆）

#### A1. 三阶有序标签 + Ordered BCE

**改动**：`build_leg_participation_labels.py`、`training.py`

```text
ideal_participate: 0/1/2 → 2(强)/1(弱)/0(不可)
  2: ROI > 0 ∧ MAE ≤ 1.5 ATR ∧ progress ≤ 0.35  （当前 ideal=1）
  1: ROI > 0 ∧ MAE ≤ 2.5 ATR ∧ progress ≤ 0.50  （放宽 MAE 与窗口）
  0: 其他

损失：ordered BCE 或 CORAL (ordinal regression)，非 binary BCE

预期收益：正样本量 ×2~3；模型学习 "可参与但风险较高" → 通道校准有更多中间态
```

| 预期 Δpart_auc | 工程成本 | 回归风险 |
|----------------|----------|----------|
| **+0.04~0.08** | M（1~2 周） | 低（不碰模型结构） |

---

#### A2. 连续参与分数 + Huber

**改动**：`build_leg_participation_labels.py`、`training.py`

```text
participation_target[t] = clamp(forward_leg_roi / atr[t] * 10, 0, 1)
  （forward_leg_roi 越高、波动越小 → target 越接近 1）
损失：Huber(participation_logit_sigmoid, target)
      或 MSE(participation_logit_sigmoid, target)

优势：无阈值切割，梯度连续，模型可学到细粒度排序
```

| 预期 Δpart_auc | 工程成本 | 回归风险 |
|----------------|----------|----------|
| **+0.05~0.10** | M（1~2 周） | 低 |

---

#### A3. 通道条件标签

**改动**：`build_leg_participation_labels.py`、`signal.py`

```text
对每条 bar 生成 "该 bar 在 slow_up 通道下是否理想参与" 的条件标签：
  ideal_participate_slow_up[t] = ideal_participate[t]
    AND leg_type == SLOW_UP_LEG
    AND slow_trend_context == SLOW_UP

同理生成 ideal_participate_std_trend[t]

模型输出三个 participate_logit（通用/slow_up/std_trend），
规则层按通道读取对应 logit。

优势：通道专用信号，校准天然独立，无需 channel_edge 混合映射
```

| 预期 Δpart_auc | 工程成本 | 回归风险 |
|----------------|----------|----------|
| **+0.03~0.06** | H（3~4 周） | 中（多头训练） |

---

### 类别 B：训练策略（释放被约束的表达）

#### B1. 渐进解冻（两阶段）

**改动**：`train_market_state_0065a.py`

```text
Stage 1（W1）：冻结 trunk，仅训练 participation + horizon 头
  → part_auc 收敛

Stage 2（W1~W2）：解冻 trunk 最后 2 层 transformer + layer_norm
  lr = 1e-5（低 100× 相对 Stage 1）
  联合优化 L_part + L_leg_dir + 0.1 × L_market_state_legacy
  cum_IC 约束放宽至 ≤10%

预期：trunk 上层的表示可被微调为腿参与特征，
      同时最低层保持 0062e 基础模式
```

| 预期 Δpart_auc | 工程成本 | 回归风险 |
|----------------|----------|----------|
| **+0.05~0.10** | M（1~2 周） | 中（cum_IC 可能退化 5~10%） |

---

#### B2. 多任务不确定性加权（Learnable σ）

**改动**：`training.py`

```text
替换手工 λ_part / λ_leg_dir 为可学习的任务噪声参数 σ_i：
  L_total = Σ (1/(2σ_i²)) · L_i + log(σ_i)

（Kendall et al., CVPR 2018）

模型在训练中自适应平衡 participation vs legacy 损失，
不需要人工 ablation 链。
```

| 预期 Δpart_auc | 工程成本 | 回归风险 |
|----------------|----------|----------|
| **+0.02~0.05** | M（1~2 周） | 低（自动平衡） |

---

#### B3. 梯度手术（PCGrad）

**改动**：`training.py`

```text
每个 step：
  1. 计算 L_part 梯度
  2. 计算 L_legacy 梯度
  3. 若两梯度冲突（cosine < 0），投影 L_part 梯度到 L_legacy 梯度正交空间
  4. 合并后更新

优势：明确解决 "提升 participation 以牺牲 legacy 为代价" 的冲突
```

| 预期 Δpart_auc | 工程成本 | 回归风险 |
|----------------|----------|----------|
| **+0.03~0.07** | H（3~4 周） | 低（梯度层面解耦） |

### 类别 C：架构轻量改造（让模型看见腿）

#### C1. 腿上下文特征注入

**改动**：`pattern_model.py`、`segment_dataset.py`

```text
新增输入特征（不改变 trunk 结构）：
  leg_progress_ratio      [0,1]  归一化到当前 bar 精度
  leg_type_embedding      可学习 4 维 → concat 到 bar embedding
  bars_since_leg_start    log(1 + bars) 归一化

加入 K 线特征序列（bar_feature 维度 +3~4），
trunk 无需结构调整即可接入。

预期：trunk 自注意力层可利用 "腿中位置" 信息，
      学习 "腿前 1/3 段 + SLOW_UP → 大概率可参与" 的模式。
```

| 预期 Δpart_auc | 工程成本 | 回归风险 |
|----------------|----------|----------|
| **+0.03~0.06** | L（< 3 天） | 极低（仅增加输入维度） |

---

#### C2. 独立腿编码器 + 交叉注意力

**改动**：`pattern_model.py`（中等新增）

```text
新增 LegContextEncoder（轻量 2 层 transformer，512 dim）：
  输入：当前 bar 所属 leg 的所有 bar 的 trunk hidden states
  输出：leg_context[h] ∈ R^d_model

ParticipationHead：
  输入：concat(bar_hidden, leg_context) → MLP → logit
  （或 cross-attention：participation_query × leg_context_keys）

优势：participation 头可 "看到整条腿" 而不依赖 trunk 编码
```

| 预期 Δpart_auc | 工程成本 | 回归风险 |
|----------------|----------|----------|
| **+0.06~0.12** | H（3~4 周） | 中（新增模块，需调参） |

---

#### C3. 参与头升级为交叉注意力（复用 HorizonReturnHead 模式）

**改动**：`pattern_model.py`（极小，~20 行）

**发现**（来自 `pattern_model.py` 架构审查）：当前 `participation_logit_long` 只是 `Linear(128→64)→GELU→Linear(64→1)`，输入是 **单个全局池化向量** `pooled [B, D]`——所有 bar 的信息被压缩为一个点。而同文件的 `HorizonReturnHead` 已实现 **learned query × full token sequence cross-attention** 模式。

```text
HorizonReturnHead 数据流（现有，可直接复用）：
  1. learned_query [1, horizon, D] + pooled [B, 1, D] → combined query
  2. cross-attn(query, tokens [B, S, D]) → context [B, horizon, D]
  3. step_out: Linear(context) → [B, horizon]
  4. cum_out:  Linear(pooled + context.mean) → [B, 1]

参与头升级（最小版）：
  1. learned_query [1, 1, D]（可学习参与查询向量）
  2. cross-attn(query, tokens [B, S, D] frozen) → context [B, 1, D]
  3. Linear(context.squeeze) → [B] → participation_logit

优势（相对 C2）：
  - 不新增模块，复用现有 HorizonReturnHead 的 cross-attn 实现
  - 参与头可看到每条 bar 的表示，而非仅全局池化
  - 冻结的 trunk tokens 已是腿级别的上下文编码（bar 间 attn 已计算）
  - trunk 也冻结，只训 cross-attn 的 Q/K/V 投影 + 输出 Linear
```

| 预期 Δpart_auc | 工程成本 | 回归风险 |
|----------------|----------|----------|
| **+0.04~0.08** | L（< 3 天，复用现有 pattern） | 极低（trunk 全冻结，仅新增 Q/K/V 投影） |

这是 **最高性价比的单点改动**——不改标签，不改 trunk，仅升级参与头的信息访问模式。实现参考 `HorizonReturnHead` 约 100 行代码，参与头版本可精简至 ~30 行。

## 5. 鉴定表（选型建议）

### 5.1 如果你是风险厌恶型（保守策略）

```

  → 若 test return 显著提升 → 025 可能被解围（不需要重训）
```

```
备选：F2 组合（A2 + B3 + C2）
```

## 6. 026 立项建议

| 项目 | 内容 |
