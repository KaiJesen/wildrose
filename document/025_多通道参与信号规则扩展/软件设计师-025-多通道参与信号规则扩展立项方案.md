## 2. 问题归纳

### 2.1 024 已证明什么

| 结论 | 证据 |
|------|------|
| 参与信号可学 | valid part_auc ≈ 0.57（constrained） |
| TEQ 接线有效 | teq 3×，counter_leg ≤5 |
| 收益可超 A0 | A2 return **9.01%** vs A0 **7.77%** |
| 单通道 coverage 有上限 | wp≥0.37 时 return 跌至 8.55% |
| 慢涨通道未参与 | phase1c `slow_up_open_count=0`，`watch_slow_uptrend` 大量 watch 无仓 |

### 2.2 未覆盖的参与盲区（按 bar 量级）

| 通道 | phase1c 表现 | 024 Scheme B | 025 目标 |
|------|-------------|--------------|----------|
| TrendEntryQualifier（teq） | teq≈1 | teq=3，已调优 | 保持 024 最佳 wp=0.35 |
| 慢涨 `OPEN_LONG_SLOW_TREND` | **0 开仓** | 不读 `teq_edge_*` | **P0：受控启用 + 参与门** |
| 标准趋势 `OPEN_*_SIGNAL` | edge 门槛高、空仓多 | 仍读 legacy `edge` | **P1：confirmed leg 子集条件混合** |
| Participation Floor（023 P2） | 默认关闭 | 未实验 | **P2：可选对照** |

### 2.3 核心假设

coverage 缺口主要来自 **「趋势已确认、规则允许、但模型 legacy edge 未过线」** 的 bar，而非 TEQ 阈值 alone。将 **已校准的 `participate_score_*` / `teq_edge_*`** 在 **白名单通道** 上作为 **附加资格或阈值放松因子**，有机会增加有效 overlap 而不复制 teq ceiling 的噪声开仓模式。

A3a 需在 **B0 已 26.7%** 基线上再抬 1.3pp——新增 slow_up 开仓必须与既有 teq 覆盖 **低重叠**，否则 coverage 分子不增或增量极小。

### 2.4 历史负面先例（须纳入 A3a 设计）

| 历史实验 | 改动 | test return | coverage | 教训 |
|----------|------|-------------|----------|------|
| **023 phase1a** | 仅启用 `slow_up_position` + 0062e，**无参与门** | **1.99%**（−7.32pp） | 26.67% | 慢涨无参与门 → 收益崩塌 |
| 024 A2 @ wp≥0.37 | TEQ 阈值放松 | 8.55% | 30.0% | 多 1 次 teq → 边际腿质量差 |
| 024 A2 @ wp=0.35 | TEQ 最佳（B0） | **9.01%** | 26.7% | 当前对照基线 |

025 假设：「趋势已确认、规则允许、legacy edge 未过线」的 bar 可通过 `participate_score` / 通道专用 edge 增量覆盖——**逻辑合理**，但 A3a 的参与门 + probe 比例 + `slow_up_false_entry` 上限为 **必要条件**，未达标即关闭 A3a。

## 4. 方案设计（Scheme A3 受控扩展）

### 4.1 信号契约（增量）

在 024 `TradingSignal` 已有字段基础上：

```python
# 诊断（全 bar 可记录，不参与未声明通道）
effective_edge_long_diag: float = 0.0
effective_edge_short_diag: float = 0.0

# 通道专用（仅声明通道读取）
slow_up_edge_long: float = 0.0      # 慢涨多头资格
std_trend_edge_long: float = 0.0    # 标准趋势多头（A3b）
std_trend_edge_short: float = 0.0   # 标准趋势空头（A3b）
```

**混合公式（valid 调参，禁止 test 泄漏）**：

```text
part_term_long  = 2·participate_score_long - 1
raw_channel_long = α·legacy_edge + β·teq_edge_long_raw + γ·part_term_long
                 （teq_edge_long_raw = compute_teq_edge_raw 输出，校准前）

effective_edge_long_diag = 同上，仅写日志/诊断
slow_up_edge_long        = apply_channel_calibration(raw_slow_up_long, channel="slow_up")
std_trend_edge_long      = apply_channel_calibration(raw_std_long, channel="std_trend")
```

默认初值建议：`α=0.6, β=0.25, γ=0.15`（慢涨偏重 legacy 防误开）；**与 TEQ 的 `w5/w24/w_part` 独立**，避免隐式双重计权。各通道 valid 网格独立选取。

### 4.1.1 通道 edge 数据流（冻结契约，响应评审 B1）

```text
MarketStateModelProvider.signal_at()
  ├─ finalize() → legacy edge (= p_up - p_down)，不改写
  ├─ _attach_teq_fields()
  │    ├─ participate_score_* ← TeqEdgeCalibrator.apply_part_*（共用 teq_edge_calibration.json）
  │    ├─ teq_edge_* ← compute_teq_edge_raw + apply_teq_calibration（TEQ 专用）
  │    └─ （TEQ 路径规则层只读 teq_edge_*，不读 effective_edge_diag）
  └─ _attach_channel_edges()  【025 新增】
       ├─ raw_slow_up_* / raw_std_* ← channel_edge.compute_channel_edge_raw(α,β,γ,...)
       ├─ slow_up_edge_* ← ChannelEdgeCalibrator["slow_up"]（valid 独立拟合，可仅仿射）
       ├─ std_trend_edge_* ← ChannelEdgeCalibrator["std_trend"]（A3b 启用时）
       └─ effective_edge_*_diag ← 同 raw 公式，不经通道 calibrator（纯诊断）

RuleEngine（消费边界）
  ├─ _try_trend_qualified_open → 仍读 teq_edge_* + wp（B0 不变）
  ├─ _try_open_slow_up         → 读 slow_up_edge_long 或 participate_score_long ≥ τ_slow
  └─ OPEN_*_TREND（A3b）       → 条件满足时读 std_trend_edge_*，否则 legacy edge
```

**校准范围（冻结）**：

| 字段 | 校准器 | 拟合数据 | 说明 |
|------|--------|----------|------|
| `participate_score_*` | `TeqEdgeCalibrator`（isotonic part） | valid | 与 024 共用 `teq_edge_calibration.json` |
| `teq_edge_*` | `TeqEdgeCalibrator`（shift anchor） | valid | TEQ 专用，B0 已冻结 wp=0.35 |
| `slow_up_edge_*` | `ChannelEdgeCalibrator` | valid，ideal=腿参与标签 | **独立** JSON，如 `channel_edge_slow_up_calibration.json` |
| `std_trend_edge_*` | `ChannelEdgeCalibrator` | valid | A3b 启用时独立拟合 |

**计算落点**：通道 edge **在 `ModelSignalProvider` 出口一次性写入** `TradingSignal`；`RuleEngine` **只读**字段，不在入口重算。

### 4.2 通道 A3a — 慢涨开仓（P0）

**背景**：023 phase1c `slow_up_position.enabled=false`；024 未改。test 上 `watch_slow_uptrend` 多但 `slow_up_open_count=0`，是 coverage 最大潜在来源。

**规则增量**（`RuleEngine._can_open_slow_up` / `_try_open_slow_up`）：

1. 配置 `slow_up_position.enabled=true`（025 实验 config）
2. 新增 `slow_up_participation_gate`：
   - `participate_score_long ≥ τ_slow` **或** `slow_up_edge_long ≥ edge_threshold_slow`
   - `τ_slow` 初值 0.55；valid 网格 [0.45, 0.65]
3. 保留 023 既有：`is_stable_slow_uptrend`、RiskBudget、`min_hold_bars`
4. 首仓 probe 比例 ≤ `slow_up_probe_ratio`（**默认 0.5**，与 TEQ probe 口径对齐，写入 config 默认值）

**A3a 继续 / 关闭判据**：

```text
若 slow_up_incremental_coverage_pp < 0.8 且 return 较 B0 下降 > 1pp → 关闭 A3a，书面记录后视情况进入 A3b；
若 2 周内 A3a 无法带来 ≥0.8pp 增量 coverage 且 return ≥8.84% → 准备 025 结案，评估 024 §6.6。
```

**归因**：`metrics.slow_up_*` 分列 PnL；`eval_participation.slow_up_false_entry_count` 必报。

### 4.3 通道 A3b — 标准趋势 confirmed-leg 条件混合（P1，后置）

**背景**：大量 confirmed leg bar 上 legacy `edge` 不够，但 `teq_edge` 或 `participate_score` 已指示参与。与 023 phase3「标准开仓放宽」**高度同构**（test return −2.03%），**默认不得与 A3a 并行**。

**规则增量**（仅当 **全部** 条件满足时替换比较对象）：

```text
is_leg_confirmed == 1
leg_type ∈ {FAST_UP_LEG, SLOW_UP_LEG, FAST_DOWN_LEG, SLOW_DOWN_LEG}
NOT chop_hard
portfolio_flat
→ 开仓阈值比较改用 std_trend_edge_*（非 signal.edge）
```

**安全阀**：

- 仅影响 `OPEN_LONG_TREND` / `OPEN_SHORT_TREND` 类 reason，不影响 crash / sentinel
- `std_trend_edge` 混合权重 `β` ≤ teq 通道（防止标准通道过度放松）
- valid 上网格；若 A3a 已闭合探索线，A3b 降为 P2 对照

### 4.4 通道 A3c — Participation Floor（P2，可选）

继承 023 §6.6.2 设计，默认 **关闭**。仅当 **A3a 已裁定** 且 **A3a+A3b 仍不达 28% coverage** 时启用实验臂：

- confirmed leg + flat + `participate_score ≥ τ_floor` → 允许最小 probe 开仓
- 不得绕过 RiskBudget；`τ_floor` valid 选取

### 4.5 与 024 Scheme B 的关系

```text
B0 = 024 最佳（仅 TEQ 读 teq_edge，wp=0.35）— 所有 A3 实验的对照基线
A3a = B0 + 慢涨通道参与门
A3b = A3a + 标准趋势条件混合（或 B0 + A3b 消融，仅 A3a 无效时）
A3c = A3b + Floor（可选）
```

**硬约束**：TEQ 路径继续读 `teq_edge_*`，**不**改为读 `effective_edge_diag`。

### 4.6 实验臂阶段闸门（响应评审 B3）

| 闸门 | 条件 |
|------|------|
| 进入 A3a | Phase 0 B0 复现 PASS |
| 进入 A3b | A3a 已 PASS 探索线，**或** A3a 裁定「慢涨无效」且书面记录 |
| 进入 A3c | A3a+A3b 仍 <28% coverage |
| 进入 Phase 4 | Phase 3 探索线双门 PASS + 项目经理启动投产评估 |

**禁止**：未过 B0 复现门时启动任何 A3 实验；A3b 与 A3a 默认并行。

## 8. 风险与缓解

| 风险 | 等级 | 缓解 |
|------|------|------|
| **A3a 重复 023 phase1a 收益崩塌**（无参与门慢涨 −7.32pp） | **高** | 参与门 + `slow_up_probe_ratio≤0.5` + `slow_up_false_entry` 上限；未达标即关 A3a |
| TEQ 膝点外推失败（+1.3pp 仍牺牲 return） | **高** | 与 024 相同；A3a 无效则转 A3b 或 A3c，否则 025 结案 |
| A3b 复制 023 phase3 负收益 | **中高** | **禁止**与 A3a 并行；confirmed-leg + `std_trend_edge` 混合 |
| 探索 PASS 但投产仍远未达标（B0 距 35% 差 8.3pp） | **中** | **预期内**；不得自动提名 prod |
| 通道调参过拟合 valid | **中** | test 一次性 + multi-seed |
| 归因混乱（TEQ vs slow_up 双重计数） | **中** | `edge_source` schema（B2 阻塞项，Phase 0 交付） |
| 误伤标准开仓 | 中 | 白名单 + confirmed leg + 不调 global edge |

## 9. 失败与回退

| 条件 | 动作 |
|------|------|
| Phase 0 复现失败 | 排查 checkpoint / 校准 / config 漂移；**不得**启动 A3 |
| A3a 慢涨无效（`slow_up_incremental_coverage_pp` <0.8 且 return −>1pp） | 关闭 A3a，书面记录后仅试 A3b |
| A3a 2 周内无 ≥0.8pp 增量 coverage 且 return <8.84% | **025 结案**；评估 024 §6.6 best-point 或标签 v2 |
| Phase 2 仍不达 28% coverage | 试 A3c Floor 或驳回 A3 路径 |
| 探索线仍近失 | 回退 B0；025 结案 |
| 任意阶段 drift 恶化 >2pp vs A0 | 回滚该通道增量 |

**回退基线**：`configs/trading_rule_v024_phase1c_teq_0065a_c1_pw20.json` + `checkpoints/0065a_leg_align_c1_pw20`。
