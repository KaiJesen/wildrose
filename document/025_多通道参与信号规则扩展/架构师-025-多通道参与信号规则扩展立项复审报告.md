## 8. 实施前澄清项

以下 3 项 **不构成立项阻塞**，建议在 **Phase 0 出口 PR** 或 **Phase 1 开跑前** 写入实现：

### C1：`slow_up_incremental_coverage_pp` 公式

§3.1 / §5.3 多次引用，但 **未给操作定义**。建议在 `eval_participation.py` 扩展或 `run_v025_ab_phase1c.py` 中冻结为：

```text
slow_up_incremental_coverage_pp =
  100 × (|{leg_id : effective_covered_A3a ∧ ¬effective_covered_B0}| / leg_count)

（leg_count 与 leg_count_coverage_ratio 同源，023 §5.3 / participation.py 口径）

辅助报告（非闸门）：coverage_A3a − coverage_B0（pp），与上式对照防止「重叠加深」误判为增量。
```

### C2：慢涨参与门 OR 语义与否决条件

§4.2 门控为 `participate_score ≥ τ_slow` **或** `slow_up_edge_long ≥ threshold`。建议实现注释中写明：

```text
若 valid 上网格显示「仅 part 分支」开仓占比 >50% 且 slow_up_pnl < 0 → 强制 AND 模式或抬高 τ_slow；
slow_up_pnl < −1% 时否决 A3a（§3.1 已述，建议写入 tune 脚本退出条件）。
```

### C3：ChannelEdgeCalibrator 标签子集

§4.1.1 表列 `ideal=腿参与标签`，未限定腿类型。建议冻结：

```text
slow_up 通道校准：ideal 正样本限于 SLOW_UP_LEG（及 align long 子集）；
std_trend 通道：confirmed leg 四类与 A3b 条件一致；
校准 JSON 元数据写入 label parquet hash（继承 024 §6.2.4 惯例）。
```
