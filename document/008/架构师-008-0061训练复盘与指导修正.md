# 0061 训练复盘与训练/结构指导修正

## 1. 文档目标

本文档基于 `reports/0061_market_state_return_direction_branch` 与 `reports/0061b_market_state_return_direction_branch` 的训练结果，分析 0061 新结构训练失败的原因，并修正此前《架构师-007-0061新结构训练目标指导》的训练目标与结构调整意见。

本文只讨论模型结构、训练目标、损失权重、选模与诊断，不涉及交易执行、下单、仓位或资金管理。

## 2. 0061 训练结果摘要

0061 与 0061b 结果一致，说明问题可复现，不是单次随机波动。

测试集关键指标：

```text

risk_positive_rate_pred        = 0.00%

```

相对 `0059c`：

```text

```

## 3. 结论先行

0061 不是整体结构失败，而是“新 return/cum return 分支有效，但分类分支和 risk 分支发生坍缩”。

判断：

1. `cum_return_ic=0.105` 说明 `HorizonReturnHead + cum_return_pred` 是有效方向，应保留。
2. `direction_pred_flat=0`、`direction_pred_down=94.2%` 说明 direction 三分类已经坍缩。
3. `risk_positive_rate_pred=0` 说明 risk head 完全坍缩为负类。
4. `cum_direction_head_acc=40.5%` 说明新增 cumulative direction head 无效，甚至反向。
5. 0061 的 hard gates 没有阻止坏模型进入最终评估，说明选模/保存逻辑需要修正。

决策：

```text
不要回退到旧结构；
保留 HorizonReturnHead 和 cum_return_pred；
暂停或重做 cum_direction_logit 分支；
取消 detach_risk_vol_heads 作为默认配置；
修正 class weights 与 hard gates；
下一轮做 0062 修复训练，而不是继续 0061 原配置。
```

## 4. 问题症结分析

### 4.1 新累计收益头有效

`cum_return_ic=0.105` 是 0061 中最有价值的结果。

这说明：

- 显式累计收益头比单纯 `return_pred.sum()` 更能学习未来累计收益排序。
- horizon-aware return head 没有破坏 step return，`return_ic=0.039` 与 `0059c=0.038` 基本持平。
- 后续不应删除 `HorizonReturnHead` 或 `cum_return_pred`。

修正方向：

```text
保留 cum_return 分支；
降低 consistency 对 step return 的牵制；
继续要求 cum_return_ic 作为主指标。
```

### 4.2 direction 分支发生类别坍缩

0061 direction 预测：

```text
pred down = 94.19%
pred flat = 0.00%
pred up   = 5.81%
```

该结果说明 direction 分类头没有学到三分类边界，而是几乎退化为“多数 down”预测器。

可能原因：

1. `balanced_class_weights` 对 down 类过度补偿。
2. 新 `DirectionStateHead` 与旧初始化不兼容，直接从 0060 checkpoint 微调时新 head 近似随机，训练被 class weight 拉偏。
3. `direction_weight=0.38` 加上 class weights 后，对新 head 的梯度过强。
4. hard gates 中没有直接拒绝 `direction_pred_flat=0` 这类坍缩模型。

修正方向：

```text
0062 先关闭 balanced_class_weights，只保留普通 train class weights 或完全关闭 direction class weights；
降低 direction_weight；
增加 direction 分布坍缩 hard gate；
必要时对 DirectionStateHead 使用 warmup 或更小学习率。
```

### 4.3 risk 分支完全坍缩为负类

0061 risk 预测：

```text
risk_positive_rate_true = 23.24%
risk_positive_rate_pred = 0.00%
risk_precision = 0
risk_recall = 0
```

这说明 risk head 完全失效。

可能原因：

1. `detach_risk_vol_heads=True` 让 risk/vol head 无法通过 shared trunk 适配新结构后的表示变化。
2. `risk_weight=0.07` 偏低。
3. risk class weight 被“正类下调”后，与 detach 叠加导致 risk 正类梯度不足。
4. 新结构训练目标主要偏向 return/cum return，risk 被边缘化。

修正方向：

```text
0062 默认关闭 detach_risk_vol_heads；
risk_weight 恢复到 0.08 或 0.10；
取消 risk 正类权重下调；
保留 risk 分布 hard gate。
```

### 4.4 cum_direction_head 无效

0061：

```text
cum_direction_head_acc = 40.54%
cum_direction_from_return_acc = 46.62%
```

两者都低于随机附近水平，说明累计方向任务没有被正确学习。

可能原因：

1. `cum_direction_logit` 是新随机 head，但从 0060 checkpoint 直接联合训练，容易被其他任务干扰。
2. `cum_direction_head_weight=0.12` 不一定太低，问题更可能是输入表示或 label/weight 与 direction 分类头冲突。
3. `cum_direction_weight=0.045` 继续作用在 `return_pred.sum()` 上，和独立 `cum_return_pred` / `cum_direction_logit` 形成多重目标拉扯。

修正方向：

```text
0062 暂时降低或关闭 cum_direction_head_weight；
先以 cum_return_pred 的符号作为累计方向来源；
观察 cum_return_ic 与 sign(cum_return_pred) 的方向准确率；
确认 cum_return 分支稳定后，再重新训练 cum_direction_logit。
```

### 4.5 选模逻辑需要修正

0061 报告中：

```text
0061 best_selection_mode=hard_gated
但最终 direction/risk 分布明显不满足 gates

0061b best_selection_mode=last_epoch
```

这暴露两个问题：

1. hard gates 主要保护 valid，但 test 仍可坍缩，说明 valid gate 不足或 valid/test 分布差距大。
2. 当没有合格 checkpoint 时，last epoch 被保存为 best，容易产生误导。

修正方向：

```text
没有 eligible checkpoint 时，不应输出 decision=可比较模型；
报告必须明确 no_valid_checkpoint=true；
last_epoch 只能用于诊断，不能作为模型候选；
hard gates 必须加入 direction/risk 预测分布下限。
```

## 5. 对《架构师-007》的修正

### 5.1 保留内容

以下指导仍然有效：

- 0061 的结构目标是验证新 return/cum return 分支。
- `cum_return_ic` 必须作为核心指标。
- 不应回到只调旧 loss 权重。
- 不应扩大主干。

### 5.2 需要修正内容

原指导中：

```text
detach_risk_vol_heads=True
balanced_class_weights=True
cum_direction_head_weight=0.12
direction_weight=0.38
risk_weight=0.07
```

应修正为 0062 配置：

```text
detach_risk_vol_heads=False
balanced_class_weights=False
use_class_weights=True 或先关闭 direction class weights
direction_weight=0.30
risk_weight=0.09
cum_direction_head_weight=0.03
cum_direction_weight=0.00
cum_return_weight=0.18
return_consistency_weight=0.01
```

解释：

- 关闭 detach，恢复 risk/vol 对 trunk 的适配能力。
- 降低 direction 分类梯度，避免 down 类坍缩。
- 降低 cumulative direction head 权重，先让 cum return 分支稳定。
- 提高 cum return 权重，继续利用当前唯一明显成功的新分支。
- 降低 consistency，避免 step return 被 cum head 过度牵制。

## 8. 需要软件工程师修正的点

### 8.1 增加分类坍缩 hard gates

位置：

```text
examples/train_market_state_model.py
```

在 `valid_hard_gates()` 中，`return_direction_branch` 阶段应加入：

```text
direction_pred_c0 <= 0.60
direction_pred_c1 >= 0.08
direction_pred_c2 >= 0.10
risk_positive_rate_pred >= 0.05
risk_positive_rate_pred / risk_positive_rate_true <= 1.8
```

当前只检查 risk ratio 区间和部分指标，不足以阻止 direction/risk 坍缩。

### 8.2 last_epoch 不得作为 best model

当前没有 eligible checkpoint 时会保存 last epoch。建议修改为：

```text
如果无 eligible checkpoint：
  - 保存 diagnostic_last.pt
  - metrics.json 标记 no_valid_checkpoint=true
  - decision=reject
  - 不写 market_state_best.pt，或写入但明确 best_selection_mode=diagnostic_last
```

避免训练师误用 last epoch 作为可候选模型。

### 8.3 class weight 拆分配置

当前 `use_class_weights` 同时影响 direction/risk。建议拆分为：

```text
--direction-class-weights / --no-direction-class-weights
--risk-class-weights / --no-risk-class-weights
```

原因：

- direction 出现 down 坍缩；
- risk 出现全负坍缩；
- 两者需要不同处理，不能共享一个开关。

### 8.4 risk/vol detach 改为阶段性开关

建议新增：

```text
--detach-risk-vol-after-epoch N
```

训练前期不 detach，让 risk/vol 适配新 head；中后期可 detach 保护 return/cum return。

默认：

```text
detach_risk_vol_heads=False
```

## 10. 是否需要再次修改模型结构

当前暂不建议继续新增更复杂结构。

理由：

- `cum_return_ic=0.105` 证明结构改造中的 return/cum return 分支有效。
- 当前失败主要来自 direction/risk 分支坍缩与训练/选模约束不足。
- 在修复 class weights、detach 策略和 hard gates 之前，不应继续添加新结构。

需要结构层修正的只有两点：

1. 保留 `HorizonReturnHead` 和 `cum_return_pred`。
2. 暂缓强化 `cum_direction_logit`，必要时让其依赖 `cum_return_pred` 或降低权重。

如果 0062 修正后仍出现：

```text
cum_return_ic 高，但 cum_direction_from_return_acc < 52%
```

再考虑将 cumulative direction label 从二分类改为三状态：

```text
cum_down / cum_flat / cum_up
```

而不是继续用二分类头。

## 11. 下一步结论

0061 的结果给出明确判断：

```text
新累计收益分支有效；
direction/risk 训练坍缩；
cum_direction head 暂时无效；
问题优先级是训练约束与分支权重，而不是继续扩展结构。
```

0062 应聚焦：

```text
保留 cum_return_ic >= 0.08
恢复 direction_macro_f1 >= 0.32
恢复 risk_f1 >= 0.50
保持 volatility_mae <= 0.070
观察 cum_direction_from_return_acc 是否回到 >=54%
```

只有完成这些修正后，才继续追求 `cum_direction_head_acc >= 58%`。
