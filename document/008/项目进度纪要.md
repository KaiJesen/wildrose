## 3. 架构师判断

### 3.1 008 指导是否被验证

- **0061 失败根因判断成立**：`detach_risk_vol_heads`、`balanced_class_weights`、direction 梯度过强、hard gates 不足，是坍缩主因。
- **cum return 分支有效**：`cum_return_ic` 从 0061 的 0.105 提升至 0062e 的 **0.135**，应作为新结构核心能力保留。
- **训练工程修正有效**：坍缩 hard gates、class weight 拆分、`no_valid_checkpoint` 诊断、last_epoch 不作 best，均已落地并发挥作用。

### 3.2 return_ic 与 cum_return_ic 的 trade-off

这不是单次随机波动，而是 **可复现的结构性行为**：

- 0062c–f 在提高 `return_weight`、关闭 direction class weights、延长训练后，`return_ic` 仍无法回到 0.035。
- valid 上 step `return_ic` 多为负，但 test `cum_return_ic` 持续为正且更强。
- 说明 trunk 表征已被 **累计收益排序** 主导，逐步收益排序能力被挤压。

**架构师裁定**：在 `return_direction_branch` 阶段，不再将 step `return_ic >= 0.035` 作为 release 硬否决项；改为：

```text
收益排序主指标：cum_return_ic >= 0.08
收益排序观察项：return_ic > 0（硬性底线，防完全失效）
成熟阶段再追求：return_ic >= 0.035 与 cum_return_ic 双达标
```

### 3.3 为何不替换 0058 主基线

0058 仍在以下维度更优：

- step `return_ic`：**0.054** vs 0062e 的 0.006
- usable 路线历史验收完整（5/5，无结构 trade-off 争议）

0062e 的优势在 **新结构能力**：cum return 排序、更均衡的 direction 分布、更低的 volatility_mae、更高的 risk_f1。

因此采用 **双基线策略**，而非单点替换。
