## 1. 文档说明

| 项 | 内容 |
|----|------|
| 角色 | 软件设计师 |

| 依据 | `document/016/软件设计师-016-趋势输出模块设计与集成方案.md`、`document/017/软件设计师-017-最佳买卖点监督信号模型设计.md` |

本文启动 018 项目。目标是解决当前系统“短收益能吃到，但长段收益吃不到”的问题，尤其是 6 月初持续下跌中已经触发 crash/trend 信号，但持仓过早退出、无法形成趋势仓的问题。

```text

crash_short_count = 7
```

结论：

```text

6 月初长跌主要仍未吃到。
```

## 3. 6 月初失败原因

### 3.1 趋势已经识别，但没有转化为持仓

6 月 1 日附近，系统已经多次输出：

```text
trend_direction = DOWN
trend_strength = NORMAL / STRONG
trend_phase = CONTINUATION / ACCELERATION
trend_is_confirmed = true
is_model_blind_crash = true
```

但系统多数时间仍为：

```text
HOLD_NO_ENTRY
FLAT
```

说明：

```text
趋势识别模块已经看到下跌；
交易状态机没有把“已确认趋势”转化为“可持续持仓”。
```

### 3.2 crash 仓被快速失败规则过早平掉

6 月初关键交易：

```text
2026-06-01 14:00 OPEN_SHORT_CRASH
2026-06-01 16:00 CLOSE_CRASH_FAILED
```

当时：

```text
trend_direction = DOWN
trend_phase = CONTINUATION
trend_is_confirmed = true
```

但依然因为 crash failed 退出。

判断：

```text
CLOSE_CRASH_FAILED 对价格短反弹过于敏感；
没有区分“崩跌失败”与“趋势中的正常反抽”。
```

### 3.3 配置禁止 crash 仓升级为趋势仓

当前配置：

```text
trend_position.allow_crash_trend_upgrade = false
```

结果：

```text
crash short 即使盈利、且趋势确认，也不能升级为 SHORT_TREND；
只能按 CRASH 短逻辑快速管理。
```

这是 6 月初不能持住的重要原因。

### 3.4 017 best-point 标签天然偏短

017 标签统计：

```text
num_trades = 2790
avg_holding_bars = 2.91
label_coverage = 90.7%
entry zone 非常密集
exit zone 非常密集
```

判断：

```text
当前 best-point 模型学到的是高频最佳点/短线机会，
不是长趋势持有能力。
```

因此 017 observe 的 best-point 输出不能直接用于平仓；否则会强化短持仓。

---

## 4. 018 总体方向

018 不再继续增加开仓信号，而是新增：

```text
LongTrendTradeManager
```

核心目标：

```text
把已经出现的 crash / model short / trend confirmed 状态，
转化为可持续持仓；
并用趋势破坏而不是短线噪声决定退出。
```

018 由四部分组成：

```text
1. 长趋势标签重建：训练/评估不再只看短线最优点；
2. 持仓生命周期状态机：ENTRY -> PROBATION -> TREND -> DISTRIBUTION -> EXIT；
3. crash 仓升级机制：允许模型失效下的崩跌仓进入趋势仓；
4. 分层退出机制：禁止单根反抽或 best-point exit 直接清仓。
```

---

## 5. 新增概念：趋势交易生命周期

新增状态：

```text
PositionLifecycle:
  NONE
  PROBATION
  TREND
  PROTECT_PROFIT
  EXIT_PENDING
```

含义：

```text
PROBATION：刚开仓，等待趋势确认或失败；
TREND：趋势确认，允许延长持仓；
PROTECT_PROFIT：已有较大浮盈，重点保护利润；
EXIT_PENDING：趋势可能失效，等待确认退出。
```

新增字段：

```text
position.lifecycle
position.lifecycle_bars
position.min_hold_until
position.trend_confirmed_at
position.exit_pending_count
position.best_point_exit_count
position.trend_pullback_count
```

## 6. 修改一：允许 crash 仓升级为趋势仓

### 6.1 配置修改

当前：

```text
allow_crash_trend_upgrade = false
```

018 改为：

```text
allow_crash_trend_upgrade = true
crash_upgrade_profit_atr = 0.8
min_trend_age_for_upgrade = 2
```

### 6.2 升级条件

crash short 满足以下条件，升级为趋势仓：

```text
position.entry_was_crash == true
position.side == SHORT
trend_signal.direction == DOWN
trend_signal.is_confirmed == true
trend_signal.phase in [CONTINUATION, ACCELERATION]
trend_signal.trend_age >= 2
profit_atr >= 0.8
p_risk < risk_exit_threshold
```

动作：

```text
UPGRADE_CRASH_TO_TREND_SHORT
hold_mode = TREND
lifecycle = TREND
```

注意：

```text
不要求 0062e 后续给 OPEN_SHORT_SIGNAL；
因为 015 已经确认模型可能在 crash regime 中失效。
```

---

## 7. 修改二：趋势确认后设置最短持有期

趋势仓不应刚升级就被短线反向信号洗出。

新增配置：

```json
{
  "trend_lifecycle": {
    "min_trend_hold_bars": 6,
    "strong_min_trend_hold_bars": 10,
    "exit_confirm_bars": 2,
    "bp_exit_confirm_bars": 3
  }
}
```

规则：

```text
进入 TREND 后，
在 min_trend_hold_bars 内，
不能因 CLOSE_REVERSE_SIGNAL、CLOSE_CRASH_FAILED、bp_exit 单独退出。
```

仍然允许：

```text
CLOSE_HARD_STOP
CLOSE_CATASTROPHE_MARGIN_LOSS
CLOSE_RISK_PROB_HIGH
```

原因：

```text
趋势仓必须给市场正常回撤空间；
否则永远只能吃到 1~3 根 K 的收益。
```

---

## 8. 修改三：重写 `CLOSE_CRASH_FAILED`

当前失败条件对短反弹敏感。

018 改成：

```text
如果 trend_signal.direction == DOWN
且 trend_signal.is_confirmed == true
且 trend_signal.phase in [CONTINUATION, ACCELERATION]
则不允许 CLOSE_CRASH_FAILED。
```

新的 crash failed 条件：

```text
price > entry_price + fail_stop_atr * atr
且 trend_signal.is_broken == true
```

或：

```text
crash_votes < min_crash_votes 连续 2 根
且 trend_signal.direction != DOWN
```

动作：

```text
CLOSE_CRASH_FAILED
```

新增动作：

```text
HOLD_CRASH_TREND_CONFIRMING
```

用于：

```text
crash_votes 暂时回落，但趋势仍确认下跌。
```

---

## 9. 修改四：best-point 输出只做二级证据

017 observe 中，best-point 对 6 月初经常给出较高：

```text
p_exit_short
```

但 hindsight 标签平均持仓仅 2.91 根，因此这更像短线退出噪声。

018 规则：

```text
bp_exit 信号不得单独触发趋势仓清仓；
只能作为 exit_pending 的一票。
```

趋势空头退出投票：

```text
trend_broken                         +2
price > ema_fast 连续 2 根            +1
ret_6_atr >= +1.2                    +1
p_exit_short >= 0.70 连续 3 根        +1
p_hold_short <= 0.30 连续 3 根        +1
0062e pred_cum_ret_5 > 0 连续 2 根    +1
```

退出：

```text
exit_votes >= 3
```

动作：

```text
CLOSE_TREND_EXIT_CONFIRMED
```

## 10. 修改五：新增长趋势标签，不再依赖短线最佳点

017 标签来自全局最优交易，平均持仓太短。018 需要新增长趋势标签集：

```text
best_point_long_horizon_v018
```

### 10.1 标注方式

在 `optimal_trade_points.py` 基础上新增长周期标注配置：

```text
min_net_roi 提高到 0.15~0.30
min_holding_bars = 8
max_holding_bars = 72
cooldown_after_trade = 3
```

新增过滤：

```text
只保留 raw_return 足够大、持仓 >= 8 bars 的趋势交易；
短于 8 bars 的交易只用于 017，不用于 018。
```

### 10.2 标签

新增：

```text
trend_entry_label:
  0 = no_trend_entry
  1 = long_trend_entry_zone
  2 = short_trend_entry_zone

trend_hold_label:
  0 = no_trend_hold
  1 = hold_long_trend
  2 = hold_short_trend

trend_exit_label:
  0 = no_trend_exit
  1 = exit_long_trend
  2 = exit_short_trend

trend_quality:
  该趋势段净收益 / 最大回撤 / 持仓 bars 的综合分数
```

### 10.3 模型定位

第一版不重新训练大模型，可先把长趋势标签用于：

```text
离线评估 TrendSignalProvider；
校准 trend_upgrade 条件；
评估 best-point 模型是否短视。
```

第二版再训练：

```text
TrendHoldSignalModel
```

## 11. 修改六：趋势仓分层止盈

当前趋势仓容易在 `CLOSE_TREND_BROKEN` 或短 trailing 下过早退出。

018 引入分层止盈：

```text
profit_atr < 1.5:
  仍在 PROBATION，失败则退出；

1.5 <= profit_atr < 3.0:
  进入 TREND，止损移动到接近保本；

3.0 <= profit_atr < 5.0:
  进入 PROTECT_PROFIT，允许回撤 1.5 ATR；

profit_atr >= 5.0:
  只退出 50%，剩余仓位用宽 trailing，允许吃尾段。
```

新增动作：

```text
REDUCE_TREND_PROFIT_LOCK
HOLD_TREND_RUNNER
```

目的：

```text
不再一次性卖飞整段趋势；
保留 runner 仓位吃长尾收益。
```

## 15. 不建议做的事

不建议：

```text
继续只调 crash_short 开仓阈值；
让 best-point exit 单独触发平仓；
用 017 短线标签训练长趋势持仓；
把所有 crash 仓无条件持满 18 bars；
取消硬止损或 p_risk 极端退出；
为了吃 6 月初一段而允许同 regime 反复开仓。
```

原因：

```text
这些做法要么回到 014a 的过度交易，
要么把短线标签误用成长线持仓标签，
要么造成单段过拟合。
```

## 16. 一句话结论

017 observe 已证明：

```text
系统不是没有看到 6 月初下跌；
而是看到后没有把仓位升级成趋势仓，并且过早执行失败退出。
```

018 的核心修正是：

```text
新增持仓生命周期；
允许 crash 仓在 confirmed trend 下升级；
用趋势退出投票代替单根失败退出；
重建长趋势标签，避免被 017 短线最佳点标签误导。
```
