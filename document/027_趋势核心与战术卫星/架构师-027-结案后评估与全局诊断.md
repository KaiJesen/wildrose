## 3. 根因分析：为什么 023→027 全序列未能突破 coverage 天花板

### 3.1 问题穿越四个项目

| 项目 | 尝试方向 | 核心失败模式 |
|------|----------|-------------|
| 023 | TEQ 规则参数调优 | wp 抬至 0.50 后 return 崩溃 |
| 024 | 模型腿对齐训练 | TEQ 信号增强，coverage 升至 26.67% |
| 025 | 多通道规则扩展（slow_up / std_trend） | A3a/A3b/A3c 穷举无效——通道条件从未满足 |
| 026 | 参与信号训练（C3/C1/A1） | 模型轨 PASS，coverage 膝点未破（26.67%） |
| 027 | 双 Slot 架构 | Core 松绑 + Satellite 接入三重否定 |

### 3.2 根因分层

**第一层：信号面——`slow_up` 通道条件从未被触发**

```
024 TEQ 依赖 leg 确认 + market_state 预测 → 仅在快跌/快涨 leg 触发
slow_up 通道（慢涨、无 leg 确认）→ 023→027 全程 0 开仓
并非「没有慢涨区」（Phase 2a 证实 523 根 slow_up bar），
而是「慢涨通道的参与条件从未被满足」。
```

这是 023→027 序列中**贯穿始终的结构性盲区**——四条研究线均瞄准 fast leg 方向，slow_up 通道始终未被覆盖。

**第二层：架构面——017 BP 独立择时力不成立**

```
Phase 2 8/8 网格 Sharpe 全负：即使给 BP 独立资金、独立决策全权
→ 回测窗内 BP 信号无法自盈利
Phase 2a 范围缩小 0 笔交易：即使只限定 slow_up 窗口
→ BP 信号在这些 bar 上无法成交
```

017 BP 是一种「在已知方向上的最优买卖点推测」信号——它需要方向确认（Core 的 crash/trend leg 方向）来支撑。当 Core FLAT 且方向中性时，BP 自身没有独立的多空方向输出，自然无成交。

**第三层：数据面——valid/test 分裂**

| 窗口 | B0 return | M2 return | 017 BP valid Sharpe |
|------|-----------|-----------|---------------------|
| valid（前 365 天） | **−3.69%** | −6.21% | **−1.35** |
| test（后 55 天） | **+9.01%** | +9.14% | —（未跑，027 P3 被禁） |

valid 窗是全策略困难窗口。B0 在此也仅 −3.69%。所有优化在 valid 上均无边际效应（Phase 1 6 臂全同值）。这意味着 valid 窗口本身可能缺乏策略可识别的行情模式——此窗口的"无信号"可能不是策略的问题，而是行情的问题。

**第四层：方法论——单标的小样本回测的固有局限**

BTCUSDT 1h、valid 365 天 + test 55 天（总计 420 天、~10,000 bar）上的时序回测，在一个高度非平稳的标的上，valid/test 的统计相关性天然弱。valid 全负但 test 全正并非特例——这是单标的小样本时序切分的固有属性。


## 5. 对 028 的定向意见

### 5.1 方向 A（推荐优先 — slow_up 专项）

**问题**：slow_up 通道在 023→027 贯穿始终未被覆盖（valid 523 bar、test recovery 176 bar）。不是缺信号，是 slow_up 的"可参与标准"从未被触发。

**方向**：重新设计 slow_up 通道的参与规则，**不依赖 leg 确认、不依赖 TEQ、不依赖 BP**。

具体思路：
- 定义 slow_up 趋势状态下的"动量突破+回踩确认"参与信号
- 可基于 channel_edge 的价格位置 + 短周期动量（如 12h ROC）+ 波动率压缩
- 完全规则层（Rule Layer），不涉及模型重训
- 验收：valid slow_up 分窗 return > 0（与 Phase 2a 同门禁）

**理由**：slow_up 是 023→027 已知的唯一可改善盲区。Phase 2a 已证明 BP 信号无法填补——需要专门设计的规则包。

**此方向的劣势**：valid 窗口困难（全策略负），即使规则设计合理也可能无正信号。

### 5.2 方向 B — 更强 participate 模型 + 通道逻辑闭环

**问题**：026 M2 的 part_auc 0.697 在 valid 上无法转化为 TEQ 开仓增量（M2 baseline return −6.21% 劣于 B0 −3.69%）。

**方向**：
1. 模型侧：参与头输出「通道条件分数」而非单一 logit——按 crash/slow_up/trend 三维输出
2. 规则侧：channel_edge 兑换为「通道可参与阈值」——M2 的 edge 以 part_auc 0.697 作为通道映射基础
3. 训练侧：窗口选择（不仅是 4h↔1h 对齐，而是按 regime 切分训练样本）

**理由**：这是 025 channel_edge + 026 C3 的延续，利用已积累的工程资产。

### 5.3 方向 C — 更长样本 / 多标的验证

**问题**：valid/test 统计分裂可能源于样本不足（420 天 BTCUSDT 1h）。

**方向**：
- 扩大历史窗口至 2~3 年，增加 valid/test 的样本量
- 纳入 ETHUSDT 作为交叉验证（同一策略体系，不同标的）
- 多标的共享 TEQ / channel_edge 框架

**此方向的劣势**：工程量较大（数据获取、回测引擎适配），且 420 天已覆盖一个完整牛熊周期。

### 5.4 方向 D — 暂停 new research，巩固 prod

**方向**：暂时不再推进 new research 支线，将注意力放在 prod v1.1.0 的监控与稳健性优化上。

具体动作：
- 持续监控 B0 0062e 的 live metrics（return、max_drawdown、coverage、trade count）
- 建立未成交原因的自动归因（如 WATCH_SLOW_UPTREND 持续期、TEQ reject 原因分布）
- 收集 live 数据，待样本足够后再启动 028

## 6. 推荐结论

| 优先级 | 方向 | 理由 | 前置 |
|--------|------|------|------|
| **P0** | 方向 D — 巩固 prod | 稳健性优先；023→027 全序列无 new prod baseline | 无 |
| **P1** | 方向 A — slow_up 专项规则 | 唯一已知的可改善盲区；低工程成本 | 028 立项 |
| P2 | 方向 B — 更强 participate 模型 | 需模型训练资源 | 028 立项（可与 A 并行） |
| P3 | 方向 C — 多标的 / 长样本 | 工程量大；收益不确定 | 待 A/B 结果后评估 |

**架构师意见**：

```text
027 结案证明了一个事实：当前数据窗内，通过"规则松绑 + BP 战术层接入"
无法突破 single-slot 的 coverage 天花板。这不是架构问题，是信号问题。

建议：
1. 暂不启动 028 立项（除非方向 A 的研究假设足够清晰）；
2. 优先巩固 prod v1.1.0（监控 + 归因）；
3. 方向 A 可在 028 立项时作为 Phase 0 快速验证；
4. 026 M2（v1.1.2）保留为研究基线——其 part_auc 0.697 是未来更强参与模型的起点。
```
