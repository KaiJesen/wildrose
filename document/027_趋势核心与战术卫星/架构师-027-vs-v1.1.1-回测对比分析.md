# 027 vs v1.1.1 回测对比分析

## 1. 版本定义

| 版本 | checkpoint | 规则配置 | TEQ wp | 架构 |
|------|-----------|----------|--------|------|
| **v1.1.1 (B0)** | `0065a_leg_align_c1_pw20` | phase1c TEQ + legacy crash + slow_up | 0.35 | 单 Slot |
| **027 Core** | `026_phase1_c1d1` (M2) | phase1c TEQ + legacy crash + slow_up | 0.30 | 双 Slot（Phase 0 等价于 B0） |
| **027 Core+Sat** | 同上 + 017 BP | 同上 + `require_core_flat` + τ=0.60 | 0.30 | 双 Slot Core+Satellite |

## 2. valid 窗口对比（可比较窗口）

### 2.1 核心指标

| 指标 | v1.1.1 B0 | 027 Core (M2) | 027 Core+Sat | 评注 |
|------|-----------|---------------|--------------|------|
| **return** | **−3.69%** | −4.12% | −11.47% | B0 最优，Sat 接入后严重恶化 |
| **max_drawdown** | **−3.69%** | −4.30% | −14.05% | B0 回撤最小 |
| **coverage** | 18.75% | **25.00%** | — | M2 覆盖更高但 return 更差 |
| **trade_count** | — | 22 | 33 | M2 交易更多（TEQ=4 vs B0 TEQ=2） |
| **TEQ trades** | 2 | 4 | 4 | M2 多开 2 笔但无效 |
| **crash trades** | 4 | 4 | 4 | crash 一致 |
| **slow_up trades** | 0 | 0 | 0 | 两版本均有 slow_up 盲区 |
| **Sharpe** | — | −2.21 | **−3.65** | 027 组合后风险报酬比恶化 |

### 2.2 结论

```text
valid 窗口上：v1.1.1 B0 > 027 Core > 027 Core+Sat
```

**v1.1.1 B0 胜出**：return −3.69% vs −4.12%（+0.43pp），MDD 更浅，交易更少（TEQ 2 vs 4）。M2 的 part_auc 更高（0.697）但 valid 上未兑现为更好的 return——多开的 2 笔 TEQ 交易无法盈利。

**027 Core+Sat 远差**：接入 017 BP 后 return 从 −4.12% 恶化到 −11.47%（−7.35pp），MDD 从 −4.30% 放大到 −14.05%。双 Slot 的组合效应在 valid 上完全是**负协同**。

## 4. 优势对比（非回测维度）

### 4.1 027 相对于 v1.1.1 的**优势**

| 维度 | 说明 |
|------|------|
| **架构扩展性** | 双 Slot（Core+Satellite）设计允许未来接入任意数量的战术策略，单 Slot 只能跑一条规则链 |
| **DAD 裁决框架** | `portfolio_slots.py` 提供了方向对齐默认（DAD）+ 跨 Slot 冲突解决——这是独立于 027 实验结果的工程资产 |
| **Phase 0 等价证明** | 已证明双 Slot FLAT ≡ 单 Slot B0，新策略插入前可以零风险验证等价性 |
| **shadow core 机制** | `engine_dual.py` 的 shadow core 在 Sat 自有持仓时仍追踪 Core 虚拟仓位，用于 DAD 判断——这是成熟的多策略管理模式 |
| **Satellite 门控体系** | `require_core_flat` + `require_core_watch_slow` 等条件门控可用于未来的战术策略接入 |

### 4.2 027 相对于 v1.1.1 的**劣势**

| 维度 | 说明 |
|------|------|
| **valid 表现** | 所有 027 配置（Core/Core+Sat/Sat-only）均劣于 v1.1.1 B0 |
| **复杂度上升** | 双引擎 + 双钱包 + DAD 裁决 + shadow core → 代码量、测试面、调试难度均翻倍 |
| **Satellite 信号失效** | 017 BP 在 valid 上 8/8 网格全负——证明在无方向信号的前提下，BP 独立择时力为零 |
| **无 test 验证** | 027 的 test 表现未知——既不知道是否优于 B0，也不知道是否进一步恶化 |
| **slow_up 盲区未解决** | 两版本 slow_up 开仓均为 0——027 的 Satellite 也无法填补此盲区（Phase 2a 523 bar 0 笔） |
| **M2 wp=0.30 的风险** | 低于 B0 的 wp=0.35，会导致 TEQ 更频繁触发——valid 上 TEQ=4 vs B0 TEQ=2，多出的 2 笔无效 |

## 5. prod v1.1.0 的对照（供参考）

| 版本 | test return | test MDD | test trade_count |
|------|-------------|----------|-----------------|
| **prod v1.1.0 (0062e)** | **+10.09%** | −0.85% | 10 |
| v1.1.1 B0 (c1_pw20) | +9.01% | −1.21% | 13 |

prod v1.1.0 用的是更旧的 0062e checkpoint，规则也无 TEQ 参与。当前 test 上表现反而更好（+10.09% vs +9.01%），但 trade_count 已从基线 8 漂移到 10（+25%），需警惕。

**关键信息**：v1.1.1 B0 的 test return +9.01% 是在 TEQ 接入后的结果——引入 TEQ 后 coverage 从 16.7% 升至 26.67%，但 return 从 +10.09% 微降到 +9.01%。多开仓 ≠ 多赚钱。

## 6. 总体评价

```text
回测表现：v1.1.1 B0 完胜 027。

v1.1.1 的优势：
  · valid −3.69% vs 027 Core −4.12%（+0.43pp）
  · 单 Slot 简单、稳定、易于维护
  · test +9.01% 已有实证，027 test 不存在

027 的唯一价值：
  · 双 Slot 架构（portfolio_slots.py + engine_dual.py）作为可复用工程资产入库
  · Phase 0 等价证明框架可用于未来任何新策略插入前的硬门禁

027 不能替换 v1.1.1，也不建议替换。
027 的架构设计正确但信号土壤不支持——这是一个「工具对、时机错」的项目。
```
