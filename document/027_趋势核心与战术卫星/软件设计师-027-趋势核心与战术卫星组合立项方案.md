## 2. 立项背景

### 2.1 026 结案证据（`backtest/v026_closure/`）

| 臂 | return | coverage | 交易数 | FLAT 占比 |
|----|--------|----------|--------|-----------|
| B0 | 9.01% | 26.67% | 13 | ~81% |
| M2 | 9.14% | **26.67%** | 14 | ~81% |
| M3 | 8.07% | **26.67%** | 14 | ~81% |

| 现象 | 量化证据 | 根因层级 |
|------|----------|----------|
| 大部分时间无交易 | **0.24 笔/天** | 规则选择性 + 单通道 |
| coverage 膝点 | 三臂均 **26.67%** | TEQ 饱和；**非模型 alone** |
| 6 月主跌只吃 ~1/3 | 价格 **−14%**；策略单笔 **+6.6%** | crash 晚触发、`same_regime_once`、短 hold |
| slow_up 未落地 | **500** 根 WATCH；**open=0** | `enabled=false`（025 A3a 亦失败） |
| 单仓位架构 | `PortfolioState.position` 唯一 | 趋势与短线互斥 |

### 2.2 用户目标澄清（纳入 027 约束，非硬 KPI）

| 用户表述 | 027 裁定 |
|----------|----------|
| 趋势 = 中长线、低频率 | → **Core Slot**（TEQ + leg 生命周期） |
| 指标 = 短线、高频率 | → **Satellite Slot**（best-point + 短 horizon） |
| 共用一个账户 | → **AccountEquity** + DAD 冲突裁决 |
| 可同时多仓 | → 双 Slot **同向可叠加**；**禁止跨 Slot 反向**（DAD） |
| 7 天 10%+ | **不作稳定 KPI**；仅 stress 情景（§4.3） |

### 2.3 继承的不可突破约束（024/025/026/027 架构师裁定）

```text
1. return 与 coverage 双轨验收，禁止 coverage-only；
2. test split 一次性验收，禁止 test 调参；
3. cum_return_ic 相对 0062e 退化 ≤10%（Core 用 026 模型时）；
4. prod v1.1.0 替换须 Phase 4 双门 + 项目经理 + 架构师书面批准；
5. 027 主路径不以「再训 part 头」为第一优先级；
6. 探索门维持 return ≥8.84%、coverage ≥28%（不得降低）；
7. DAD 为跨 Slot 冲突默认立场（见冲突裁决表 v1）；
8. Phase 0「双 Slot 均 FLAT ≡ B0」失败 → 不得进入 Phase 1。
```

## 3. 问题归纳（根因链）

```text
PortfolioState.position 单一（LONG|SHORT|FLAT）
  → 持 trend 时无法开 tactical；平仓后 regime 门阻止再入
  → TEQ 为唯一有效开仓路径（teq_open≈3），coverage 卡在 26.67%
  → 026 三 checkpoint 同 coverage → 排除模型信号残余
```

**027 核心假设**：突破 coverage 需 **（1）双 Slot 并行** + **（2）Core 规则松绑（valid 实证）** + **（3）Satellite 独立预算**，而非第四次调 w_part。

## 4. 目标与范围

### 4.1 总目标

```text
主目标 = 探索线双门 PASS（test return ≥8.84%，coverage ≥28%）
       + Core / Satellite / 合计 分策略归因可复现（附录 B 模板）
       + 账户 max_drawdown ≥ −5%（探索线有条件接受，见 §11）
       + 0062e legacy cum_IC 退化 ≤10%（Core 路径）；

不承诺：稳定「7 天 ≥10%」；
027 PASS：可提名 v1.1.2 研究包；不默认替换 prod。
```

### 4.2 双轨验收（继承 024/025/026 + 架构师 §7）

| 轨道 | 门槛 | 说明 |
|------|------|------|
| **探索线**（test 一次性） | return ≥ **8.84%**；coverage ≥ **28%** | 与 023 起一致，**不得降低** |
| **研究投产线**（Phase 4） | return ≥ **9.31%**；coverage ≥ **35%** | 条件评估 |
| **Satellite 独立**（valid） | trades ≥8；**Sharpe ≥ 0** | 禁止以 Core 拉平 Sat 亏损 |
| **Core 独立**（valid） | coverage ≥ B0+1.5pp；return ≥ B0−0.5pp | 证明 Core Slot 有效 |

### 4.3 收益预期（写入 REPORT，非合同 KPI）

| 窗口 | B0 实测 | 027 合理区间（须分解归因验证） |
|------|---------|-------------------------------|
| 滚动 7D 最好 | +6.5% | +7~9% |

**说明**：§4.3 区间须在 Phase 1/2 以 **Core standalone / Satellite standalone / Combined** 分解表验证（附录 B），不得仅用合计数掩盖单 Slot 失败。

### 4.4 范围

**In scope**

- 双 Slot 组合账户 + DAD 冲突裁决
- Core：TEQ / crash / trend（slow_up **valid 对照后**可能启用）
- Satellite：017 best-point 从 observe 升级为实仓 Slot
- `run_v027_*` 脚本、分 Slot 归因、6 月案例回归

**Out of scope**

- 跨 Slot 反向对冲（DAD 例外须架构师额外批准）
- 多 symbol / 实盘对接

## 5. 方案设计

### 5.1 账户架构：Core + Satellite

```text
AccountEquity（账户级 DD、总 margin、DAD 门控）
├── slot_core      (budget 55–65%, max_gross 12–15%)
│   ├── 通道: TEQ / trend_qualified / crash / slow_up（对照后）
│   ├── 模型: checkpoints/026_phase1_c1d1（M2，冻结）
│   └── 目标: leg coverage、波段持有
│
├── slot_satellite (budget 25–35%, max_gross 8–10%)
│   ├── 通道: best-point + 短 edge（017，冻结 checkpoint）
│   ├── hold: 6–48 bars
│   ├── DAD: Core 有仓 → 仅同向或 FLAT；Core FLAT → 可双向
│   └── 目标: 填空白期、局部波段
│
└── account_risk   (日/周 DD、总杠杆、P0 全平)
```

**当前硬伤**（`portfolio.py` 单 `position`；`engine.py` 单决策循环）→ 须重写净值核算、决策顺序、总风险预算。**非「加个 dict」级改动**（架构师 §5.1）。

**引擎改造要点**：

| 模块 | 改动 |
|------|------|
| `portfolio.py` | `AccountEquity`：`core_position` + `satellite_position` + netting |
| `engine.py` | Core step → DAD 过滤 → Satellite step → 账户校验 |
| `risk.py` | 总 `margin_used`、DAD 方向门、账户 DD |
| `rules.py` | `CoreRuleEngine` / `SatelliteRuleEngine` 或 `strategy_id` 参数化 |
| `logger.py` | `slot_id`、`slot_pnl`、`strategy_reason` |
| `backtest/report.py` | 附录 B 分解表 |

### 5.2 Core Slot 规则包

配置：`configs/trading_rule_v027_core.json`（基于 023 phase1c 主体）。

| 参数 | B0 | v1.1 初值 | 备注 |
|------|-----|-----------|------|
| `slow_up_position.enabled` | false | **false**（默认） | Phase 1 valid **enabled vs disabled** 对照；见 §6.2 |
| `crash_short.same_regime_once` | true | valid 网格 `[true, false]` | 6 月续跌再入 |
| `crash_short.max_hold_bars` | 30 | valid `[30, 48, 72]` | 独立维度，非全 factorial |
| `trend_position.max_trend_hold_bars` | 48 | valid `+0% / +50%` | 018 趋势仓 |
| TEQ `w_part` | 0.30 | valid 重调（M2 ckpt） | 继承 026 Phase3 |

**slow_up 强制关闭条件**（继承 025 A3a 裁定逻辑）：

```text
valid 上 slow_up_pnl < 0 且 slow_up_false_entry ≥ 3 → 强制 disabled，不得进入 test。
```

**禁止**：为抬 coverage 将 `edge_threshold` 降至 A3b 级别（025 已证 return 崩塌）。

### 5.3 Satellite Slot 规则包

| 项 | 设计 |
|----|------|
| 信号 | 017 `BestPointSignal` + `pred_cum_ret_5` |
| checkpoint | **冻结** hash 写入 Phase 2 报告 |
| 开仓 | 独立 `tau_satellite`、`edge_satellite`；不走 TEQ |
| DAD | Core LONG → 禁 Sat SHORT；Core SHORT → 禁 Sat LONG |
| 平仓 | best-point exit + max_hold 48 |
| sizing | `max_position_ratio` ≤ 8%；日开上限 valid 定 3–5 |
| 与 v1.0 差异 | 从 observe-only → **实仓 PnL 计入账户** |

### 5.4 冲突裁决（DAD，冻结）

完整条文见 **`软件设计师-027-Core-Satellite冲突裁决表.md` v1.0**。

```text
DAD：禁止跨 Slot 反向开仓（账户层净多净空并存不允许）。
Core FLAT 时 Satellite 可双向。
P0：账户日 DD → 全 Slot 平仓。
```

## 附录 C — 账户级 max_drawdown 定义

```text
peak_equity = max_t AccountEquity.equity(t)
drawdown(t) = (peak_equity - equity(t)) / peak_equity
```
