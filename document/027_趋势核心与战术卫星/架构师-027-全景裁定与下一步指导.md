## 4. 选项裁定

### 4.1 选项 1：Satellite 规则再设计（仅 Core WATCH_SLOW_UPTREND 时允许 Sat）— **有条件同意**

| 理由 | 说明 |
|------|------|
| **仅填 Phase 1 空白** | Core `WATCH_SLOW_UPTREND` 176 bar test、0 开仓是唯一已知的可改善盲区 |
| 017 BP 信号问题仍存 | 即使约束范围，Satellite 仍依赖 017 BP；BP 在 valid 上 8/8 负——需验证「只在慢涨窗内」时是否表现不同 |
| 范围缩小可快速验证 | 限制 Sat 仅在 `CoreFlatSlowUp` 条件时开仓 → 不推高复杂度 |

**裁定**：同意作为 **Phase 2a 最小实验**。前置条件：
```text
1. Satellite 仅在 {Core FLAT ∧ WATCH_SLOW_UPTREND} 时允许开仓；
2. max_daily_opens ≤ 1（此模式下）；
3. valid 独立验收：Satellite 分窗 return ≥ 0；
4. 2 天内出信号 → 无正值则 027 结案（options 3）。
```

**不建议**：在无实证时大幅重写 Satellite 规则引擎（scope creep）。

### 4.2 选项 2：重训 / 换 BP checkpoint — **暂不执行**

| 理由 | 说明 |
|------|------|

## 5. 全景诊断：023→027 中 valid 窗口的角色

### 5.1 跨项目 valid 表现一览

| 项目 | checkpoint | valid return | test return | ratio |
|------|------------|-------------|-------------|-------|
| 023 baseline (B0) | 0062e | −3.69% | **+9.01%** | −0.41 |
| 024 TEQ wp=0.35 | c1_pw20 | −1.71%（est） | **+9.01%** | — |
| 026 M2 baseline | 026_c1d1 | −6.21% | +9.14% | −0.68 |
| 027 Phase 2 core | 026_c1d1 | −4.12% | — | — |

```text
valid 窗始终为负、test 窗始终为正。
valid 窗可能是：低波动、高 CHOP、趋势反转集中；
test 窗（2026-04-30 ~ 06-23）包含 6 月显著下跌（B0 crash 贡献 +6.6%）。

关键推论：
  valid 上的 PHASE 门禁（如 return ≥ B0_valid − 0.5pp）是合理安全阀，
  但不能 100% 反映 test 上的策略质量。
  M2 return +9.14% on test 但 valid −6.21% → 表明 valid/test 分歧。
```

### 5.2 是否可以接受「valid 全负、test 已知正」直接推 Phase 3？

**否决**。理由：

1. Phase 1 的 valid 消融臂中，**所有参数变化完全无效**（同 return/coverage）。这意味着你无法在 valid 上区分「改了 crash hold 48」和「没改」——参数敏感性为零。**在敏感性为零的窗口上做的任何决策都是盲目的**。
2. Phase 2 Satellite 独立 8/8 负 Sharpe——即使「valid 窗困难」，−1.35 的 Sharpe 也很难仅由窗口属性解释。
3. 如果接受 valid 全负，相当于放弃了 Phase 1~2 的全部筛选功能——此后全依赖 test（违反一次性验收约定）。

**裁定**：维持 valid 门禁。不因 valid 困难就绕过。

## 9. 对 028 的提前判断（若 027 Phase 2a FAIL）

如果 Phase 2a 仍无法带来正边际贡献，027 结案。028 应着眼以下方向：

```text
方向 A：新数据窗口
  当前 valid+test 共计 ~420 天（1h），BTC 主要处于震荡→下跌→反弹。
  扩大历史窗口（2~3 年）可能提供更多 slow_up / trend / crash 模式。

方向 B：慢涨专项规则
  176 bar WATCH_SLOW_UPTREND、0 开仓是 023→027 贯穿的问题。
  不是缺模型信号，是 slow_up 条件从未被满足。
  可能需要重新定义 slow_up 的「可参与」标准（与 019 设计一致）。

方向 C：回到训练侧但方向不同
  不是再训 participation 头（C3/C1/A1 已做），
  而是训一个专门的「通道条件参与分类器」——
  输入 slow_up/crash/trend context + trunk features → 输出通道可参与概率。

方向 D：停止此研究线
  若 027 结案且 028 无清晰假设，保留 B0 为研究终点。
  prod 仅维护 0062e（7.77% / 16.7%），持续监控不主动退化。
```
