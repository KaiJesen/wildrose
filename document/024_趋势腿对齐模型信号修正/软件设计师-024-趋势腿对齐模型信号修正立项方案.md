## 4. 根因分析（模型侧）

### 4.1 时间尺度错配（P0）

当前 `MarketStateHead` 默认 `pred_horizon=5`（5×1h），交易规则却消费：

```text
TrendSegment 腿：min_leg_bars=12，典型 FAST_* 20~40 bar，SLOW_UP 48~96 bar
TrendEntryQualifier：要求 confirmed leg + 放宽后 edge/prob
```

模型优化的是 **未来 5 bar 方向/收益**，规则需要的是 **腿内前 1/3~2/3 段的可参与性与方向一致性**。5 bar 信号在腿中段往往已衰减或反转，导致「趋势模块说 UP、模型说 flat/弱多」。

### 4.2 训练目标与业务 KPI 未对齐（P0）

`market_state_loss` 主权重为 step return + 3-class direction（008/0062e 配方）。023 业务 KPI 为：

```text
leg_count_coverage_ratio、leg_pnl_capture_ratio、aligned overlap
```

训练阶段 **未出现「腿」或「参与」概念**，valid 上 cum_return_ic 可达标，但 **confirmed leg 空仓 bar 的 edge 分位数不抬升**。

### 4.3 方向头未条件于趋势上下文（P1）

推理时 `edge = p_up - p_down` 为 **全局 3-class softmax**，未区分 CHOP / RANGE vs TREND leg。023 Phase 3 在规则层用 regime 乘子补救，已证伪「标准开仓也放宽」路径；需在 **模型输出层** 提供 **条件参与分数**，且 **仅 TEQ 通道消费**（评审 B3）。

### 4.4 监督信号与 017 best-point 未形成闭环（P1）

017 best-point 在 observe 回测中总收益较好，但与 **腿覆盖** 未联合验收。024 主路径不依赖 best-point；仅当 Phase 4 模型轨不达标时条件触发融合（评审 H7）。

### 4.5 008 结构性 trade-off 未在 023 中解决（P2）

0062e：`cum_return_ic` 高、`return_ic` 低。crash 与累计方向受益，**腿内逐步参与**依赖 step 方向质量。024 需在 **不丢弃 cum 分支** 前提下，增加 **腿对齐 step 监督**。

### 6.2 WP1：腿对齐离线标签（P0，1 周）

**交付**：`examples/build_leg_participation_labels.py` → `data/labels/leg_participation_{split}.parquet`

#### 6.2.1 每 bar 字段（最小集）

| 字段 | 说明 |
|------|------|
| `leg_id` / `leg_type` | 来自 segment |
| `is_leg_confirmed` | 腿是否确认 |
| `align_direction` | UP / DOWN / NONE（与 `eval_participation.py` 一致） |
| `leg_progress_ratio` | 0~1，`(bar_idx - leg_start) / leg_len` |
| `ideal_participate_long` | 见 §6.2.3 |
| `ideal_participate_short` | 见 §6.2.3 |
| `forward_leg_roi_12/24/48` | 腿内多视野 ROI |

#### 6.2.2 TEQ 腿类型对齐（评审 H2）

当前 `TrendEntryQualifier._SHORT_LEG_TYPES = {FAST_DOWN_LEG}`，**不含** `SLOW_DOWN_LEG`。

**v1.1 裁定（标签侧对齐 TEQ，不扩 Qualifier）**：

| 方向 | 正样本 `leg_type` 集合 |
|------|------------------------|
| long | `FAST_UP_LEG`, `SLOW_UP_LEG`（与 TEQ long 资格一致） |
| short | **`FAST_DOWN_LEG` only** |

`SLOW_DOWN_LEG` 上 `ideal_participate_short=0`（或不入 short 正样本集），避免「标签有、规则永不消费」。

#### 6.2.3 `ideal_participate_*` 操作定义（评审 B1）

**成本与风控常量**（与回测一致，写入 label 元数据）：

```text
fee_bps = 4.0, slippage_bps = 2.0（来自 trading_rule execution）
round_trip_cost = 2 * (fee_bps + slippage_bps) / 10000
mae_atr_limit = 1.5   # 持仓期间最大不利波动 ≤ 1.5 ATR
```

**候选 entry bar**（与验收 `effective_leg_covered` 可对照）：

```text
eligible_entry_bar t 当且仅当：
  is_leg_confirmed[t] == true
  align_direction[t] in {UP, DOWN}
  leg_progress_ratio[t] <= 0.35        # 腿前 1/3 窗口
  leg_type not in {EXHAUSTION, LEG_END 对应类型}
  chop_level[t] != CHOP_HARD           # 与 022 一致
```

**ideal_participate_long[t]**（hindsight，仅训练用）：

```text
设 entry_price = close[t], 持有至 leg_end_bar 或首个 leg 结束信号 bar
forward_net_roi = (close[exit] - entry_price) / entry_price - round_trip_cost
mae = max_{s in [t, exit]} (entry_price - low[s]) / atr[t]

ideal_participate_long[t] = 1 当且仅当 eligible_entry_bar(t)
  AND align_direction[t] == UP
  AND forward_net_roi > 0
  AND mae <= mae_atr_limit
否则 0
```

**ideal_participate_short[t]**：对称，用 `high[s]` 计算 MAE，`forward_net_roi < 0` 为盈利方向；且 `leg_type == FAST_DOWN_LEG`。

**与 §023.5.3 对照**：若 bar `t` 上 `ideal_participate_long=1`，则该 leg 在 `effective_leg_covered` 语义下 **应可被** 合理参与覆盖（不要求回测一定开仓，但参与潜力成立）。

#### 6.2.4 标签元数据（评审 H4）

parquet 侧车 `metadata.json` 必填：

```json
{
  "segment_module_version": "022_p10_ir7",
  "trend_signal_config_sha256": "<hash>",
  "rule_config_sha256": "<phase1c hash>",
  "split_windows": {"train": [...], "valid": [...], "test": [...]},
  "teacher_label_version": "leg_participation_v024_v1",
  "aligned_leg_types_long": ["FAST_UP_LEG", "SLOW_UP_LEG"],
  "aligned_leg_types_short": ["FAST_DOWN_LEG"]
}
```

Phase 0 出口：标签 hash 与 `eval_participation.py` 在同一 split 上 leg 计数偏差 **< 2%**。

### 6.4 WP3：参与分数推理与校准（P1，1~2 周）

### 6.5 WP4：模型级 + 全链路验收（P0，1 周）

#### 6.5.1 模型级 KPI 公式冻结（评审 B2）

**脚本**：`examples/eval_model_participation.py`

**输出 schema**（`metrics.json`）：

| 字段 | 计算公式 |
|------|----------|

## 13. 结论
