## 3. 初审阻塞项复审

### B1：`ideal_participate_*` 操作定义不完整

**复审状态：已关闭。**

v1.1 §6.2.3 已冻结：

| 要素 | 内容 |
|------|------|
| 成本 | `fee_bps=4`、`slippage_bps=2`，`round_trip_cost` 公式 |
| MAE | `mae_atr_limit=1.5`，long 用 `low[s]`、short 用 `high[s]` |
| Entry 窗口 | `leg_progress_ratio ≤ 0.35`，confirmed + align_direction |
| 腿类型 | long：`FAST_UP`/`SLOW_UP`；short：**仅 `FAST_DOWN`**（§6.2.2） |
| 与 023 对照 | 明确 ideal=1 与 `effective_leg_covered` 语义可对齐 |

**复审意见：通过。**

**实施提示（非阻塞）**：short 侧「`forward_net_roi < 0` 为盈利方向」沿用 long 同公式时语义正确，但建议在 `build_leg_participation_labels.py` 注释中写清 **同一 ROI 定义下 short 理想条件为 ROI 足够负**，避免实现者另写 `(entry-exit)/entry` 双口径。

**复审状态：基本关闭，Phase 2 前需补 1 项公式。**

§6.5.1 已给出 `eval_model_participation.py` 输出 schema 及与 `eval_participation.py` 对照表，满足初审「模型轨先于全链路」要求。

**仍缺公式**（初审 §5.3 亦提及）：

| 指标 | 状态 |
|------|------|
| `precision@teq_open` | §5.1 / §8 Phase 2 **门槛已引用**，§6.5.1 **未定义** |
| `teq_open_on_counter_leg_count` | §6.5.3 有上限，**未入 schema** |

**复审意见：不阻塞 Phase 0；须在 Phase 2 校准接线 PR 前写入 §6.5.1 或 `eval_model_participation.py` docstring。**

建议冻结为：

```text
precision@teq_open =

teq_open_on_counter_leg_count =
  teq 开仓且 align_direction 与开仓方向相反的 leg 数（全链路，eval_participation 扩展）
```

## 8. 实施前澄清项

以下 3 项 **不构成立项阻塞**，建议在 Phase 0 PR 或 Phase 2 前写入实现：

### C1：`precision@teq_open` 与 `teq_open_on_counter_leg_count` 公式

见 §3 B2。Phase 2 门槛已引用，须在 `eval_model_participation.py` 或 `eval_participation.py` 扩展中冻结，避免校准验收口径漂移。

### C2：模型轨 `portfolio_flat` 定义

§6.5.1 中 `confirmed_leg_flat_edge_p50_*` 使用 `portfolio_flat` 子集。模型级评估 **无真实持仓**，应明确为：

```text
portfolio_flat ≡ 标签推理上下文下无模拟持仓
（或：全 bar 集合上的 edge 分位数，与回测 flat 子集对照验收）
```

建议在 `eval_model_participation.py` 设计说明中二选一并固定。

### C3：Phase 0 冻结报告为训练前置硬门槛

`REPORT_023_RULE_STACK_FROZEN.md` 须作为 **Phase 0 第一个合并 PR** 或与其同 PR 交付；`run_v023_phase1c.py`、`diagnose_v023_teq_ceiling.py` 产出入库后，024 全文引用数字方可审计。
