## 5. 软件工程师修改方向

本轮软件工程师只做训练流程小改，不改主干结构。

### 5.1 增加 horizon-specific return loss

当前 step `return_ic` 低，且历史多轮显示 h1/h2/h3 经常偏弱。建议在 `market_state_loss()` 中增加逐 horizon 权重：

```text
return_horizon_weights = [w1, w2, w3, w4, w5]
```

默认：

```text
[1.4, 1.3, 1.2, 1.0, 1.0]
```

实现要求：

```python
step_loss = huber(output.return_pred, target.future_log_ret, reduction="none")
step_loss = step_loss * horizon_weights
ret_loss = step_loss.mean()
```

新增 CLI 参数：

```text
--return-horizon-weights 1.4 1.3 1.2 1.0 1.0
```

如果未传，保持旧行为：

```text
全部 horizon 权重 = 1.0
```

### 5.2 降低 return consistency 对 step return 的牵制

0063 的 `cum_return_ic` 已经很强，但 step `return_ic` 弱。说明 step return 可能被累计目标牵制。

建议本轮：

```text
return_consistency_weight = 0.00 或 0.005
```

软件工程师只需确保 CLI 支持传入 `0.0`，报告中写清实际值。

### 5.3 保留 cum return 分支

不要删除：

```text
use_cum_heads=True
use_horizon_return_head=True
cum_return_weight > 0
```

`cum_return_ic` 是当前新结构的核心成果，0064 不能以牺牲它为代价追 step return。

### 5.4 选模 score 加强 step return_ic，但设置稳定性硬约束

建议新增 `target_stage=step_return_recovery`，或在当前 `return_direction_branch` 下增加 score 选项。

建议 score：

```text
score_0064 =
  0.28 * clipped_return_ic
  + 0.20 * clipped_cum_return_ic
  + 0.16 * cum_direction_from_return_acc
  + 0.14 * direction_macro_f1
  + 0.10 * risk_f1
  - 0.05 * volatility_mae
  + 0.07 * distribution_health
```

其中：

```text
clipped_return_ic = clip(return_ic / 0.04, -1, 1)
clipped_cum_return_ic = clip(cum_return_ic / 0.12, -1, 1)
```

hard gates：

```text
return_ic > 0.015
cum_return_ic >= 0.09
cum_direction_from_return_acc >= 56%
direction_macro_f1 >= 0.31
risk_f1 >= 0.50
collapse_gates 全通过
```

注意：

```text
没有 eligible checkpoint 时，不允许把 last_epoch 当作 best。
```

### 5.5 报告增加 step return 改善字段

报告必须包含：

```text
return_ic_h1...h5
return_horizon_weights
return_consistency_weight
cum_return_ic
step_cum_return_gap_mae
是否通过 collapse gates
是否通过 0064 hard gates
```

## 6. 训练师执行步骤（新手版）

### 6.1 第一步：确认基线

先确认你不是从 0058 或 0061 开始。

本轮基线应是：

```text
checkpoints/0062c_market_state_cum_return_stabilized/market_state_best.pt
```

或者项目经理确认的：

```text
0062e / 0063_s42 等价基线
```

如果 checkpoint 不存在，不要自己换路径，先停止并反馈。

### 6.2 第二步：先跑 0064a 主实验

目标：

```text
轻微提高 h1/h2/h3 的 step return 权重，
同时保留 cum_return_ic。
```

建议命令：

```bash
python examples/train_market_state_model.py \
  --source binance_vision --symbol BTCUSDT --interval 1h --days 365 \
  --target-stage return_direction_branch \
  --epochs 30 --batch-size 64 --device cpu \
  --direction-threshold-quantile 0.25 \
  --risk-threshold-quantile 0.70 \
  --return-weight 0.48 \
  --direction-weight 0.20 \
  --volatility-weight 0.10 \
  --risk-weight 0.09 \
  --cum-return-weight 0.12 \
  --cum-direction-head-weight 0.03 \
  --return-consistency-weight 0.005 \
  --cum-direction-weight 0.00 \
  --return-horizon-weights 1.4 1.3 1.2 1.0 1.0 \
  --use-cum-heads \
  --use-horizon-return-head \
  --use-class-weights \
  --no-use-direction-class-weights \
  --use-risk-class-weights \
  --no-balanced-class-weights \
  --init-market-checkpoint checkpoints/0062c_market_state_cum_return_stabilized/market_state_best.pt \
  --checkpoint-dir checkpoints/0064a_market_state_step_return_recovery \
  --report-dir reports/0064a_market_state_step_return_recovery
```

如果脚本报不认识 `--return-horizon-weights`，说明软件工程师还没完成 5.1，先不要跑，反馈给软件工程师。

### 6.3 第三步：看 0064a 是否达标

训练完成后打开：

```text
reports/0064a_market_state_step_return_recovery/REPORT.md
reports/0064a_market_state_step_return_recovery/metrics.json
```

先看这几个数：

```text
return_ic
cum_return_ic
cum_direction_from_return_acc
direction_macro_f1
risk_f1
volatility_mae
collapse gates
```

判断：

```text
如果 return_ic >= 0.020 且 cum_return_ic >= 0.100，继续看其他指标。
如果 return_ic < 0.015，0064a 失败。
如果 collapse gates 未通过，0064a 失败。
```

### 6.4 第四步：根据结果选择 0064b 或停止

#### 情况 A：return_ic 提升，cum_return_ic 保住

表现：

```text
return_ic >= 0.020
cum_return_ic >= 0.100
collapse gates 通过
```

动作：

```text
0064a 条件成功；
再用 seed 43 跑 0064b 复现。
```

0064b 只改 seed 和输出目录：

```bash
--seed 43
--checkpoint-dir checkpoints/0064b_market_state_step_return_recovery
--report-dir reports/0064b_market_state_step_return_recovery
```

#### 情况 B：return_ic 没提升，但 cum_return_ic 保住

表现：

```text
return_ic < 0.015
cum_return_ic >= 0.100
collapse gates 通过
```

动作：

```text
不要继续同配置多 seed；
把 return_horizon_weights 提高一点，跑 0064b。
```

0064b 权重：

```text
--return-horizon-weights 1.8 1.5 1.3 1.0 1.0
--return-weight 0.52
--cum-return-weight 0.10
```

#### 情况 C：return_ic 提升，但 cum_return_ic 掉了

表现：

```text
return_ic >= 0.020
cum_return_ic < 0.090
```

动作：

```text
不要接受；
说明 step return 抢走了 cum return 能力。
恢复 cum_return_weight 到 0.14，再跑。
```

#### 情况 D：出现坍缩

表现：

```text
direction_pred_flat < 8%
或 risk_positive_rate_pred < 5%
或 direction_pred_down > 60%
```

动作：

```text
立即停止该方向；
不要继续调 return；
回到 0063 稳定配置。
```

## 10. 架构师判断口径

架构师会按以下优先级判断 0064：

1. 是否无坍缩；
2. `cum_return_ic` 是否仍 >= 0.10；
3. `cum_direction_from_return_acc` 是否仍 >= 58%；
4. `return_ic` 是否从约 0.006 提升到 >= 0.020；
5. risk / macro_f1 / volatility 是否仍在可接受区间。

如果 0064 能把 `return_ic` 提升到 0.020 以上，同时保住其他指标，则说明下一阶段值得继续做收益恢复。

如果 0064 无论如何都无法提升 `return_ic`，则接受当前分支定位：

```text
新结构分支主要增强累计收益 / 累计方向，
不再强求 step return 排序。
```

## 11. 一句话结论

下一轮不改大结构，只做训练流程小改：

```text
给 h1-h3 step return 更高权重，
降低 return consistency，
保护 cum_return_ic 和坍缩 gates，
用 0064a 单轮验证是否能恢复 step return_ic。
```

新手训练师只需记住：

```text
先跑 0064a；
看 return_ic 有没有到 0.020；
看 cum_return_ic 有没有保住 0.100；
看有没有坍缩；
再决定是否跑 0064b/0064c。
```
