# 顺逆势分层抑制规则设计

## 1. 文档说明

| 项 | 内容 |

| 目标读者 | 程序员、策略设计师、回测师 |

本文补充 021 期趋势优先交易框架中的关键细节：**顺势与逆势不是绝对概念，而应按趋势层级、趋势一致性和当前子阶段做分层抑制**。

本文的重点不是简单回答“能不能逆势做”，而是定义：

```text
逆势可以被允许到什么程度，
需要多高门槛，
能用多大仓位，
能持有多久，
是否允许加仓。
```

---

## 2. 问题背景

当前系统已经有逆势阻断逻辑，例如：

- 下跌腿中阻止逆势多头
- 上涨腿中阻止逆势空头

这种逻辑在强共振趋势中是必要的，但它隐含了一个过于粗糙的假设：

```text
只要大趋势向上，做空就是逆势；
只要大趋势向下，做多就是逆势。
```

这个假设在真实市场中并不总成立。

例如：

1. 大趋势向上，但中趋势处于确认中的下跌回撤段
2. 大趋势向下，但小趋势出现强力反弹脉冲
3. 大趋势仍向上，但已经进入衰竭区，继续追多反而风险更高

因此，021 期必须把“逆向抑制”从**二元开关**升级为**分层控制**。

---

## 3. 核心原则

### 3.1 顺势 / 逆势必须带时间尺度

建议至少区分三层：

| 层级 | 含义 | 典型时间尺度 |
|------|------|--------------|
| 大趋势（Macro / Regime） | 宏观偏多、偏空、中性背景 | 数天到数周 |
| 中趋势（Trend Leg） | 当前主交易腿方向 | 数小时到数天 |
| 小趋势（Sub-leg / Micro） | 当前腿内推进、回撤、反弹、整理 | 数根到数十根 K |

因此，顺势 / 逆势不应写成一个布尔值，而应写成：

```text
相对哪一层趋势是顺势？
相对哪一层趋势是逆势？
```

### 3.2 真正需要抑制的是“低质量逆向”

不是所有逆势交易都应该禁止。

更合理的目标是：

- 禁止 **强共振背景下的重逆势**
- 允许 **高质量、短持有、低仓位的逆向回撤交易**
- 允许 **中趋势已确认反向时的谨慎逆向**

### 3.3 逆势控制不只影响“开不开”

逆势抑制必须同时作用于：

1. 开仓阈值
2. 初始仓位
3. 持仓时长
4. 加仓权限
5. 退出速度

---

## 4. 趋势一致性定义

建议引入一个统一的“趋势一致性”概念，用于表达不同层级趋势是否共振。

### 4.1 建议字段

```text
macro_direction
leg_direction
micro_direction
```

每一层取值：

- `UP`
- `DOWN`
- `NONE`

### 4.2 一致性分数（按方向拆分）

不再使用单一 `alignment_score`，而是分别计算：

```text
alignment_score_long  ∈ {-2, -1, 0, +1, +2}   # 做多视角
alignment_score_short ∈ {-2, -1, 0, +1, +2}   # 做空视角
```

计算公式与权重见 `软件设计师-021-偏置数学公式说明.md` §9。`macro_direction == NONE` 时 macro 层贡献 0 分。

### 4.3 逆向级别

建议基于一致性分数映射为更易读的业务等级：

| 级别 | 含义 |
|------|------|
| `NONE` | 非逆势或高度顺势 |
| `LIGHT` | 轻度逆势，可谨慎参与 |
| `MEDIUM` | 中度逆势，应明显收紧 |
| `HARD_BLOCK` | 强逆势，应禁止或近似禁止 |

---

## 5. 分层抑制矩阵

### 5.1 基础矩阵

| 大趋势 | 中趋势 | 小趋势 | 逆向级别 | 交易建议 |
|--------|--------|--------|----------|----------|
| 同向 | 同向 | 同向 | `NONE` | 放宽开仓、允许加仓、延长持仓 |
| 同向 | 同向 | 反向 | `LIGHT` | 允许小仓位逆向短打，禁止加仓 |
| 同向 | 反向 | 同向 | `MEDIUM` | 可能是趋势切换早期，谨慎，缩小仓位 |
| 同向 | 反向 | 反向 | `MEDIUM` | 可做中短逆向，但更快退出 |
| 反向 | 反向 | 反向 | `HARD_BLOCK` | 禁止重仓逆势，必要时直接 block |

### 5.2 特别说明

这个矩阵不是机械地看“向上 / 向下”，而是看：

```text
当前想开的仓位方向，与三层趋势关系如何。
```

例如：

- 大趋势 `UP`
- 中趋势 `DOWN`
- 小趋势 `DOWN_ACCELERATION`

此时做空虽然“逆大趋势”，但不一定应视为 `HARD_BLOCK`，更可能是 `MEDIUM`。

## 6. 分层抑制对交易五要素的影响

## 6.1 开仓阈值

### `NONE`

- 放宽同向开仓门槛
- 允许较弱但干净的顺势信号入场

### `LIGHT`

- 逆向单需要更高 `edge`
- `pred_cum_ret_5` 需要更强
- `p_risk` 必须更低

### `MEDIUM`

- 开仓阈值进一步提高
- 只允许高质量逆向 setup
- 不满足则观望

### `HARD_BLOCK`

- 直接阻断
- 或仅保留极小试探仓特权（若配置允许）

---

## 6.2 初始仓位

建议把抑制级别映射成仓位系数：

| 级别 | 建议仓位系数 |
|------|--------------|
| `NONE` | `1.0x ~ 1.5x` |
| `LIGHT` | `0.4x ~ 0.7x` |
| `MEDIUM` | `0.2x ~ 0.5x` |
| `HARD_BLOCK` | `0x` 或 `0.1x` 探测仓 |

注意：

- 这只是对基础仓位的系数，不改变总风控边界
- 固定 20 倍杠杆、最大 20% 仓位、最大损失约束仍然有效

---

## 6.3 持仓时长

### `NONE`

- 可延长最小持仓期
- pullback 不视为立即失败
- runner 模式更宽容

### `LIGHT`

- 允许短持有
- 但不鼓励长期持仓

### `MEDIUM`

- 明显缩短持有窗口
- 达到小利润优先保护

### `HARD_BLOCK`

- 原则上不进入持仓
- 若极小试探仓进入，也应极短持有

---

## 6.4 加仓权限

### `NONE`

- 若趋势阶段为 `CONTINUATION` / `ACCELERATION`
- 且风险预算允许
- 可正常考虑加仓

### `LIGHT`

- 默认禁止加仓
- 仅在趋势重新回归顺势后恢复

### `MEDIUM`

- 明确禁止加仓

### `HARD_BLOCK`

- 无开仓，更无加仓

---

## 6.5 退出门槛

### `NONE`

- 需要更多退出票
- 允许正常 pullback

### `LIGHT`

- 少量不利信号即可减仓

### `MEDIUM`

- 退出确认门槛显著降低
- 模型累计收益一旦翻反向，应更快处理

### `HARD_BLOCK`

- 强不变量：`allow_open=false`、`open_bias=0`、`size_bias=0`、`allow_add=false`（见字段契约 §5.1）
- 若有遗留逆势仓，应优先退出

## 7. 建议写入 `TrendBiasContext` 的字段

以 `软件设计师-021-TrendBiasContext最终字段契约.md` 为准。与本文件直接相关的核心字段：

```python
macro_direction / leg_direction / micro_direction
alignment_score_long / alignment_score_short
counter_level_long / counter_level_short   # 非 counter_trend_level

open_bias_long / open_bias_short
size_bias_long / size_bias_short
hold_bias_long / hold_bias_short
exit_bias_long / exit_bias_short

allow_open_long / allow_open_short
allow_add_long / allow_add_short
# 逆向禁止由 counter_level_* == HARD_BLOCK 或 allow_open_* == false 表达
```

这些字段把“趋势是否冲突”变成可直接消费的交易控制变量。

## 8. 对 021 主文档思路的补充修正

本文件对 021 的核心修正是：

### 8.1 修正一：逆势不等于统一收紧

更准确的表达应为：

```text
逆势交易需要按趋势层级和一致性分层收紧；
并非所有逆势都应一刀切禁止。
```

### 8.2 修正二：顺势不等于一律放宽

若大趋势向上但当前已处于：

- `EXHAUSTION`
- `REVERSAL_RISK`
- `LEG_END`

则即便是“顺大趋势多头”，也不应继续机械放宽。

### 8.3 修正三：大趋势、中趋势、小趋势应分工

建议职责：

- 大趋势：决定背景偏置
- 中趋势：决定主交易方向与持仓模式
- 小趋势：决定入场时机与微观退出

## 10. 结论

021 要做的不只是“统一趋势偏置层”，还应进一步完成：

```text
把顺势 / 逆势从布尔判断，
升级为带层级、带强弱、带风险预算的分层控制体系。
```

只有这样，系统才能既避免：

- 在强趋势中逆势硬扛

又避免：

- 错杀高质量的中短逆向机会。

这将使 021 的趋势优先框架更接近真实交易中的“多层级趋势驱动”系统。
