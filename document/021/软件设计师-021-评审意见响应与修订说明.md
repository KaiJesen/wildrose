## 5. 实施顺序（与架构师 §5 对齐）

```text
Phase 0  契约冻结（本文档族 + 配置 schema）— 不改 rules.py 主流程
Phase A  observe：engine 生成 bias，logger 记录，rules 不消费
Phase B  open_only：接管 allow_open + open_bias；disable_legacy_trend_rules
Phase C  open_size：size_bias + RiskBudget
Phase D  full：add/hold/exit/force_exit；清理 RuleEngine 直接读趋势原始字段
```

## 7. Phase 0.1 复审修订（架构师立项材料复审）

复审报告结论：**阻塞项修正后可进入 Phase 0 代码实现**，无需再次大评审。

| # | 复审问题 | 修订 |
|---|----------|------|
| 1 | `exit_bias` 公式与语义相反 | 改为 `ceil(base_exit_votes * exit_bias_side)` |
| 2 | `RiskBudget` 仅 `allow_add` | 扩展 `allow_open_*`、`allow_add_*`、`allow_reverse` + reject reason |
| 3 | P0 与 Builder 边界冲突 | P0 移出 Builder；新增 `RiskEvent` + `RuleEngineActionResolver` |
| 4 | `HARD_BLOCK` “或”语义 | 强不变量四字段同时置位；试探仓独立配置 |
| 5 | 旧字段别名残留 | 模块改造清单、logger 对齐冻结 schema |
| 6 | 6 月样本年份不明 | 改为“6 月初持续下跌样本（以回测报告窗口为准）” |
| 7 | `trend_add_rejected > 0` 无条件 gate | 拆为 candidate / evaluated / rejected 条件型验收 |
| 8 | `source_confidence` 无定义 | 字段契约补充四档计算规则 |

**当前状态：Phase 0.1 修订已完成，可进入 Phase 0 实现。**
