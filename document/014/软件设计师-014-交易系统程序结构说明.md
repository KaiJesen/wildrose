## 2. 总体实现目标

014 阶段不直接把旧回测脚本继续堆大，而是抽象成一个独立交易系统模块，使它能同时服务：

```text
历史回测
纸面仿真
未来实盘执行适配
```

核心原则：

```text
模型只输出信号；
交易系统负责状态、仓位、风控、执行和记录；
所有行为必须可解释、可配置、可复盘。
```

## 4. 核心数据对象

### 4.1 标准交易信号 `TradingSignal`

落点：

```text
trading_system/signal.py
```

字段：

```text
ts
price
atr
p_up
p_down
p_flat
p_risk
pred_ret_1
pred_ret_2
pred_ret_3
pred_ret_4
pred_ret_5
pred_cum_ret_5
edge
conf
cum_edge

source

```

实现规则：

```text
edge = p_up - p_down
conf = abs(edge)
cum_edge = pred_cum_ret_5
risk_ok = p_risk <= config.risk.open_risk_max
```

异常拦截：

```text
缺失关键字段 -> INVALID_SIGNAL
概率不是有限数 -> INVALID_SIGNAL
p_up/p_down/p_flat 不在 [0, 1] -> INVALID_SIGNAL
price <= 0 -> INVALID_SIGNAL
atr <= 0 -> INVALID_SIGNAL
pred_cum_ret_5 缺失时，允许由 pred_ret_1..5 求和兜底，并记录 fallback_reason
```

## 5. 配置设计

落点：

```text
trading_system/config.py
configs/trading_rule_v014_conservative.json
configs/trading_rule_v014_aggressive.json
```

配置应集中保存，不允许在多个模块散落硬编码。

### 5.1 基础约束配置

```text
max_position_ratio = 0.20
fixed_leverage = 20.0
max_add_count = 2
decision_interval = "1h"
execute_timing = "next_bar_open"
max_margin_loss_ratio = 1.00
catastrophe_margin_loss_buffer = 0.90
```

解释：

```text
max_position_ratio 表示单方向总目标仓位不超过当前权益 20%；
fixed_leverage 固定为 20 倍；
catastrophe_margin_loss_buffer=0.90 表示保证金亏损达到 90% 时提前强平，避免等到 100%。
```

### 5.2 稳健版默认阈值

参考 v012：

```text
open_edge_threshold = 0.08
open_prob_threshold = 0.42
open_flat_max = 0.34
risk_open_max = 0.38
risk_exit_threshold = 0.48
long_continue_edge_min = -0.03
short_continue_edge_max = 0.03
reverse_edge_threshold = 0.05
max_hold_bars = 6
stop_atr_mult = 1.2
tp1_atr_mult = 1.0
tp2_atr_mult = 2.0
trail_atr_mult = 0.8
day_drawdown_stop = 0.02
week_drawdown_defensive = 0.05
loss_streak_limit = 3
cooldown_bars = 12
```

### 5.3 进攻版配置

参考 v013，但必须受 014 总约束限制：

```text
open_edge_threshold = 0.05
open_prob_threshold = 0.39
open_flat_max = 0.40
risk_open_max = 0.45
risk_exit_threshold = 0.55
reverse_edge_threshold = 0.04
max_hold_bars = 8
stop_atr_mult = 1.4
tp1_atr_mult = 1.2
tp2_atr_mult = 2.5
trail_atr_mult = 1.0
day_drawdown_stop = 0.03
week_drawdown_defensive = 0.07
loss_streak_limit = 4
cooldown_bars = 10
```

注意：

```text
即使使用进攻版，max_position_ratio 仍不得超过 20%，fixed_leverage 仍固定 20 倍。
```

## 6. 模块职责

### 6.1 `SignalProvider`

落点：

```text
trading_system/adapters/market_state_model.py
trading_system/adapters/csv_signal.py
```

职责：

```text
读取模型推理输出；
映射为 TradingSignal；
补充 price / atr；
保证输出不依赖训练脚本内部对象。
```

实现路径：

1. 先把 `examples/backtest_market_state_rule_v012.py` 中 `collect_signals()` 的模型推理逻辑迁移到 `MarketStateSignalProvider`。
2. 再实现 `CsvSignalProvider`，用于读取离线信号文件，便于回测与排查。
3. 所有 provider 都只返回 `TradingSignal`，不得直接参与交易决策。

### 6.2 `RuleEngine`

落点：

```text
trading_system/rules.py
```

职责：

```text
判断开仓、维持、加仓、减仓、平仓；
生成目标动作 TradingAction；
输出每个动作的 reason_code 与诊断字段。
```

输出动作：

```text
OPEN_LONG
OPEN_SHORT
ADD
REDUCE
CLOSE
REVERSE
HOLD
BLOCK
```

### 6.3 `RiskManager`

落点：

```text
trading_system/risk.py
```

职责：

```text
检查账户级熔断；
检查硬止损、灾难止损；
检查风险概率退出；
检查保证金损失边界；
对 RuleEngine 的目标动作做二次拦截。
```

原则：

```text
RiskManager 优先级高于 RuleEngine。
任何风控强制退出或禁止开仓，RuleEngine 不得覆盖。
```

### 6.4 `PositionSizer`

落点：

```text
trading_system/sizing.py
```

职责：

```text
计算初始开仓比例；
计算加仓比例；
计算减仓后目标比例；
计算 20 倍杠杆下保证金占用；
保证总仓位不超过 20%。
```

仓位输出统一使用：

```text
position_ratio
notional_exposure
margin_required
```

### 6.5 `ExecutionEngine`

落点：

```text
trading_system/execution.py
```

职责：

```text
按下一根 K 线开盘价执行；
模拟手续费；
模拟滑点；
反手时先平后开；
返回成交结果 FillEvent。
```

回测阶段实现：

```text
BacktestExecutionEngine
```

未来实盘阶段可新增：

```text
ExchangeExecutionEngine
```

### 6.6 `TradingEngine`

落点：

```text
trading_system/engine.py
```

职责：

```text
编排完整决策流程；
维护 PortfolioState；
按固定优先级调用各模块；
把每根 K 线的信号、动作、成交、状态写入日志。
```

### 6.7 `TradeLogger`

落点：

```text
trading_system/logger.py
```

职责：

```text
记录逐 bar 决策日志；
记录逐笔交易；
记录加仓/减仓事件；
记录权益曲线；
记录所有 BLOCK / CLOSE / REDUCE 的触发原因。
```

输出文件：

```text
decisions.csv
orders.csv
fills.csv
trades.csv
equity_curve.csv
metrics.json
REPORT.md
```

## 7. 主流程

落点：

```text
trading_system/engine.py
```

每根 K 线收盘后生成信号，在下一根 K 线开盘执行。

伪代码：

```python
def on_bar_close(signal: TradingSignal, current_bar: Bar, next_bar: Bar) -> None:
    if not signal.is_valid:
        action = TradingAction.block("INVALID_SIGNAL")
        logger.record_decision(signal, action, portfolio)
        return

    portfolio.mark_to_market(current_bar.close)
    risk_event = risk_manager.check_pre_decision_risk(signal, portfolio, current_bar)

    if risk_event.force_close:
        action = TradingAction.close(risk_event.reason)
    elif risk_event.block_open and portfolio.is_flat:
        action = TradingAction.block(risk_event.reason)
    else:
        action = rule_engine.decide(signal, portfolio, current_bar)

    action = risk_manager.validate_action(action, signal, portfolio, current_bar)
    sized_action = position_sizer.apply(action, signal, portfolio, current_bar)
    fill = execution_engine.execute(sized_action, next_bar.open)
    portfolio.apply_fill(fill)
    logger.record_all(signal, sized_action, fill, portfolio)
```

严格执行优先级：

```text
1. 信号合法性检查
2. 账户级熔断状态检查
3. 持仓浮盈浮亏更新
4. 硬止损 / 灾难止损
5. 追踪止盈
6. 风险概率退出
7. 时间退出
8. 反向信号退出
9. 持仓状态下判断减仓 / 加仓 / 继续持有
10. 空仓状态下判断新开仓
11. 执行成交
12. 写入日志与权益记录
```

## 8. 行为触发规则

### 8.1 禁止新开仓触发

任一条件满足，触发 `BLOCK_OPEN`：

```text
signal invalid
portfolio.account_circuit_breaker == true
portfolio.daily_open_block == true
current_bar_idx < portfolio.cooldown_until
p_risk > risk_open_max
p_flat > open_flat_max
weekly_defensive_mode 且配置禁止新开仓
交易所异常或执行质量异常
```

记录字段：

```text
reason_code = BLOCK_OPEN_*
blocked_by = signal / risk / account / execution
```

### 8.2 多头开仓触发

状态要求：

```text
position.side == FLAT
```

全部满足时触发 `OPEN_LONG`：

```text
edge >= open_edge_threshold
p_up >= open_prob_threshold
p_flat <= open_flat_max
pred_cum_ret_5 > 0
p_risk <= risk_open_max
not in cooldown
not daily_open_block
not account_circuit_breaker
```

动作：

```text
target_side = LONG
position_ratio = PositionSizer.initial_size(signal)
position_ratio <= 0.20
leverage = 20
```

原因：

```text
reason_code = OPEN_LONG_SIGNAL
```

### 8.3 空头开仓触发

状态要求：

```text
position.side == FLAT
```

全部满足时触发 `OPEN_SHORT`：

```text
edge <= -open_edge_threshold
p_down >= open_prob_threshold
p_flat <= open_flat_max
pred_cum_ret_5 < 0
p_risk <= risk_open_max
not in cooldown
not daily_open_block
not account_circuit_breaker
```

动作：

```text
target_side = SHORT
position_ratio = PositionSizer.initial_size(signal)
position_ratio <= 0.20
leverage = 20
```

原因：

```text
reason_code = OPEN_SHORT_SIGNAL
```

### 8.4 初始仓位触发规则

`PositionSizer.initial_size()` 按信号强度分档：

```text
conf < 0.08                 -> weak, 5%~8%
0.08 <= conf < 0.14         -> medium, 8%~12%
0.14 <= conf < 0.20         -> strong, 12%~15%
conf >= 0.20 且 p_risk低    -> very_strong, 15%~20%
```

风险折扣：

```text
risk_penalty = clip((risk_exit_threshold - p_risk) / (risk_exit_threshold - risk_open_max), 0, 1)
volatility_penalty = clip(config.reference_atr_ratio / atr_ratio, 0.5, 1.0)
final_ratio = base_ratio * risk_penalty * volatility_penalty
```

硬约束：

```text
final_ratio = min(final_ratio, max_position_ratio)
final_ratio > 0
```

### 8.5 加仓触发

状态要求：

```text
position.side in [LONG, SHORT]
position.add_count < max_add_count
position.position_ratio < max_position_ratio
```

多头全部满足时触发 `ADD_LONG`：

```text
pred_cum_ret_5 > 0
edge >= entry_edge - add_edge_tolerance
p_risk <= add_risk_max
unrealized_pnl >= add_profit_atr_mult * atr
加仓后 position_ratio <= 0.20
加仓后最坏损失 <= 当前保证金 100%
```

空头全部满足时触发 `ADD_SHORT`：

```text
pred_cum_ret_5 < 0
edge <= entry_edge + add_edge_tolerance
p_risk <= add_risk_max
unrealized_pnl >= add_profit_atr_mult * atr
加仓后 position_ratio <= 0.20
加仓后最坏损失 <= 当前保证金 100%
```

禁止加仓触发：

```text
p_risk > add_risk_max
pred_cum_ret_5 与持仓方向相反
position_ratio >= 0.20
add_count >= max_add_count
unrealized_pnl < 0 且无强确认信号
daily_open_block == true
account_circuit_breaker == true
```

原因：

```text
reason_code = ADD_SIGNAL_CONFIRMED
或 BLOCK_ADD_*
```

### 8.6 减仓观察触发

持仓状态下，若未触发全平但维持条件部分失效，触发 `REDUCE`：

多头：

```text
edge < long_continue_edge_min
或 pred_cum_ret_5 <= 0
或 p_risk > risk_open_max 但 p_risk < risk_exit_threshold
```

空头：

```text
edge > short_continue_edge_max
或 pred_cum_ret_5 >= 0
或 p_risk > risk_open_max 但 p_risk < risk_exit_threshold
```

动作：

```text
target_position_ratio = current_position_ratio * reduce_scale
默认 reduce_scale = 0.50
continue_fail_count += 1
```

原因：

```text
reason_code = REDUCE_WEAK_CONTINUE_SIGNAL
```

### 8.7 继续持有触发

多头持仓满足：

```text
edge > long_continue_edge_min
pred_cum_ret_5 >= 0
p_risk < risk_exit_threshold
bars_held < max_hold_bars
未触发止损/止盈/熔断
```

空头持仓满足：

```text
edge < short_continue_edge_max
pred_cum_ret_5 <= 0
p_risk < risk_exit_threshold
bars_held < max_hold_bars
未触发止损/止盈/熔断
```

动作：

```text
HOLD
```

原因：

```text
reason_code = HOLD_SIGNAL_VALID
```

### 8.8 反向信号平仓触发

多头任一满足：

```text
edge <= -reverse_edge_threshold
pred_cum_ret_5 < 0
```

空头任一满足：

```text
edge >= reverse_edge_threshold
pred_cum_ret_5 > 0
```

动作：

```text
CLOSE
```

若配置允许反手，并且反向开仓条件也满足：

```text
REVERSE = CLOSE old position first, then OPEN opposite side
```

原因：

```text
reason_code = CLOSE_REVERSE_SIGNAL
reason_code = REVERSE_SIGNAL
```

### 8.9 风险概率退出触发

任一持仓下：

```text
p_risk >= risk_exit_threshold
```

动作由配置决定：

```text
risk_exit_mode = full_close -> CLOSE
risk_exit_mode = reduce_first -> REDUCE 到 defensive_position_ratio
```

若连续两根仍高风险：

```text
force CLOSE
```

原因：

```text
reason_code = CLOSE_RISK_PROB_HIGH
或 REDUCE_RISK_PROB_HIGH
```

### 8.10 时间退出触发

任一持仓下：

```text
bars_held >= max_hold_bars
```

动作：

```text
time_exit_mode = full_close -> CLOSE
time_exit_mode = reduce_then_close -> 先 REDUCE，下一根仍弱则 CLOSE
```

原因：

```text
reason_code = CLOSE_MAX_HOLD_BARS
或 REDUCE_MAX_HOLD_BARS
```

### 8.11 硬止损触发

多头：

```text
current_price <= stop_price
```

空头：

```text
current_price >= stop_price
```

止损线：

```text
LONG stop_price = entry_price - stop_atr_mult * atr_at_entry
SHORT stop_price = entry_price + stop_atr_mult * atr_at_entry
```

动作：

```text
CLOSE
```

原因：

```text
reason_code = CLOSE_HARD_STOP
```

### 8.12 灾难止损触发

任一持仓下：

```text
margin_loss_ratio >= catastrophe_margin_loss_buffer
```

默认：

```text
catastrophe_margin_loss_buffer = 0.90
```

动作：

```text
FORCE_CLOSE
```

原因：

```text
reason_code = CLOSE_CATASTROPHE_MARGIN_LOSS
```

说明：

```text
项目要求最高损失不超过当前保证金 100%；
程序必须提前于 100% 保证金损失强平，不允许等于或超过 100% 后才处理。
```

### 8.13 分批止盈触发

多头：

```text
current_price >= entry_price + tp1_atr_mult * atr 且 tp1_done == false
current_price >= entry_price + tp2_atr_mult * atr 且 tp2_done == false
```

空头：

```text
current_price <= entry_price - tp1_atr_mult * atr 且 tp1_done == false
current_price <= entry_price - tp2_atr_mult * atr 且 tp2_done == false
```

动作：

```text
TP1 -> REDUCE 40%
TP2 -> REDUCE 40%
剩余仓位进入 trailing stop
```

原因：

```text
reason_code = REDUCE_TAKE_PROFIT_1
reason_code = REDUCE_TAKE_PROFIT_2
```

### 8.14 追踪止盈触发

多头：

```text
peak_unrealized_pnl 更新为历史最高浮盈
current_unrealized_pnl <= peak_unrealized_pnl - trail_atr_mult * atr
```

空头同理按有利方向浮盈回撤计算。

动作：

```text
CLOSE 剩余仓位
```

原因：

```text
reason_code = CLOSE_TRAILING_TAKE_PROFIT
```

### 8.15 连续失败退出触发

每根 bar 若维持条件失败：

```text
continue_fail_count += 1
```

若恢复满足维持条件：

```text
continue_fail_count = 0
```

触发：

```text
continue_fail_count >= continue_fail_limit
默认 continue_fail_limit = 2
```

动作：

```text
CLOSE
```

原因：

```text
reason_code = CLOSE_CONTINUE_SIGNAL_FAILED
```

### 8.16 账户级熔断触发

日内停止开新仓：

```text
equity <= day_start_equity * (1 - day_drawdown_stop)
```

动作：

```text
daily_open_block = true
已有持仓继续接受止损/止盈/风控退出
禁止新开仓和加仓
```

周内防守模式：

```text
equity <= week_start_equity * (1 - week_drawdown_defensive)
```

动作：

```text
weekly_defensive_mode = true
新开仓 position_ratio *= defensive_size_scale
默认 defensive_size_scale = 0.30
```

连续亏损冷却：

```text
loss_streak >= loss_streak_limit
```

动作：

```text
cooldown_until = current_bar_idx + cooldown_bars
禁止新开仓和加仓
```

原因：

```text
reason_code = BLOCK_DAILY_DRAWDOWN
reason_code = DEFENSIVE_WEEKLY_DRAWDOWN
reason_code = BLOCK_LOSS_STREAK_COOLDOWN
```

## 9. 执行模块设计

### 9.1 回测执行口径

```text
信号产生：当前 K 线收盘后
执行价格：下一根 K 线开盘
手续费：按成交名义金额 * fee_bps
滑点：按方向对成交价做不利调整
```

成交价：

```text
BUY price = next_open * (1 + slippage_bps / 10000)
SELL price = next_open * (1 - slippage_bps / 10000)
```

反手：

```text
先 CLOSE 当前仓位；
更新权益、手续费、loss_streak；
若仍满足反向开仓和风控条件，再 OPEN 新方向；
两笔成交分别记录。
```

## 15. 结论

014 程序结构应从旧脚本式回测升级为模块化交易系统：

```text
SignalProvider -> TradingSignal
TradingEngine -> RuleEngine + RiskManager + PositionSizer
ExecutionEngine -> FillEvent
PortfolioState -> TradeLogger -> BacktestReport
```

关键实现要求：

```text
先风控后交易；
先平仓后开仓；
仓位不超过 20%；
杠杆固定 20 倍；
保证金损失提前强平；
所有 OPEN / ADD / REDUCE / CLOSE / BLOCK 都必须有 reason_code。
```
