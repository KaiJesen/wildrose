## 1. 摘要

在 **v014 交易系统** 上，以 **0065a**（`cum_return` 分支，seed=45）为信号源，完成进攻模式回测、阈值网格调优及样本外验证。

| 阶段 | 配置 | 区间 | 策略收益 | 基准收益 | 超额 | 最大回撤 | 交易数 | 胜率 | 盈亏比 |

## 3. 基线回测：零成交根因

### 3.1 现象

`configs/trading_rule_v014_aggressive.json` 在 test 集（1308 bars）产生 **0 笔交易**。

### 3.2 信号分布诊断（test 集）

| 字段 | p25 | 中位数 | p75 | 最大值 |
|------|-----|--------|-----|--------|
| edge | -0.014 | 0.006 | 0.020 | 0.074 |
| p_up | 0.301 | 0.312 | 0.322 | **0.352** |
| p_down | 0.298 | 0.307 | 0.319 | 0.348 |
| p_flat | 0.369 | 0.380 | 0.389 | 0.450 |
| p_risk | 0.403 | 0.417 | 0.432 | 0.483 |
| pred_cum_ret_5 | -0.001 | 0.094 | 0.182 | 0.457 |

### 3.3 门槛与分布冲突

进攻配置要求 `p_up >= 0.39` 或 `p_down >= 0.39`，但模型在 test 集上 **p_up/p_down 最大值仅约 0.35**，该条件永远无法满足。

各条件单独满足次数（test，long 方向）：

- `edge >= 0.05`：8 次
- `p_up >= 0.39`：**0 次** ← 硬阻塞
- `p_flat <= 0.40`：1123 次
- `cum5 > 0`：979 次
- `risk_ok`：1207 次

**建议**：进攻档阈值应基于模型 softmax 校准分布设定，而非照搬 v012 时代概率尺度。

## 6. 风险与合规

- 日/周回撤熔断、连亏冷却、硬止损、最大持仓时间（8 bars）均按配置生效
- 第 2 笔 LONG 触发 `CLOSE_HARD_STOP`，单笔 ROE -1.81%，为最大亏损单
- 加仓：多笔交易 `add_count=2`，符合进攻档顺势加仓设计
- 已知限制：`trades.csv` 中 `bars_held=0` 为引擎记账字段未递增，不影响 PnL 计算

## 7. 结论与建议

### 7.1 可交付结论

1. **v014 交易链路可用**：在调参后能产生真实成交、完整日志与合规指标。
2. **进攻模式需模型感知调参**：固定 `open_prob_threshold=0.39` 对 0065a 无效。
3. **OOS test 初步正向**：+4.24% / 超额 +22% / MDD -4.5%，但交易频次低。

### 7.2 后续建议

| 优先级 | 动作 |
|--------|------|
| P0 | 将 `trading_rule_v014_aggressive.json` 的 `open_prob_threshold` 下调至 ≤0.34，或增加 softmax 温度校准 |
| P1 | 扩展调参至 `max_hold_bars`、止损 ATR 倍数；用 walk-forward 替代单次 valid 调参 |
| P1 | 多 checkpoint 对比（0058 usable vs 0062e/0065a） |
| P2 | 修复 `bars_held` 统计；补充 Sharpe / Calmar 至 `metrics.json` |
