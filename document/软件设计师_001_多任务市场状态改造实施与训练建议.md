# 软件设计师_001_多任务市场状态改造实施与训练建议

## 1. 本次实施范围与目标

依据《架构师-001-模型结构改造指导》，本次实现聚焦于：

1. 统一标签契约（`labels.py`）。
2. 在数据集层显式输出多任务目标（return / direction / volatility / risk）。
3. 在现有 `KlinePatternPredictor` 内新增可切换的多任务市场状态头（保留旧路径兼容）。
4. 新增独立多任务训练脚本（不把新逻辑继续堆入旧实验损失路径）。
5. 在真实 BTC 数据上完成 CPU smoke run，产出可复现实验指标。

---

## 2. 代码实施清单

### 2.1 新增：`transformer_kit/labels.py`

新增内容：

- `MarketStateTargets`：统一多任务标签 dataclass。
- `MarketStateThresholds`：训练集阈值容器。
- `build_market_state_targets()`：
  - `future_log_ret`
  - `direction_label`（三分类：down/flat/up）
  - `volatility`（未来窗口 realized volatility）
  - `risk_label`（基于 vol 阈值）
  - `move_label`（可选辅助）
- `estimate_market_state_thresholds()`：
  - 仅基于 train future windows 拟合阈值
  - 输出 `direction_threshold` 与 `risk_vol_threshold`

设计要点：

- 标签口径集中，不再散落在训练循环里临时计算。
- 阈值拟合与应用分离，避免 valid/test 泄漏。

### 2.2 修改：`transformer_kit/segment_dataset.py`

`PatternSequenceDataset` 新增可选目标输出能力（兼容旧逻辑）：

- 新参数：
  - `return_market_state_targets: bool = False`
  - `direction_threshold: float`
  - `risk_vol_threshold: float`
- 在 `future_raw_log_ret` 可用时，额外产出：
  - `target_return`
  - `target_direction`
  - `target_volatility`
  - `target_risk`
  - `target_move`（可选）

兼容性：

- 默认不启用 `return_market_state_targets`，旧脚本行为不变。

### 2.3 修改：`transformer_kit/pattern_model.py`

新增/改造：

- `PatternPredictorConfig` 新增：
  - `use_market_state_head`
  - `direction_classes`
  - `risk_classes`
- 新增 `MarketStateOutput` dataclass：
  - `return_pred`
  - `direction_logits`
  - `volatility_pred`
  - `risk_logits`
  - `aux`
- 新增 `MarketStateHead`：
  - `return_head`
  - `direction_head`（horizon-level）
  - `volatility_head`
  - `risk_head`
- `KlinePatternPredictor.forward()`：
  - `use_market_state_head=True` 时返回 `MarketStateOutput`
  - 否则保持旧返回路径（回归/旧辅助头）

兼容性：

- 旧 Stage3 训练仍可跑；新市场状态路径由配置显式启用。

### 2.4 修改：`transformer_kit/training.py`

新增多任务独立训练组件：

- `market_state_loss()`：
  - return: Huber
  - direction: CE（3-class）
  - volatility: log-vol Huber
  - risk: CE（二分类）
- `train_market_state_epoch()`
- `evaluate_market_state()`
- 指标：
  - `direction_acc`
  - `direction_macro_f1`
  - `cum_direction_acc`
  - `return_ic`
  - `return_mae`
  - `volatility_mae`
  - `risk_f1`

### 2.5 新增脚本：`examples/train_market_state_model.py`

职责：

1. 默认真实数据加载（遵循 real-data-only 规则）。
2. 构造时序样本并从 train split 拟合标签阈值。
3. 初始化 `use_market_state_head=True` 的模型。
4. 多任务训练 + 验证 + 测试评估。
5. 输出：
   - `reports/<run>/metrics.json`
   - `reports/<run>/metrics.txt`

## 4. 训练建议（下一轮正式训练）

### 4.1 第一轮正式配置（建议）

- `symbol=BTCUSDT`
- `interval=1h`
- `days=365`
- `context_bars=128`
- `pred_horizon=5`
- `d_model=128`
- `n_heads=4`
- `trunk_layers=2`
- `epochs=30~50`
- `batch_size=64`
- `encoder_lr_scale=0.03~0.08`

损失权重建议：

- `return_weight=0.40`
- `direction_weight=0.40`
- `volatility_weight=0.15`
- `risk_weight=0.05`

### 4.2 训练与选模准则

建议使用复合选模分数（valid）：

`score = cum_direction_acc + 0.5 * return_ic + 0.25 * direction_macro_f1 - 0.1 * volatility_mae`

理由：

- 避免单一 accuracy 导致目标冲突；
- 与架构师文档的“多指标一致性”要求一致。

### 4.3 泄漏与口径检查（每轮必做）

1. 方向/风险阈值仅从 train 拟合。
2. valid/test 不重新拟合阈值。
3. `future_*` 标签严格从 `context_end` 之后生成。
4. 归一化统计只用 trailing 历史窗口。

### 4.4 对比基线建议

每轮正式实验至少对比：

1. 现有 CPC direction baseline（step/cum/all5）。
2. 本市场状态模型：
   - `cum_direction_acc`
   - `return_ic`
   - `volatility_mae`
   - `risk_f1`

合并默认路径前建议满足：

- `cum_direction_acc` 不低于当前 baseline；
- `return_ic` 在 valid/test 同号且为正（至少稳定接近 0）；
- `volatility_mae` 稳定下降；
- `risk_f1` 高于类别频率基线。
