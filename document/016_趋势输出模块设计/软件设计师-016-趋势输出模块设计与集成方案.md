本文启动 016 项目。目标是把目前偏“下跌过滤器”的 `TrendRegimeFilter` 升级为完整的趋势输出模块，使交易系统能够识别趋势方向、趋势强度、趋势阶段、趋势持续性与趋势失效，从而解决“短收益能吃到，长段收益吃不到”的结构性问题。

---

## 2. 背景与问题

当前交易系统已经具备：

```text
0062e 市场状态模型：负责短周期方向/累计收益信号；
014b TrendHoldMode：模型确认空头后延长持仓；
015 CrashRegimeDetector：模型失效下识别崩跌。
```

但仍存在核心短板：

```text
1. 模型预测窗口偏短，主要服务未来 5 根 1h K；
2. 交易模块仍以开仓信号为中心；
3. 盈利仓缺少系统化“趋势升级”机制；
4. 趋势判断只偏向下跌，不完整支持上涨趋势；
5. 缺少趋势阶段输出，无法区分启动、延续、加速、衰竭、反转。
```

016 要解决的是：

```text
让趋势模块回答“是否值得继续持有”，而不是只回答“现在能不能开仓”。
```

---

## 3. 金融定义

程序中的趋势应定义为：

```text
趋势 = 价格在一段时间内沿某一方向持续移动，
且回撤没有破坏该方向结构的市场状态。
```

它必须同时包含五个维度：

```text
方向：上涨 / 下跌 / 无趋势；
强度：移动幅度是否足够大；
结构：是否形成 higher-high/higher-low 或 lower-high/lower-low；
持续性：是否连续存在，而不是单根冲击；
失效：趋势何时被破坏。
```

因此 016 的趋势模块不应直接预测收益，而应输出：

```text
当前是否处在可持仓趋势中；
当前趋势属于哪个阶段；
当前趋势是否足够强，可升级仓位；
当前趋势是否破坏，应减仓或退出。
```

## 6. 趋势计算维度

### 6.1 均线方向

默认：

```text
ema_fast = 12
ema_mid = 24
ema_slow = 72
```

上涨：

```text
close > ema_fast
ema_fast > ema_mid
ema_mid > ema_slow
```

下跌：

```text
close < ema_fast
ema_fast < ema_mid
ema_mid < ema_slow
```

原因：

```text
12/24/72 在 1h 周期上分别代表半日、1 日、3 日级别趋势；
比 12/36 更适合判断稍长趋势。
```

### 6.2 ATR 标准化收益

计算：

```text
ret_6_atr = (close[t] - close[t-6]) / atr[t]
ret_12_atr = (close[t] - close[t-12]) / atr[t]
ret_24_atr = (close[t] - close[t-24]) / atr[t]
```

判断：

```text
abs(ret_12_atr) >= 1.5 -> 有趋势移动
abs(ret_12_atr) >= 2.5 -> 强趋势
abs(ret_24_atr) >= 4.0 -> 极端趋势 / crash-surge 候选
```

### 6.3 价格结构

第一版不做复杂波峰波谷，使用滚动高低点：

上涨结构：

```text
close >= rolling_high_24
rolling_low_12 > rolling_low_24_previous
```

下跌结构：

```text
close <= rolling_low_24
rolling_high_12 < rolling_high_24_previous
```

可选第二版再引入 swing high / swing low。

### 6.4 持续性

定义：

```text
persistence_ratio_up = 过去 12 根 close > ema_fast 的比例
persistence_ratio_down = 过去 12 根 close < ema_fast 的比例
```

判断：

```text
persistence_ratio >= 0.67 -> 趋势具备持续性
persistence_ratio >= 0.80 -> 趋势稳定
```

### 6.5 波动扩张

计算：

```text
range_expansion = mean(true_range[t-3:t]) / mean(true_range[t-24:t])
```

用途：

```text
range_expansion > 1.4 且同向 ret_6_atr 强 -> 趋势加速；
range_expansion > 2.0 且价格远离 ema_fast -> 可能过热/衰竭。
```

## 7. 趋势打分

### 7.1 上涨趋势投票

```text
close > ema_fast                         +1
ema_fast > ema_mid > ema_slow            +1
ret_12_atr >= 1.5                        +1
ret_24_atr >= 2.5                        +1
close >= rolling_high_24                 +1
persistence_ratio_up >= 0.67             +1
higher_high_low == true                  +1
```

### 7.2 下跌趋势投票

```text
close < ema_fast                         +1
ema_fast < ema_mid < ema_slow            +1
ret_12_atr <= -1.5                       +1
ret_24_atr <= -2.5                       +1
close <= rolling_low_24                  +1
persistence_ratio_down >= 0.67           +1
lower_high_low == true                   +1
```

### 7.3 判定

```text
score <= 2 -> NONE / WEAK
score == 3 -> NORMAL trend candidate
score >= 4 -> confirmed trend
score >= 5 -> strong trend
score >= 6 且 abs(ret_24_atr) >= 4 -> extreme trend
```

方向选择：

```text
若 score_up - score_down >= 2 -> UP
若 score_down - score_up >= 2 -> DOWN
否则 NONE
```

置信度：

```text
confidence = abs(score_up - score_down) / max(score_up, score_down, 1)
```

## 8. 趋势阶段定义

### 8.1 EARLY

```text
score 从 <=2 升到 >=3
trend_age <= 3
价格刚突破 ema_fast 或 rolling high/low
```

用途：

```text
可用于观察或小仓位，不直接高仓位追入。
```

### 8.2 CONTINUATION

```text
score >= 4
trend_age >= 3
persistence_ratio >= 0.67
未出现明显衰竭
```

用途：

```text
允许盈利仓升级为趋势仓；
允许趋势仓继续持有。
```

### 8.3 ACCELERATION

```text
score >= 5
range_expansion >= 1.4
ret_6_atr 与趋势方向一致且绝对值 >= 2
```

用途：

```text
允许顺势加仓，但必须受 20% 总仓位上限约束。
```

### 8.4 EXHAUSTION

上涨衰竭：

```text
distance_from_ema_slow_atr >= 5
range_expansion >= 2
ret_6_atr 开始回落
```

下跌衰竭：

```text
distance_from_ema_slow_atr <= -5
range_expansion >= 2
ret_6_atr 负值开始收敛
```

用途：

```text
不新增仓；
收紧 trailing stop；
禁止盲目追涨追空。
```

### 8.5 REVERSAL_RISK

上涨趋势中：

```text
close < ema_fast 连续 2 根
或 ret_6_atr <= -1.5
或跌破 rolling_low_12
```

下跌趋势中：

```text
close > ema_fast 连续 2 根
或 ret_6_atr >= 1.5
或突破 rolling_high_12
```

用途：

```text
减仓或退出趋势仓。
```

## 9. 状态记忆

趋势模块必须有状态，不能每根 bar 独立判断。

新增：

```python
@dataclass
class TrendMemory:
    current_direction: TrendDirection
    trend_age: int
    invalid_count: int
    last_confirmed_direction: TrendDirection
    last_confirmed_ts: Any
    phase: TrendPhase
```

更新规则：

```text
同方向 confirmed -> trend_age += 1
方向变 NONE -> invalid_count += 1
方向反向 confirmed -> 重置 trend_age，并切换方向
invalid_count >= 2 -> is_broken = true
```

## 11. 行为规则

### 11.1 盈利仓升级为趋势仓

多头升级：

```text
position.side == LONG
position unrealized_profit_atr >= 1.5
trend_signal.direction == UP
trend_signal.is_confirmed == true
trend_signal.phase in [CONTINUATION, ACCELERATION]
p_risk < risk_exit_threshold
```

动作：

```text
UPGRADE_TO_TREND_LONG
hold_mode = TREND
```

空头升级：

```text
position.side == SHORT
position unrealized_profit_atr >= 1.5
trend_signal.direction == DOWN
trend_signal.is_confirmed == true
trend_signal.phase in [CONTINUATION, ACCELERATION]
p_risk < risk_exit_threshold
```

动作：

```text
UPGRADE_TO_TREND_SHORT
hold_mode = TREND
```

### 11.2 趋势仓继续持有

多头趋势仓：

```text
trend_signal.direction == UP
trend_signal.is_broken == false
trend_signal.phase not in [REVERSAL_RISK]
```

空头趋势仓：

```text
trend_signal.direction == DOWN
trend_signal.is_broken == false
trend_signal.phase not in [REVERSAL_RISK]
```

动作：

```text
HOLD_TREND_CONTINUATION
```

### 11.3 趋势仓加仓

只允许一次或两次，且必须满足：

```text
hold_mode == TREND
trend_signal.phase == ACCELERATION
position.add_count < max_add_count
current_position_ratio < max_position_ratio
p_risk < risk_open_max
unrealized_profit_atr >= 2.0
```

动作：

```text
ADD_TREND_CONTINUATION
```

仓位：

```text
每次增加 25%~35% 当前仓位；
总仓位不得超过 20%。
```

### 11.4 趋势衰竭减仓

触发：

```text
trend_signal.phase == EXHAUSTION
或 trend_signal.score_abs 从峰值回落 >= 2
```

动作：

```text
REDUCE_TREND_EXHAUSTION
```

默认：

```text
减仓 30%~50%，剩余仓位使用 trailing stop。
```

### 11.5 趋势破坏退出

触发：

```text
trend_signal.is_broken == true
或 trend_signal.phase == REVERSAL_RISK
```

动作：

```text
CLOSE_TREND_BROKEN
```

注意：

```text
硬止损、灾难止损、账户熔断仍优先。
```

## 12. 与 0062e 模型的分工

保持职责分离：

```text
0062e 模型：负责开仓候选、短期方向和累计收益判断；
TrendSignalProvider：负责趋势状态、持仓升级、继续持有、趋势退出；
CrashRegimeDetector：负责模型失效下的极端崩跌补救；
RuleEngine：融合三者输出为交易动作。
```

开仓优先级：

```text
1. 标准模型开仓仍为主；
2. Crash/Surge 只处理极端模型失效；
3. 趋势模块第一版不单独普通开仓，只负责升级与持有。
```

原因：

```text
014a 已证明单纯趋势开仓会造成过度交易；
016 第一阶段必须避免重新引入高频 probe。
```

```text
普通盈利仓持仓周期延长；
固定 max_hold_bars 不再截断趋势仓。
```

### 阶段三：趋势退出与减仓

实现：

```text
REDUCE_TREND_EXHAUSTION
CLOSE_TREND_BROKEN
趋势 trailing stop
```

验收：

```text
长趋势能保留利润；
趋势破坏后能退出。
```

### 阶段四：趋势加仓

实现：

```text
ADD_TREND_CONTINUATION
```

验收：

```text
只在 ACCELERATION 阶段加仓；
无过度交易；
仓位不超过 20%。
```

---

## 18. 不建议做的事

不建议：

```text
让趋势模块直接替代 0062e 做普通开仓；
把所有 confirmed trend 都立即开仓；
用单一 EMA 判断趋势；
只用收益最大化调参趋势阈值；
取消固定风险边界；
趋势仓无限持有。
```

原因：

```text
014a 已经证明趋势规则直接开仓容易过度交易；
016 的重点是持仓管理，不是增加交易频率。
```

## 19. 结论

016 的核心是新增可复用的趋势输出模块：

```text
TrendSignalProvider
  -> 输出方向、强度、阶段、持续性、失效状态
  -> 支持盈利仓升级为趋势仓
  -> 支持趋势继续持有、减仓、退出、加仓
```

一句话：

```text
0062e 负责发现短期机会；
016 趋势模块负责把短期机会中真正发展成趋势的部分拿住。
```
