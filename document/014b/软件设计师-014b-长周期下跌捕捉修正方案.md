# 长周期下跌捕捉修正方案

## 1. 文档说明

| 项 | 内容 |
|----|------|
| 角色 | 软件设计师 |
| 流水号 | 014b |
| 主题 | 修正 014a 收益下降与 6 月长周期下跌漏捕捉问题 |
| 依据 | `document/014a/软件设计师-014a-下跌识别补强实施指导.md` |
| 回测依据 | `backtest/backtest_rule_v014a_downside_protection_0062e_test/`、`backtest/backtest_rule_v014a_tuned_0062e_test/` |
| 目标读者 | 程序员、回测师 |

本文提出 014b 修改方案，用于替代 014a 的“频繁试探空单”路线。014b 的重点不是继续增加 probe short，而是把已经出现的模型确认空头和趋势确认空头持得更久、更稳定。

---

## 2. 014a 回测结论

### 2.1 原始 014a 结果

```text
total_return = -5.01%
max_drawdown = -7.09%
trade_count = 25
probe_short_count = 25
probe_short_total_return = -3.26%
short_coverage_downtrend_ratio = 13.9%
```

结论：

```text
014a 的趋势识别确实触发了 probe short，
但 probe 质量差、交易过密、频繁小亏，
导致收益严重下降。
```

### 2.2 014a + tuned 结果

```text
total_return = +2.81%
max_drawdown = -4.87%
trade_count = 28
probe_short_count = 24
probe_short_total_return = -4.14%
blocked_long_downtrend_count = 4
```

结论：

```text
恢复 tuned 阈值后，标准模型交易重新贡献收益；
但 probe 仍持续拖累；
6 月大空头因为 max_hold_bars=8 被提前截断。
```

## 3. 失败原因判断

### 3.1 probe short 不是主解决方案

014a 中所有新增空头基本都是：

```text
OPEN_SHORT_PROBE_DOWNTREND
```

但 probe 只有低胜率：

```text
win_rate 约 28%~29%
probe PnL 为负
```

判断：

```text
价格趋势规则适合做防守过滤，不适合独立开仓。
```

### 3.2 6 月大跌不是没有任何信号，而是没有持住

在 baseline v014 tuned 中，6 月 17 日出现标准空头：

```text
OPEN_SHORT_SIGNAL
```

并贡献约：

```text
+12.9% ROE
```

014a + tuned 中也出现该标准空头，但：

```text
2026-06-17 18:00 开空
2026-06-18 02:00 因 CLOSE_MAX_HOLD_BARS 平仓
收益约 +5.3%
```

判断：

```text
014b 要解决的核心不是更早乱开空，
而是当模型和趋势都确认下跌后，不应被固定 8 根 bar 强行退出。
```

### 3.3 固定 max_hold_bars 不适合长周期趋势

当前预测窗口是 5 根 bar，但交易系统面对的是趋势段。

固定：

```text
max_hold_bars = 8
```

会导致：

```text
趋势仍在继续；
模型或价格结构仍支持空头；
系统仍强制平仓。
```

这正是 6 月长跌段收益被截断的直接原因。

---

## 4. 014b 总体方向

014b 采用：

```text
标准模型开仓优先；
趋势规则只做过滤和持仓管理；
probe short 降级为极少数强趋势预警仓；
固定时间退出改为趋势条件退出。
```

核心改造：

```text
1. 禁止 probe 主导交易
2. 新增 TrendHoldMode 趋势持仓模式
3. 标准空头进入趋势模式后延长持仓
4. probe short 必须升级为标准空头，否则快速退出
5. max_hold_bars 分为普通持仓和趋势持仓两套
```

---

## 5. 新增概念：`TrendHoldMode`

### 5.1 目的

当空头已经得到模型确认，且价格趋势也支持下跌时，系统进入趋势持仓模式。

```text
TrendHoldMode 不是开仓信号；
TrendHoldMode 是持仓延长机制。
```

### 5.2 触发条件

空头持仓时，满足以下条件进入：

```text
position.side == SHORT
entry_reason == OPEN_SHORT_SIGNAL
trend_context.is_downtrend == true
pred_cum_ret_5 < 0
p_risk <= risk_exit_threshold
```

若强趋势：

```text
trend_context.is_strong_downtrend == true
```

可直接进入。

状态字段：

```text
position.hold_mode = NORMAL / TREND
position.trend_hold_bars
position.trend_break_count
```

## 6. 修改一：停止普通 probe short 泛滥

### 6.1 默认关闭 probe

014b 配置默认：

```json
{
  "protection": {
    "allow_probe_short": false
  }
}
```

原因：

```text
014a 已证明 probe 作为独立开仓源会显著拉低收益。
```

### 6.2 仅保留强趋势预警仓

如果保留 probe，必须改名并收紧为：

```text
OPEN_SHORT_SENTINEL
```

触发条件必须全部满足：

```text
position.side == FLAT
trend_context.is_strong_downtrend == true
ret_6_atr <= -2.0
close < ema_fast < ema_slow
p_risk <= 0.42
p_flat <= 0.38
edge <= 0
pred_cum_ret_5 <= 0 或 最近 3 根 pred_cum_ret_5 均值 <= 0
```

仓位：

```text
position_ratio = 0.03
max_position_ratio = 0.05
```

频率限制：

```text
同一 downtrend regime 只允许 1 次 sentinel；
sentinel 止损后，至少 cooldown 12 根 bar；
sentinel 不允许连续重开。
```

验收目标：

```text
sentinel_short_count <= baseline_trade_count
sentinel_short_total_return 不得显著为负
```

---

## 7. 修改二：标准空头趋势持仓延长

### 7.1 新增配置

```json
{
  "trend_hold": {
    "enabled": true,
    "normal_max_hold_bars": 8,
    "short_trend_max_hold_bars": 24,
    "strong_short_trend_max_hold_bars": 36,
    "trend_break_confirm_bars": 2,
    "min_profit_to_extend_atr": 0.5,
    "allow_extend_only_for_model_short": true
  }
}
```

解释：

```text
普通持仓仍使用 8 根；
模型确认空头 + 趋势确认后，可延长到 24~36 根；
probe/sentinel 不能自动享受完整趋势持仓，除非升级。
```

### 7.2 时间退出改造

当前：

```text
if bars_held >= max_hold_bars:
    CLOSE_MAX_HOLD_BARS
```

014b 改为：

```text
if hold_mode == NORMAL and bars_held >= normal_max_hold_bars:
    CLOSE_MAX_HOLD_BARS

if hold_mode == TREND and bars_held >= short_trend_max_hold_bars:
    CLOSE_TREND_MAX_HOLD_BARS
```

强趋势：

```text
if is_strong_downtrend:
    max_hold = strong_short_trend_max_hold_bars
```

---

## 8. 修改三：趋势持仓退出规则

趋势持仓不能只看固定时间，也不能只看单根模型反向。

### 8.1 趋势破坏计数

空头趋势持仓中，若以下任一出现，累计一次 `trend_break_count`：

```text
close > ema_fast
edge >= reverse_edge_threshold
pred_cum_ret_5 > 0
ret_3_atr > 0.8
```

若以下条件恢复，清零：

```text
close < ema_fast
且 pred_cum_ret_5 <= 0
且 edge < reverse_edge_threshold
```

触发退出：

```text
trend_break_count >= trend_break_confirm_bars
```

动作：

```text
CLOSE
reason_code = CLOSE_SHORT_TREND_BROKEN
```

### 8.2 利润保护退出

空头趋势持仓中，加入追踪止盈，不等固定时间：

```text
peak_profit_atr = max(peak_profit_atr, (entry_price - current_price) / atr)
```

若：

```text
peak_profit_atr >= 2.0
且 profit_atr 回撤 >= 1.0
```

触发：

```text
CLOSE_SHORT_TREND_TRAIL
```

目的：

```text
长跌中尽量持住；
反弹明显时保护利润。
```

### 8.3 风控仍优先

以下规则不受趋势持仓延长影响：

```text
CLOSE_HARD_STOP
CLOSE_CATASTROPHE_MARGIN_LOSS
CLOSE_RISK_PROB_HIGH
账户级熔断
```

## 9. 修改四：sentinel 空头升级机制

若保留 `OPEN_SHORT_SENTINEL`，必须设计升级条件。

### 9.1 升级为标准空头

sentinel 持仓中，若后续出现：

```text
edge <= -open_edge_threshold
p_down >= open_prob_threshold
pred_cum_ret_5 < 0
p_flat <= open_flat_max
p_risk <= risk_open_max
```

动作：

```text
ADD 或 RESIZE_TO_MODEL_SHORT
reason_code = UPGRADE_SENTINEL_TO_MODEL_SHORT
position.hold_mode = TREND
```

仓位：

```text
按标准 PositionSizer 重新计算；
总仓位仍不得超过 20%。
```

### 9.2 未升级则快速退出

sentinel 若在以下时间内未升级：

```text
sentinel_max_hold_bars = 4
```

则退出：

```text
CLOSE_SENTINEL_NOT_CONFIRMED
```

这可以避免 014a 中大量低质量 probe 持续消耗收益。

---

## 10. 修改五：下跌识别从“开仓条件”降级为“状态过滤”

014b 中 `TrendRegimeFilter` 主要用于：

```text
1. 阻止下跌趋势中错误做多；
2. 延长已确认空头持仓；
3. 保护空头利润；
4. 极端情况下发出 sentinel 预警仓。
```

不再用于：

```text
普通下跌结构一出现就 OPEN_SHORT_PROBE。
```

因此 014a 中以下规则应删除或默认关闭：

```text
OPEN_SHORT_PROBE_DOWNTREND
probe_short_min_edge
probe_short_min_p_down
probe_short_require_cum_ret_negative = false
```

## 11. 程序修改点

### 11.1 `state.py`

`PositionState` 新增：

```text
hold_mode
trend_hold_bars
trend_break_count
entry_was_sentinel
sentinel_bars
peak_profit_atr
```

### 11.2 `config.py`

新增配置段：

```text
trend_hold
sentinel_short
```

保留 `trend`，但 `protection.allow_probe_short` 默认 false。

### 11.3 `rules.py`

修改：

```text
删除普通 OPEN_SHORT_PROBE_DOWNTREND 主路径；
新增 OPEN_SHORT_SENTINEL；
新增 UPGRADE_SENTINEL_TO_MODEL_SHORT；
新增 trend hold 的继续持仓判断；
新增 CLOSE_SHORT_TREND_BROKEN。
```

### 11.4 `risk.py`

修改时间退出：

```text
NORMAL 用 normal_max_hold_bars；
TREND 用 short_trend_max_hold_bars；
strong downtrend 用 strong_short_trend_max_hold_bars。
```

风险强平、硬止损保持最高优先级。

### 11.5 `engine.py`

每根 bar 更新：

```text
trend_context
position.hold_mode
position.trend_hold_bars
position.trend_break_count
position.peak_profit_atr
position.sentinel_bars
```

### 11.6 `logger.py`

新增日志字段：

```text
hold_mode
trend_hold_bars
trend_break_count
entry_was_sentinel
sentinel_bars
peak_profit_atr
```

## 13. 回测验证要求

### 13.1 必须对照

```text
baseline: backtest_rule_v014_aggressive_tuned_0062e_test
failed:   backtest_rule_v014a_tuned_0062e_test
new:      backtest_rule_v014b_trend_hold_0062e_test
```

### 13.2 验收目标

硬性：

```text
position_limit_violations == 0
risk_rule_violations == 0
max_margin_loss_ratio_observed < 1.00
trade_count <= baseline_trade_count * 2
sentinel_short_count <= baseline_trade_count
```

收益与风险：

```text
total_return > 014a_tuned
max_drawdown <= 014a_tuned
profit_factor > 014a_tuned
probe/sentinel_total_return 不得显著为负
```

6 月长跌专项：

```text
6/17 标准空头不得因 8 根 bar 固定时间退出；
若 trend_context 仍为 downtrend，应继续持有；
该段收益应接近或超过 baseline 同段收益的 80%；
若提前退出，必须是 CLOSE_SHORT_TREND_BROKEN 或风险退出，而不是 CLOSE_MAX_HOLD_BARS。
```

### 13.3 行为指标

新增：

```text
model_short_trend_hold_count
avg_model_short_hold_bars
close_max_hold_bars_in_downtrend_count
close_short_trend_broken_count
sentinel_short_count
sentinel_upgrade_count

```

关键验收：

```text
close_max_hold_bars_in_downtrend_count 应显著下降；
sentinel_upgrade_count / sentinel_short_count 应可解释；
若 sentinel 大量未升级，应关闭 sentinel。
```

## 14. 不建议继续的方向

不建议：

```text
继续增加 probe short 触发次数；
继续降低 probe_short_min_edge；
继续放宽 p_flat；
继续用普通 EMA 下跌结构直接开空；
继续用固定 max_hold_bars 管理趋势空头；
用收益调参掩盖 probe 质量问题。
```

014a 已证明这些方向会造成：

```text
交易次数暴增；
胜率下降；
手续费和小亏累积；
错过真正的大趋势持仓。
```

---

## 15. 一句话结论

014b 应从：

```text
“看到下跌就小仓位试空”
```

改为：

```text
“模型确认空头后，用趋势状态帮助持住；趋势规则只做过滤、延长和极少数预警。”
```

优先级：

```text
先修复 6 月标准空头被 8 根 bar 截断的问题；
再评估是否需要极小仓位 sentinel；
若 sentinel 继续亏损，默认关闭。
```
