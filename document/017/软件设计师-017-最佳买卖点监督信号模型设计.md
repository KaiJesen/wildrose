# 最佳买卖点监督信号模型设计

## 1. 文档说明

| 项 | 内容 |
|----|------|
| 角色 | 软件设计师 |
| 流水号 | 017 |
| 主题 | 使用事后最优买卖点构造可训练信号模型 |
| 依据 | `trade/tools/optimal_trade_points.py`、`document/016/软件设计师-016-趋势输出模块设计与集成方案.md` |
| 目标读者 | 程序员、模型训练师、回测师 |

本文将“最佳买卖点工具”细化为可执行的模型训练项目。核心原则是：最佳买卖点可以做老师，但不能原样做答案。程序必须把事后最优点转成因果可预测的机会区、持仓区、退出区标签，再训练辅助信号模型。

---

## 2. 项目目标

017 项目目标：

```text
基于 hindsight 最优交易序列，
生成可监督学习的交易机会标签，
训练一个 BestPointSignalModel，
为交易模块提供辅助的 entry / hold / exit / opportunity 信号。
```

该模型不替代：

```text
0062e 市场状态模型；
016 TrendSignalProvider；
015 CrashRegimeDetector。
```

它的定位是：

```text
辅助识别高质量交易机会区；
辅助判断是否继续持有；
辅助识别退出区域；
辅助过滤低质量开仓。
```

---

## 3. 关键风险

`trade/tools/optimal_trade_points.py` 明确说明：

```text
全局最优多空点标注器使用未来 K 线信息；
不可用于实盘实时信号。
```

因此 017 必须遵守：

```text
标签可以来自未来；
输入特征必须只来自 t 时刻及之前；
训练/验证/测试必须时间切分；
不得把未来收益、未来最高低点、未来最优点位置作为输入特征。
```

禁止训练：

```text
“当前这一根就是最佳买点/卖点”的点分类器。
```

推荐训练：

```text
未来一段是否进入高质量机会区；
当前是否处于值得持仓区；
当前是否接近退出区；
当前机会的期望收益和风险。
```

## 6. 标签设计

### 6.1 不能使用点标签

错误标签：

```text
t == entry_index -> long_entry / short_entry
t == exit_index -> exit
```

问题：

```text
最佳点通常只有极少数 bar；
类别极度不平衡；
点位对手续费/滑点/价格微扰极敏感；
模型容易学成噪声。
```

### 6.2 使用区域标签

将每笔 hindsight trade 扩展成：

```text
ENTRY_ZONE
HOLD_ZONE
EXIT_ZONE
NO_TRADE
```

对一笔交易：

```text
entry_index = i
exit_index = j
direction = long / short
holding_bars = j - i
```

定义：

```text
entry_zone = [i - pre_entry_bars, i + post_entry_bars]
hold_zone = [i + 1, j - 1]
exit_zone = [j - pre_exit_bars, j + post_exit_bars]
```

默认：

```text
pre_entry_bars = 2
post_entry_bars = 2
pre_exit_bars = 2
post_exit_bars = 1
```

注意：

```text
entry_zone 可以包含 entry_index 之前的 bar；
这是允许的，因为标签代表“未来即将进入机会区”；
但特征仍只能用当时之前数据。
```

## 7. 多任务标签

### 7.1 Entry 标签

```text
entry_label:
  0 = no_entry
  1 = long_entry_zone
  2 = short_entry_zone
```

### 7.2 Hold 标签

```text
hold_label:
  0 = flat_or_no_hold
  1 = hold_long
  2 = hold_short
```

### 7.3 Exit 标签

```text
exit_label:
  0 = no_exit
  1 = exit_long_zone
  2 = exit_short_zone
```

### 7.4 Opportunity 回归标签

```text
future_best_net_roi:
  从 t 起未来 opportunity_window 内可获得的最佳净收益

future_best_direction:
  0 = none
  1 = long
  2 = short

future_best_holding_bars:
  最佳机会对应持仓长度

future_max_adverse_excursion:
  从 entry 到 exit 区间的最大不利波动
```

第一版可以只实现：

```text
entry_label
hold_label
exit_label
future_best_net_roi
```

## 8. 标签冲突处理

不同 hindsight 交易区间可能相邻或重叠。处理优先级：

```text
EXIT_ZONE > ENTRY_ZONE > HOLD_ZONE > NO_TRADE
```

方向冲突时：

```text
净收益 net_roi 高者优先；
若差距小于 min_label_roi_gap，则标为 ambiguous，不参与分类损失。
```

默认：

```text
min_label_roi_gap = 0.01
ambiguous_label = -100
```

PyTorch 中可用 `ignore_index=-100`。

---

## 9. 标签质量控制

程序员需要在 `labeler.py` 中输出质量统计：

```text
entry_label 分布
hold_label 分布
exit_label 分布
long/short 比例
ambiguous 比例
平均 holding_bars
平均 net_roi
标签覆盖率
```

拒绝条件：

```text
entry zone 占比 > 30%
entry zone 占比 < 1%
ambiguous 占比 > 20%
long/short 极度单边，例如任一方向 < 10%
```

原因：

```text
标签太密会退化成总是交易；
标签太稀会无法训练；
方向严重不平衡会导致模型坍缩。
```

## 13. Loss 设计

```text
loss = entry_weight * CE(entry_logits, entry_label)
     + hold_weight * CE(hold_logits, hold_label)
     + exit_weight * CE(exit_logits, exit_label)
     + opportunity_weight * Huber(opportunity_pred, future_best_net_roi)
```

默认：

```text
entry_weight = 1.0
hold_weight = 0.6
exit_weight = 0.7
opportunity_weight = 0.4
```

类别不平衡处理：

```text
entry/exit 使用 class_weight 或 focal loss；
hold 可使用轻度 class_weight；
ignore_index = -100 跳过 ambiguous。
```

## 16. 交易模块使用规则

### 16.1 开仓辅助

不允许 017 单独直接开仓。

第一版只作为确认因子：

多头开仓增强：

```text
0062e OPEN_LONG_SIGNAL
且 p_long_entry_zone >= long_entry_confirm_threshold
```

空头开仓增强：

```text
0062e OPEN_SHORT_SIGNAL
且 p_short_entry_zone >= short_entry_confirm_threshold
```

若 0062e 未触发，但 017 极强：

```text
进入 WATCH_LONG / WATCH_SHORT
不直接下单；
等待 016 trend 或 015 crash/surge 确认。
```

### 16.2 持仓辅助

多头持仓：

```text
p_hold_long 高 -> 延长持仓 / 允许趋势升级
p_exit_long 高 -> 减仓或退出
```

空头持仓：

```text
p_hold_short 高 -> 延长持仓 / 允许趋势升级
p_exit_short 高 -> 减仓或退出
```

### 16.3 仓位辅助

仓位缩放：

```text
entry_conf = max(p_long_entry_zone, p_short_entry_zone)
opportunity_scale = clip(expected_opportunity_roi / target_roi, 0.5, 1.2)
```

但：

```text
不得突破最大仓位 20%；
不得突破固定 20 倍杠杆；
不得覆盖 RiskManager 的强平/熔断。
```

## 18. 验收与拒绝条件

### 18.1 模型验收

接受：

```text
entry_macro_f1 明显高于随机/多数类 baseline；
opportunity_ic > 0.03；
top 10% opportunity 样本的真实 hindsight ROI 明显高于平均；
long/short 均无坍缩。
```

拒绝：

```text
只预测 no_entry；
只预测单方向；
valid 好但 test 崩溃；
top 信号真实收益不升反降；
标签分布不稳定。
```

### 18.2 交易集成验收

接受：

```text
trade_count 不暴增；
max_drawdown 不显著恶化；
趋势持仓收益改善；
missed_major_trade_count 下降；
6 月初/中旬专项有改善。
```

拒绝：

```text
017 信号导致过度交易；
017 信号覆盖风控；
收益改善只来自 hindsight 过拟合；
OOS test 明显不稳定。
```

## 20. 不建议做的事

不建议：

```text
直接训练最佳点 one-hot；
用随机切分；
把 future return 作为输入；
把 017 模型直接接管开仓；
只用 test 图调标签参数；
为了回测收益反复调 label window。
```

原因：

```text
这些都会造成 hindsight 过拟合，回测漂亮但实盘无效。
```

## 21. 结论

017 的正确方向是：

```text
用最优买卖点工具生成 hindsight 老师；
把点标签扩展为机会区、持仓区、退出区；
训练 BestPointSignalModel 输出辅助概率和机会排序；
先观察、再确认、再参与交易；
不允许替代风控和主模型。
```

一句话：

```text
最佳买卖点不是实盘答案，而是训练模型理解“好机会长什么样”的监督教材。
```
