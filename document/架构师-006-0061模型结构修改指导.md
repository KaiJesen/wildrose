# 0061 模型结构修改指导

## 1. 文档目标

本文档面向软件工程师，基于 `0060_market_state_cum_return_recovery` 的训练结果，给出下一阶段模型结构修改建议。目标是解决当前多任务共享 head 下的指标拉扯问题，尤其是：

- `cum_direction_acc` 无法稳定恢复到目标区间；
- `return_ic` 长期低于目标；
- direction 类别分布在不同训练轮次中反复偏斜；
- risk/volatility 已经可用，不应继续通过训练权重挤压 return/direction。

本文只涉及模型结构、训练接口和评估逻辑，不涉及交易执行、下单、仓位或资金管理。

## 2. 当前结构问题判断

当前模型路径：

```text
AutoSegmentVQEncoder
  -> CausalTransformer trunk
  -> SegmentAttentionPool / raw_context
  -> MarketStateHead
       - return_head
       - direction_head
       - volatility_head
       - risk_head
```

当前 `MarketStateHead` 的四个任务都主要依赖同一个 `pooled` 表示：

```text
pooled -> return_head
pooled -> direction_head
pooled -> volatility_head
pooled -> risk_head
```

`0058` 到 `0060` 的训练结果表明：

1. risk/volatility 已经能被当前结构学习。
2. direction 的逐步三分类和累计方向存在明显拉扯。
3. return IC 长期不稳定，且短 horizon `h1-h3` 多次为负。
4. 通过 loss weight、class weights、focal loss、cum direction loss 调参，只是在多个指标之间转移收益，未能同时解决 `cum_direction_acc` 与 `return_ic`。

因此，下一步不建议继续单纯训练调参，应进行轻量结构解耦。

## 3. 修改总原则

本次不是重写模型主干，而是保留已验证有效的 encoder/trunk，只修改 market state head 与训练/选模接口。

必须保留：

- `AutoSegmentVQEncoder`
- `CausalTransformer`
- `SegmentAttentionPool`
- `raw_context_features`
- 现有 `MarketStateOutput` 的基本字段
- 旧训练路径兼容开关

需要新增：

- return/cum direction 专用分支；
- horizon-aware return head；
- cumulative return / cumulative direction 显式输出；
- risk/volatility 梯度降权或可冻结机制；
- valid 选模 hard gates。

## 4. 目标结构

建议将当前单一 `MarketStateHead` 改为：

```text
shared pooled representation
        |
        +-> HorizonReturnHead
        |      - return_pred [B,H]
        |      - cum_return_pred [B]
        |
        +-> DirectionStateHead
        |      - direction_logits [B,H,3]
        |      - cum_direction_logit [B]
        |
        +-> VolatilityHead
        |      - volatility_pred [B,H]
        |
        +-> RiskHead
               - risk_logits [B,H,2]
```

核心变化：

1. `return` 不再只由普通 MLP 从 `pooled` 一次性输出。
2. `cum_return_pred` 成为显式输出，不再只依赖 `return_pred.sum(dim=1)`。
3. `cum_direction_logit` 成为正式主输出，而不是旧 auxiliary 方向头。
4. direction horizon 分类与 cumulative direction 分类分开建模。
5. risk/volatility 保持现有能力，但不再主导共享表示的更新。

## 5. 具体修改点

### 5.1 修改 `MarketStateOutput`

位置：

```text
transformer_kit/pattern_model.py
```

当前：

```python
@dataclass
class MarketStateOutput:
    return_pred: torch.Tensor
    direction_logits: torch.Tensor
    volatility_pred: torch.Tensor
    risk_logits: torch.Tensor
    aux: dict[str, torch.Tensor]
```

建议改为兼容扩展：

```python
@dataclass
class MarketStateOutput:
    return_pred: torch.Tensor
    direction_logits: torch.Tensor
    volatility_pred: torch.Tensor
    risk_logits: torch.Tensor
    aux: dict[str, torch.Tensor]
    cum_return_pred: torch.Tensor | None = None
    cum_direction_logit: torch.Tensor | None = None
```

兼容要求：

- 旧代码不传 `cum_return_pred` / `cum_direction_logit` 时仍可运行。
- 新训练脚本启用 `use_cum_heads=True` 后必须返回这两个字段。

### 5.2 扩展 `PatternPredictorConfig`

位置：

```text
transformer_kit/pattern_model.py
```

新增配置：

```python
use_cum_heads: bool = False
use_horizon_return_head: bool = False
detach_risk_vol_heads: bool = False
return_direction_hidden_mult: float = 1.0
```

含义：

- `use_cum_heads`: 启用显式 cumulative return / direction 输出。
- `use_horizon_return_head`: 启用 horizon-aware return head。
- `detach_risk_vol_heads`: risk/volatility head 使用 `pooled.detach()`，避免训练早期继续挤压 return/direction。
- `return_direction_hidden_mult`: 允许 return/direction 分支使用略大的隐藏层，但默认不扩大主干。

### 5.3 新增 `HorizonReturnHead`

位置：

```text
transformer_kit/pattern_model.py
```

建议实现：

```python
class HorizonReturnHead(nn.Module):
    def __init__(self, d_model: int, n_heads: int, horizon: int) -> None:
        super().__init__()
        self.horizon = horizon
        self.query = nn.Parameter(torch.randn(1, horizon, d_model) * 0.02)
        self.cross_attn = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        self.norm = nn.LayerNorm(d_model)
        self.step_out = nn.Sequential(
            nn.Linear(d_model, d_model),
            nn.GELU(),
            nn.Linear(d_model, 1),
        )
        self.cum_out = nn.Sequential(
            nn.Linear(d_model, d_model),
            nn.GELU(),
            nn.Linear(d_model, 1),
        )

    def forward(
        self,
        pooled: torch.Tensor,
        tokens: torch.Tensor,
        seg_mask: torch.Tensor,
    ) -> tuple[torch.Tensor, torch.Tensor]:
        ...
```

输出：

```text
return_pred      [B,H]
cum_return_pred  [B]
```

设计要点：

- 每个 horizon 用独立 query cross-attend trunk tokens，避免所有 horizon 被同一个 pooled MLP 平滑输出。
- `cum_return_pred` 使用 pooled 或 cross-attention 后的 horizon 表示单独预测。
- `cum_return_pred` 不要求等于 `return_pred.sum(dim=1)`，但训练时可以加入一致性约束。

### 5.4 新增 `DirectionStateHead`

位置：

```text
transformer_kit/pattern_model.py
```

建议实现：

```python
class DirectionStateHead(nn.Module):
    def __init__(self, d_model: int, horizon: int, direction_classes: int = 3) -> None:
        super().__init__()
        self.step_head = nn.Sequential(
            nn.Linear(d_model, d_model),
            nn.GELU(),
            nn.Linear(d_model, horizon * direction_classes),
        )
        self.cum_head = nn.Sequential(
            nn.Linear(d_model, d_model // 2),
            nn.GELU(),
            nn.Linear(d_model // 2, 1),
        )

    def forward(self, pooled: torch.Tensor) -> tuple[torch.Tensor, torch.Tensor]:
        ...
```

输出：

```text
direction_logits       [B,H,3]
cum_direction_logit    [B]
```

设计要点：

- `direction_logits` 继续负责 down/flat/up 三分类。
- `cum_direction_logit` 负责未来累计收益方向二分类。
- 不再用 `return_pred.sum(dim=1)` 间接承担全部累计方向任务。

### 5.5 修改 `MarketStateHead`

当前：

```text
return_head
direction_head
volatility_head
risk_head
```

建议改造为：

```python
class MarketStateHead(nn.Module):
    def __init__(
        self,
        d_model: int,
        horizon: int,
        *,
        n_heads: int,
        direction_classes: int = 3,
        risk_classes: int = 2,
        use_cum_heads: bool = False,
        use_horizon_return_head: bool = False,
        detach_risk_vol_heads: bool = False,
    ) -> None:
        ...

    def forward(
        self,
        pooled: torch.Tensor,
        tokens: torch.Tensor | None = None,
        seg_mask: torch.Tensor | None = None,
    ) -> MarketStateOutput:
        ...
```

兼容逻辑：

- `use_horizon_return_head=False` 时保留旧 `return_head`。
- `use_cum_heads=False` 时不输出 `cum_return_pred` / `cum_direction_logit`。
- `detach_risk_vol_heads=True` 时：

```python
risk_vol_pooled = pooled.detach()
```

只让 risk/volatility 更新自己的 head，不回传到 shared trunk。

默认行为必须保持旧实验兼容：

```text
use_cum_heads=False
use_horizon_return_head=False
detach_risk_vol_heads=False
```

### 5.6 修改 `KlinePatternPredictor.forward()`

当前 market-state 路径中，模型已经构造了：

```text
tokens
h
seg_mask
pooled
aux
```

需要将：

```python
self.market_state_head(pooled)
```

改为：

```python
self.market_state_head(pooled, h, seg_mask)
```

并确保 `aux` 继续包含：

```text
vq_loss
perplexity
break_reg_loss
num_segments
break_logits
segment_codes
ctx_num_segments
```

## 6. 训练逻辑修改

### 6.1 修改 `market_state_loss`

位置：

```text
transformer_kit/training.py
```

新增参数：

```python
cum_return_weight: float = 0.0
cum_direction_head_weight: float = 0.0
return_consistency_weight: float = 0.0
```

新增 loss：

```python
if output.cum_return_pred is not None:
    target_cum_return = target.future_log_ret.sum(dim=1)
    cum_return_loss = F.huber_loss(output.cum_return_pred, target_cum_return, delta=0.5)

if output.cum_direction_logit is not None:
    target_cum_dir = (target.future_log_ret.sum(dim=1) > 0).float()
    cum_dir_loss = F.binary_cross_entropy_with_logits(
        output.cum_direction_logit,
        target_cum_dir,
    )

if output.cum_return_pred is not None:
    consistency_loss = F.huber_loss(
        output.return_pred.sum(dim=1),
        output.cum_return_pred.detach(),
        delta=0.5,
    )
```

注意：

- `cum_return_pred` 监督累计收益。
- `cum_direction_logit` 监督累计方向。
- consistency loss 只用于约束 step return 与 cum return 不完全背离。
- 不要让 consistency loss 权重过大，否则会抵消独立 cum head 的意义。

### 6.2 修改 `evaluate_market_state`

新增指标：

```text
cum_return_ic
cum_return_mae
cum_direction_head_acc
cum_direction_from_return_acc
step_cum_return_gap_mae
```

定义：

- `cum_return_ic`: `cum_return_pred` 与真实累计收益的相关性。
- `cum_return_mae`: `cum_return_pred` 与真实累计收益的 MAE。
- `cum_direction_head_acc`: `cum_direction_logit > 0` 的累计方向准确率。
- `cum_direction_from_return_acc`: `return_pred.sum(dim=1)` 的累计方向准确率。
- `step_cum_return_gap_mae`: `return_pred.sum(dim=1)` 与 `cum_return_pred` 的差距。

保留旧指标：

```text
direction_acc
direction_macro_f1
cum_direction_acc
return_ic
return_mae
volatility_mae
risk_f1
```

### 6.3 修改选模逻辑

位置：

```text
examples/train_market_state_model.py
```

新增 hard gates：

```text
valid return_ic > 0.02
valid cum_return_ic > 0.02
valid cum_direction_head_acc >= 56%
valid direction_macro_f1 >= 0.33
valid risk_positive_rate_pred / true in [0.8, 1.6]
```

不满足 hard gates 的 epoch 不允许保存为 best。

如果早期训练全部不满足 hard gates，允许 fallback 保存最高分模型，但报告中必须标记：

```text
best_selection_mode = fallback
```

新增或扩展脚本运行配置，建议 run id：

```text

```

建议参数：

```text
use_market_state_head      = true
use_cum_heads              = true
use_horizon_return_head    = true
detach_risk_vol_heads      = true
return_weight              = 0.35

```

说明：

- 降低普通 step direction 权重，让 cumulative direction head 承担累计方向任务。
- 显式增加 `cum_return_weight`，保护收益排序。
- 暂时 detach risk/vol，避免它们继续挤压共享 trunk。
- 保持阈值不变，避免引入新的标签变量。

## 8. 验收标准

`0061` 最低目标：

```text
cum_direction_head_acc      >= 58%
cum_direction_from_return_acc >= 56%
direction_macro_f1          >= 0.34
return_ic                   >= 0.04
cum_return_ic               >= 0.05
volatility_mae              <= 0.075
risk_f1                     >= 0.50
```

注意：

- `0061` 的目标不是要求 risk/vol 再创新高。
- 如果 `detach_risk_vol_heads=True` 导致 risk_f1 小幅回落，但仍 `>=0.50`，可以接受。
- `return_ic` 可暂时低于原 `0.05`，但 `cum_return_ic` 必须达标，证明新 cum return 分支有效。

优秀目标：

```text
cum_direction_head_acc      >= 59%
cum_direction_from_return_acc >= 58%
direction_macro_f1          >= 0.35
return_ic                   >= 0.05
cum_return_ic               >= 0.07
volatility_mae              <= 0.070
risk_f1                     >= 0.52
```

## 9. 不建议做的事

本轮不建议：

- 扩大 `d_model` 或 trunk layer。
- 更换数据周期。
- 改动标签阈值。
- 继续提高 risk weight。
- 继续只靠 class weights / focal loss 调参。
- 删除旧 `MarketStateHead` 逻辑。

原因：

- 当前瓶颈已经不是模型容量不足，而是多任务 head 与累计目标表达不足。
- 修改太多变量会破坏 `0058-0060` 的可比性。

## 10. 软件工程师实施顺序

建议按以下顺序开发：

1. 扩展 `MarketStateOutput` 和 `PatternPredictorConfig`。
2. 实现 `HorizonReturnHead`。
3. 实现 `DirectionStateHead`。
4. 改造 `MarketStateHead`，保持旧路径兼容。
5. 修改 `KlinePatternPredictor.forward()`，把 `h` 和 `seg_mask` 传给 market state head。
6. 扩展 `market_state_loss()`。
7. 扩展 `evaluate_market_state()`。
8. 修改训练脚本参数与选模 hard gates。
9. 做一个 CPU smoke run 验证 shape 和 metrics。
10. 跑正式 `0061_market_state_return_direction_branch`。

- 旧 `use_cum_heads=False` 路径仍可用。

## 12. 结论

`0060` 表明：现有共享 pooled 多任务 head 已进入任务拉扯区间，继续只调训练权重大概率只能在 `cum_direction_acc`、`direction_macro_f1`、`return_ic`、`risk_f1` 之间做交换。

下一步应进行轻量结构改造：

```text
保留 encoder/trunk
-> 解耦 return/cum direction 分支
-> 显式输出 cum_return_pred 与 cum_direction_logit
-> 暂时降低 risk/vol 对共享 trunk 的梯度影响
-> 用 hard gates 保护 return_ic 与 cumulative direction
```

这比继续训练更符合当前指标现状，也比重构整个模型更稳健。
