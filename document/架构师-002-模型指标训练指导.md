# BTC 合约预测模型指标训练指导

## 1. 文档目标

本文档面向模型训练师，给出当前多任务市场状态模型的指标体系、选模准则、训练调参方向与验收标准。

本文只讨论模型训练与指标评估，不涉及交易执行、资金管理、下单规则或仓位规则。

## 2. 当前模型现状

当前项目已经从单一收益/方向预测，演进为多任务市场状态模型：

```text
ctx_bars
  -> KlinePatternPredictor(use_market_state_head=True)
  -> MarketStateOutput
     - return_pred
     - direction_logits
     - volatility_pred
     - risk_logits
```

当前训练与评估指标主要来自：

- `examples/train_market_state_model.py`
- `transformer_kit/training.py`
- `reports/0049_market_state_smoke/metrics.json`
- `reports/0050_market_state_formal/metrics.json`

`0050_market_state_formal` 相对 `0049_market_state_smoke` 的测试集表现：

```text
direction_acc        : 34.59% -> 34.59%
direction_macro_f1   : 0.283  -> 0.269
cum_direction_acc    : 54.73% -> 55.41%
return_ic            : -0.026 -> 0.073
return_mae           : 0.0403 -> 0.0260
volatility_mae       : 0.1505 -> 0.0852
risk_f1              : 0.453  -> 0.455
```

结论：

1. `return` 与 `volatility` 任务已有明显改善。
2. `cum_direction_acc` 略有改善，但 `direction_macro_f1` 下降，说明方向分类质量仍不均衡。
3. `risk_f1` 几乎没有改善，且验证集中长期停留在约 `0.432`，高度疑似受类别不平衡或预测坍缩影响。
4. 下一阶段训练不应继续单纯追求总 loss 下降，应围绕方向质量、风险识别和指标稳定性来选模。

## 3. 指标分层

### 3.1 一票否决指标

以下问题出现时，本轮训练结果不得作为有效结论：

- valid/test 指标使用了重新拟合的阈值、scaler 或类别权重。
- `direction_threshold` 或 `risk_vol_threshold` 不是仅由 train split 估计。
- `return_ic`、`direction_acc`、`risk_f1` 在 valid 与 test 上方向相反且无法解释。
- 训练中出现 NaN、loss 爆炸、预测全为单一类别且未在报告中说明。
- 只报告最优 test，不报告选模依据与 best valid。

### 3.2 主决策指标

训练师做模型选择时优先看以下指标：

1. `cum_direction_acc`
   - 衡量未来 `H` 步累计收益方向是否判断正确。
   - 当前测试基准约 `55.4%`。
   - 下一阶段最低目标：`>= 56%`。

2. `direction_macro_f1`
   - 衡量 down / flat / up 三类是否均衡学习。
   - 当前测试基准约 `0.269`，弱于 smoke 的 `0.283`。
   - 下一阶段最低目标：`>= 0.30`。

3. `return_ic`
   - 衡量收益预测排序相关性。
   - 当前测试基准约 `0.073`。
   - 下一阶段要求保持为正，且 valid/test 同方向。

4. `risk_f1`
   - 衡量高波动/风险状态识别能力。
   - 当前测试基准约 `0.455`。
   - 下一阶段最低目标：`>= 0.48`。

### 3.3 辅助约束指标

以下指标不能单独决定选模，但用于防止模型偏科：

- `return_mae`
  - 当前测试基准约 `0.0260`。
  - 不要求每轮下降，但不能显著劣化。

- `volatility_mae`
  - 当前测试基准约 `0.0852`。
  - 如果方向任务提升但该指标大幅恶化，说明多任务权重可能过度偏向分类。

- `loss`
  - 只作为训练稳定性指标。
  - 不建议用总 loss 作为唯一选模标准，因为多任务 loss 的量纲和权重会改变。

### 3.4 必须新增的诊断指标

建议训练脚本后续补充以下诊断项，并写入 `metrics.json`：

- `direction_class_distribution_true`
- `direction_class_distribution_pred`
- `risk_positive_rate_true`
- `risk_positive_rate_pred`
- `direction_confusion_matrix`
- `risk_confusion_matrix`
- `direction_acc_h1...hH`
- `return_ic_h1...hH`
- `pred_return_std`
- `pred_volatility_std`

原因：

- 当前 `risk_f1` 长期停留在固定水平，必须确认是否预测坍缩。
- `direction_macro_f1` 偏低，需要知道是 flat 类过多，还是 up/down 混淆。
- `return_ic` 为正但方向 F1 弱，说明回归排序和分类边界可能不一致。

## 4. 推荐选模分数

当前脚本的选模分数是：

```text
score = cum_direction_acc
      + 0.5 * return_ic
      + 0.25 * direction_macro_f1
      - 0.1 * volatility_mae
```

该分数容易让 `return_ic` 和 `cum_direction_acc` 主导选择，导致 `direction_macro_f1` 与 `risk_f1` 被忽略。

下一阶段建议改为：

```text
score = 0.45 * cum_direction_acc
      + 0.25 * direction_macro_f1
      + 0.15 * max(return_ic, -0.05)
      + 0.10 * risk_f1
      - 0.05 * volatility_mae
```

解释：

- `cum_direction_acc` 仍是第一优先级，但不允许完全压过分类均衡性。
- `direction_macro_f1` 权重提高，避免模型只学会多数类。
- `risk_f1` 进入选模分数，防止风险头长期无效。
- `return_ic` 使用下限截断，避免少数负 IC epoch 对分数造成过度扰动。
- `volatility_mae` 作为惩罚项，而不是主目标。

## 5. 训练调参指导

### 5.1 标签阈值

当前配置：

```text
direction_threshold_quantile = 0.35
risk_threshold_quantile      = 0.80
```

问题：

- direction 三分类的 `macro_f1` 偏低，可能 flat 类比例过高或边界过宽。
- risk 正样本可能过少，导致 `risk_f1` 停滞。

下一轮建议做小网格：

```text
方向阈值：
direction_threshold_quantile in [0.25, 0.30, 0.35]

风险阈值：
risk_threshold_quantile in [0.70, 0.75, 0.80]
```

优先组合：

```text
direction_threshold_quantile = 0.25
risk_threshold_quantile      = 0.70
```

要求：

- 阈值只能用 train split 估计。
- 每次报告必须记录阈值数值和各类别占比。
- 不允许为了提升 test 指标反向调阈值。

### 5.2 损失权重

当前配置：

```text
return_weight     = 0.40
direction_weight  = 0.40
volatility_weight = 0.15
risk_weight       = 0.05
```

根据 `0050` 结果，return/vol 已明显改善，direction/risk 是短板。

下一轮建议：

```text
return_weight     = 0.30
direction_weight  = 0.50
volatility_weight = 0.10
risk_weight       = 0.10
```

若 `risk_f1` 仍不动：

```text
return_weight     = 0.25
direction_weight  = 0.45
volatility_weight = 0.10
risk_weight       = 0.20
```

注意：

- 不要同时大幅改变模型结构和 loss 权重。
- 每次只做一类主变量修改，否则无法归因。

### 5.3 类别不平衡

建议按顺序处理：

1. 在 train split 统计 direction 三分类和 risk 二分类的类别频率。
2. 给 direction CE 与 risk CE 增加 class weights。
3. 如果 class weights 无效，再对 risk 使用 focal loss。
4. focal loss 只用于 direction/risk，不用于 return/volatility。

训练师需要重点观察：

- `pred` 类别分布是否接近 `true` 类别分布。
- rare class 的 recall 是否提升。
- macro F1 提升是否来自真实召回，而不是牺牲全部 precision。

### 5.4 训练轮数

当前 formal run 使用 `epochs=30`。

建议下一轮：

```text
epochs = 60
early_stopping_patience = 10
```

如果暂时未实现 early stopping，至少保留完整 history，并用新的 composite score 选 best valid。

不建议当前阶段扩大模型：

```text
d_model = 128
n_heads = 4
trunk_layers = 2
context_bars = 128
```

扩大模型只能在以下条件满足后考虑：

- direction/risk 指标不再明显受类别不平衡限制。
- valid/test 指标稳定同向。
- 小模型多次训练结果方差可控。

## 6. 指标解释规则

### 6.1 当 `return_ic` 提升但 `direction_macro_f1` 下降

解释：

- 模型可能学到收益排序，但分类边界不合适。
- 三分类标签阈值可能过宽或 flat 类过多。

动作：

- 优先调 `direction_threshold_quantile`。
- 检查 direction confusion matrix。
- 不要仅因 `return_ic` 提升就接受模型。

### 6.2 当 `cum_direction_acc` 提升但 `direction_acc` 不变

解释：

- 模型可能只在累计方向上有弱信号，逐 horizon 仍不稳定。

动作：

- 报告 `direction_acc_h1...hH`。
- 检查是否某个 horizon 贡献了全部提升。
- 若后几个 horizon 明显差，考虑 horizon-specific loss weight，但不要先改主干。

### 6.3 当 `risk_f1` 长期固定

解释：

- 风险头可能预测坍缩。
- risk 阈值可能导致正样本太少。
- risk loss 权重过低。

动作：

- 先看 `risk_positive_rate_true/pred`。
- 将 `risk_threshold_quantile` 从 `0.80` 降到 `0.70`。
- 将 `risk_weight` 从 `0.05` 提到 `0.10` 或 `0.20`。
- 必要时使用 class weights 或 focal loss。

### 6.4 当 `volatility_mae` 改善但风险无改善

解释：

- 模型能回归波动率水平，但二分类风险边界没有学好。

动作：

- 检查 risk label 是否直接来自 volatility 阈值。
- 将 risk head 的监督改成更明确的高波动分类目标。
- 对 risk 分类使用与 volatility regression 不同的 loss 权重。

## 7. 下一轮训练验收标准

下一轮建议命名为 `0051_market_state_metric_tuned`。

最低验收标准：

```text
cum_direction_acc    >= 56%
direction_macro_f1   >= 0.30
risk_f1              >= 0.48
return_ic            >  0
volatility_mae       <= 0.10
```

合格条件：

- 上述五项至少满足四项。
- 未满足项必须在报告中解释原因。
- 如果 `return_ic` 变负，本轮不能作为主线模型。
- 如果 `direction_macro_f1 < 0.27`，即使 `cum_direction_acc` 提升，也不能作为默认模型。

优秀条件：

```text
cum_direction_acc    >= 58%
direction_macro_f1   >= 0.33
risk_f1              >= 0.52
return_ic            >= 0.05
volatility_mae       <= 0.08
```

达到优秀条件后，才建议考虑结构层优化，例如 direction/risk 专用 decoder 或 CPC encoder 接入同一 market state head。

## 8. 建议训练计划

### 8.1 0051：指标权重与阈值调优

配置：

```text
direction_threshold_quantile = 0.25
risk_threshold_quantile      = 0.70
return_weight                = 0.30
direction_weight             = 0.50
volatility_weight            = 0.10
risk_weight                  = 0.10
epochs                       = 60
```

目标：

- 验证 direction/risk 指标是否对阈值和权重敏感。

### 8.2 0052：类别权重实验

前提：

- 0051 的 `direction_macro_f1` 或 `risk_f1` 未达标。

配置：

- direction CE 使用 train class weights。
- risk CE 使用 train class weights。
- 其他参数保持 0051 不变。

目标：

- 确认是否存在类别不平衡主导的问题。

### 8.3 0053：风险头专项实验

前提：

- 0052 后 `risk_f1` 仍低于 `0.48`。

配置：

- risk 使用 focal loss。
- `risk_weight` 提高到 `0.20`。
- 保持模型主干不变。

目标：

- 判断风险头是否需要结构级改造。

## 10. 训练师结论

当前模型已经证明多任务结构对 return/volatility 有帮助，但 direction/risk 仍是主瓶颈。下一阶段训练师的工作重点不是扩大模型，而是：

```text
统一指标解释
→ 调整标签阈值
→ 调整多任务权重
→ 补充类别分布诊断
→ 使用新 composite score 选模
```

只有当 direction/risk 指标在小模型上达到最低验收标准后，才进入结构扩展或更复杂预训练阶段。
