# 下跌识别补强实施指导

## 1. 文档说明

| 项 | 内容 |
|----|------|
| 角色 | 软件设计师 |
| 流水号 | 014a |
| 主题 | 0062e + 交易模块漏识别大段下跌的工程补强 |
| 依据 | `document/014_交易系统功能设计/项目经理-014-交易系统功能设计说明书.md` |
| 参考结果 | `backtest/backtest_rule_v014_aggressive_tuned_0062e_test/` |
| 目标读者 | 程序员、回测工程师 |

本文用于指导程序员在 014 交易系统上增加“下跌识别补强”能力。目标不是替换 0062e 模型，而是在模型对大段下跌反应不足时，用价格行为和状态机规则补足交易系统的防守能力。

---

## 2. 问题复盘

当前 `0062e` 配合 `trading_rule_v014_aggressive_tuned_0062e.json` 回测后，图中出现大段下跌未能及时开空的问题。

从 `decisions.csv` 可见典型症状：

```text
价格连续下跌；
edge 多数没有跌破做空阈值；
pred_cum_ret_5 经常仍为正；
系统保持 HOLD_NO_ENTRY；
短暂开空后，若 pred_cum_ret_5 转正，会很快 CLOSE。
```

当前空头开仓条件过度依赖模型：

```text
edge <= -open_edge_threshold
p_down >= open_prob_threshold
p_flat <= open_flat_max
pred_cum_ret_5 < 0
p_risk <= risk_open_max
```

问题判断：

```text
0062e 擅长累计收益排序，但并不能稳定识别所有下跌趋势起点；
交易模块不能只等模型给出完整空头确认；
需要加入价格行为 fallback 和空头持仓滞后退出。
```

---

## 3. 改造目标

014a 目标：

```text
在不破坏 014 风控边界的前提下，
减少大段下跌中的空仓时间，
降低下跌趋势中错误开多概率，
让系统能够用小仓位试探性跟随确认后的下跌趋势。
```

不追求：

```text
不要求每个下跌起点都精准开空；
不放宽最大仓位 20%；
不改变固定 20 倍杠杆；
不绕过 p_risk 风控；
不把价格规则变成无限制反向交易器。
```

---

## 4. 总体方案

新增三类能力：

```text
1. TrendRegimeFilter：价格趋势确认层
2. DownsideProtection：下跌保护规则
3. ShortExitHysteresis：空头退出滞后规则
```

模块位置建议：

```text
trading_system/
  trend.py          # 新增：价格趋势与下跌结构判断
  rules.py          # 修改：接入趋势上下文，新增 probe short / 禁止开多
  risk.py           # 小改：保持风控优先
  engine.py         # 修改：每根 bar 先计算 trend_context 再决策
  logger.py         # 修改：记录 trend_context 与 fallback 触发原因
  config.py         # 修改：新增 trend/protection 配置
configs/
  trading_rule_v014a_downside_protection_0062e.json
examples/
  backtest_trading_system_v014a.py  # 可选；也可复用 v014 入口加 --config
```

## 5. 新增模块：`TrendRegimeFilter`

### 5.1 文件

新增：

```text
trading_system/trend.py
```

### 5.2 输入

每根 bar 至少需要：

```text
close
high
low
atr
历史 close 序列
历史 high / low 序列
```

### 5.3 输出对象

建议定义：

```python
@dataclass
class TrendContext:
    is_downtrend: bool
    is_strong_downtrend: bool
    is_uptrend: bool
    trend_score: float
    ret_3_atr: float
    ret_6_atr: float
    ema_fast: float
    ema_slow: float
    breakdown_low_n: bool
    lower_high_low: bool
    reason_codes: list[str]
```

### 5.4 下跌趋势判断

建议第一版使用简单、可解释、无未来函数的规则。

`is_downtrend` 满足任意两项：

```text
ret_6_atr <= -1.2
close < ema_fast
ema_fast < ema_slow
close < rolling_low_12
最近 6 根 high/low 呈 lower-high / lower-low 结构
```

`is_strong_downtrend` 满足任意三项，或：

```text
ret_6_atr <= -1.8
且 close < ema_fast
```

指标定义：

```text
ret_3_atr = (close[t] - close[t-3]) / atr[t]
ret_6_atr = (close[t] - close[t-6]) / atr[t]
ema_fast = EMA(close, 12)
ema_slow = EMA(close, 36)
rolling_low_12 = min(low[t-12:t])
```

注意：

```text
rolling_low_12 只能使用当前 bar 之前或当前已收盘 bar；
不得使用下一根 K 线信息。
```

```text
但仍要求 p_risk 合格、p_flat 不过高、edge 不明显偏多。
```

## 7. 行为规则改造

### 7.1 下跌中禁止开多

触发条件：

```text
trend_context.is_downtrend == true
protection.block_long_in_downtrend == true
```

若模型满足 `OPEN_LONG_SIGNAL`，也必须拦截为：

```text
action = BLOCK
reason_code = BLOCK_LONG_DOWNTREND
blocked_by = trend
```

例外：

```text
若 trend_context.is_strong_downtrend == false
且模型 p_up / edge 极强，可暂不做例外，第一版先全部禁止。
```

实施位置：

```text
RuleEngine.decide()
或 RiskManager.validate_action()
```

推荐放在 `RiskManager.validate_action()`，因为它是对动作的二次拦截层。

### 7.2 下跌确认后的试探空单

当前为空仓时，若标准空头开仓条件不满足，但价格趋势确认下跌，允许触发 `OPEN_SHORT_PROBE`。

触发条件：

```text
position.side == FLAT
trend_context.is_downtrend == true
protection.allow_probe_short == true
p_risk <= probe_short_risk_max
p_flat <= probe_short_flat_max
edge <= probe_short_min_edge
p_down >= probe_short_min_p_down
未触发账户级熔断
不在冷却期
```

强下跌时可放宽：

```text
trend_context.is_strong_downtrend == true
edge <= 0.00
p_down >= 0.29
```

动作：

```text
action = OPEN_SHORT
reason_code = OPEN_SHORT_PROBE_DOWNTREND
position_ratio = probe_short_position_ratio
```

硬约束：

```text
position_ratio <= probe_short_max_position_ratio
position_ratio <= max_position_ratio
fixed_leverage = 20
```

第一版不允许 probe short 直接加仓。只有后续模型也转为空头确认，才允许走正常 `ADD_SIGNAL_CONFIRMED`。

### 7.3 空头退出滞后

当前规则：

```text
SHORT 持仓时：
edge >= reverse_edge_threshold 或 pred_cum_ret_5 > 0
立即 CLOSE
```

问题：

```text
pred_cum_ret_5 单根转正会过早关闭空头。
```

014a 改为：

```text
SHORT 持仓时，若 trend_context.is_downtrend == true：
  不因单根 pred_cum_ret_5 > 0 立即平仓；
  需要连续 short_exit_confirm_bars 根反向确认。
```

反向确认定义：

若 `short_exit_require_edge_and_cum == true`：

```text
edge >= reverse_edge_threshold
且 pred_cum_ret_5 > 0
```

若 `short_exit_require_edge_and_cum == false`：

```text
edge >= reverse_edge_threshold
或 pred_cum_ret_5 > 0
```

触发：

```text
short_reverse_confirm_count >= short_exit_confirm_bars
```

动作：

```text
CLOSE
reason_code = CLOSE_SHORT_REVERSE_CONFIRMED
```

重置：

```text
只要 edge < reverse_edge_threshold 或 pred_cum_ret_5 <= 0，
short_reverse_confirm_count = 0
```

注意：

```text
硬止损、灾难止损、p_risk 高风险退出、最大持仓时间退出不受滞后规则影响。
```

### 7.4 下跌趋势中 LONG 持仓的退出加速

若已有多头，且出现下跌趋势确认：

```text
position.side == LONG
trend_context.is_downtrend == true
```

触发：

```text
若 edge <= 0 或 pred_cum_ret_5 <= 0 -> CLOSE
否则 -> REDUCE 50%
```

原因：

```text
reason_code = CLOSE_LONG_DOWNTREND_CONFIRMED
reason_code = REDUCE_LONG_DOWNTREND_RISK
```

目标：

```text
避免大段下跌中模型仍偏多时继续持有多头。
```

## 8. 引擎接入方式

修改：

```text
trading_system/engine.py
```

主流程变更：

```python
trend_context = trend_filter.compute(history_bars, current_bar)
risk_event = risk_manager.check_pre_decision_risk(signal, portfolio, bar_index, current_price)
action = rule_engine.decide(signal, portfolio, trend_context)
action = risk_manager.validate_action(action, signal, portfolio, trend_context)
sized = position_sizer.apply(action, signal, portfolio, trend_context)
logger.record_decision(signal, action, portfolio, trend_context)
```

接口修改：

```text
RuleEngine.decide(signal, portfolio, trend_context)
RiskManager.validate_action(action, signal, portfolio, trend_context)
PositionSizer.apply(action, signal, portfolio, trend_context)
TradeLogger.record_decision(..., trend_context)
```

如果为了减少改动，也可以让 `trend_context` 作为可选参数：

```python
trend_context: TrendContext | None = None
```

## 9. 仓位设计

标准模型空头：

```text
OPEN_SHORT_SIGNAL 仍使用正常 sizing 逻辑，最高不超过 20%。
```

趋势试探空头：

```text
OPEN_SHORT_PROBE_DOWNTREND 初始仓位固定 5%；
若 strong_downtrend 可提高到 8%；
不可直接打满；
不可跳过风险过滤。
```

加仓条件：

```text
probe short 开出后，只有满足标准 ADD 条件，或者后续出现标准 OPEN_SHORT_SIGNAL 等价确认，才允许加仓。
```

禁止：

```text
因为价格继续下跌而无脑加仓；
因为浮亏而加仓；
因为单一 ret_6_atr 触发而满仓开空。
```

### 11.1 必跑对照

至少跑：

```text

experiment: v014a downside protection 0062e
```

输出目录建议：

```text
backtest/backtest_rule_v014a_downside_protection_0062e_test/
```

### 11.2 核心验收指标

必须满足：

```text
position_limit_violations == 0
risk_rule_violations == 0
max_margin_loss_ratio_observed < 1.00
trade_count 不得暴增到 baseline 的 3 倍以上
max_drawdown 不得明显恶化
```

建议目标：

```text
short_coverage_downtrend_ratio 提升
missed_downtrend_bars 下降
blocked_long_downtrend_count > 0
总收益不低于 baseline 的 80%
最大回撤不超过 baseline + 3 个百分点
```

注意：

```text
014a 首要验收是行为修复，不是单次收益最大化。
```

### 11.3 逐图检查

程序员需要输出带买卖点的图，并人工检查：

```text
大段下跌中是否仍长时间 FLAT；
下跌趋势中是否仍错误 OPEN_LONG；
probe short 是否过密；
空头是否被单根 pred_cum_ret_5 转正过早洗出；
硬止损和最大持仓时间是否仍生效。
```

## 13. 实施顺序

### 阶段一：只加趋势识别与日志

实现：

```text
trading_system/trend.py
engine.py 接入 trend_context
logger.py 输出趋势字段
```

不改变交易行为，只观察：

```text
大段下跌是否能被 trend_context 标出。
```

### 阶段二：禁止下跌中开多

实现：

```text
BLOCK_LONG_DOWNTREND
REDUCE_LONG_DOWNTREND_RISK
CLOSE_LONG_DOWNTREND_CONFIRMED
```

先减少错误多头。

### 阶段三：允许试探空单

实现：

```text
OPEN_SHORT_PROBE_DOWNTREND
probe short sizing
probe short 日志与统计
```

仓位从 5% 起步，不允许直接高仓位。

### 阶段四：空头退出滞后

实现：

```text
short_reverse_confirm_count
CLOSE_SHORT_REVERSE_CONFIRMED
```

验证是否减少“刚开空就被洗掉”。

## 14. 不建议做的事

不建议：

```text
直接把 open_edge_threshold 调到 0；
直接取消 pred_cum_ret_5 条件；
直接提高最大仓位；
用未来 N 根收益反推下跌标签参与规则；
让趋势规则绕过 p_risk；
把 probe short 和普通 short 混在日志里。
```

原因：

```text
这些做法会让回测看似改善，但失去可解释性和风险边界。
```

## 16. 结论

014a 的核心不是“让模型重新训练”，而是：

```text
用价格趋势确认层识别模型盲区；
用下跌保护规则阻止错误做多；
用小仓位 probe short 覆盖大段下跌；
用空头退出滞后避免单根模型噪声洗出；
用日志和指标验证行为确实改善。
```

第一版实现必须保守：

```text
先保护，再进攻；
先小仓位，再观察；
先行为正确，再优化收益。
```
