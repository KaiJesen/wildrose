# 0061 新结构训练目标指导

## 1. 文档目标

本文档面向模型训练师，基于软件设计师已经完成的 0061 架构调整，明确新结构下的训练目标、指标优先级、选模原则与下一轮训练计划。

本文只讨论模型训练、验证和指标判断，不涉及交易执行、下单、仓位或资金管理。

## 2. 当前架构状态

软件设计师已经完成 0061 结构调整，当前训练脚本默认进入：

```text
target_stage = return_direction_branch
report_dir   = reports/0061_market_state_return_direction_branch
checkpoint   = checkpoints/0061_market_state_return_direction_branch
```

新结构已具备以下能力：

```text
MarketStateOutput
  - return_pred [B,H]
  - direction_logits [B,H,3]
  - volatility_pred [B,H]
  - risk_logits [B,H,2]
  - cum_return_pred [B]
  - cum_direction_logit [B]
```

关键配置已经落地：

```text
use_cum_heads=True
use_horizon_return_head=True
detach_risk_vol_heads=True
return_weight=0.35
direction_weight=0.38
volatility_weight=0.10
risk_weight=0.07
cum_return_weight=0.15
cum_direction_head_weight=0.12
return_consistency_weight=0.03
cum_direction_weight=0.045
```

当前结构与 0060 的核心差异：

- 新增 `HorizonReturnHead`，用 horizon query + cross attention 预测逐 horizon return 和累计 return。
- 新增 `DirectionStateHead`，将逐 horizon 三分类和累计方向二分类分开。
- 新增 `cum_return_ic`、`cum_direction_head_acc`、`cum_direction_from_return_acc` 等评估指标。
- `detach_risk_vol_heads=True` 时，risk/volatility head 不再反向挤压共享 trunk。

## 3. 训练目标定位

0061 不是继续追求 risk/volatility 的增量，而是验证结构解耦是否解决 0060 暴露的两个核心瓶颈：

```text
1. cum_direction_acc / cum_direction_head_acc 无法稳定达标；
2. return_ic 长期低于目标，且 h1-h3 return IC 多次为负。
```

因此 0061 的首要目标是：

```text
验证 return/cumulative direction 专用分支是否有效。
```

不要把 0061 训练目标误解为“所有历史指标全面刷新”。如果新结构让 `risk_f1` 或 `volatility_mae` 小幅回落，但显著改善 `cum_return_ic` 与 `cum_direction_head_acc`，本轮仍然有价值。

## 4. 指标优先级

### 4.1 第一优先级

0061 选模首先看：

```text
cum_direction_head_acc
cum_return_ic
cum_direction_from_return_acc
return_ic
```

解释：

- `cum_direction_head_acc` 判断新增累计方向头是否有效。
- `cum_return_ic` 判断新增累计收益头是否有效。
- `cum_direction_from_return_acc` 判断逐步 return 累加后是否仍有方向价值。
- `return_ic` 判断 horizon return head 是否改善逐步收益排序。

### 4.2 第二优先级

继续保持：

```text
direction_macro_f1
volatility_mae
risk_f1
```

解释：

- `direction_macro_f1` 用于防止 direction 三分类坍缩。
- `volatility_mae` 用于确认 detach risk/vol 后波动率任务没有失效。
- `risk_f1` 用于确认风险头仍保持可用。

### 4.3 第三优先级

以下指标作为诊断，不作为单独选模依据：

```text
loss
return_mae
cum_return_mae
step_cum_return_gap_mae
direction_acc
risk_precision
risk_recall
```

## 5. 0061 最低验收目标

0061 的最低目标不是成熟模型目标，而是结构有效性目标。

```text
cum_direction_head_acc        >= 58%
cum_direction_from_return_acc >= 56%
direction_macro_f1            >= 0.34
return_ic                     >= 0.04
cum_return_ic                 >= 0.05
volatility_mae                <= 0.075
risk_f1                       >= 0.50
```

分布约束：

```text
risk_positive_rate_pred / risk_positive_rate_true in [0.8, 1.6]
direction_pred_down in [30%, 48%]
direction_pred_flat in [20%, 40%]
direction_pred_up   in [24%, 40%]
```

允许项：

- 如果 `risk_f1` 从 0060 的 `0.546` 回落到 `0.50 - 0.53`，但累计方向与累计收益指标显著改善，可以接受。
- 如果 `volatility_mae` 从 0060 的 `0.056` 回落到 `0.065 - 0.075`，但新分支指标达标，可以接受。

不允许项：

- `cum_return_ic <= 0.02` 时，不得判定新 return 分支有效。
- `cum_direction_head_acc < 56%` 时，不得保存为主线模型。
- `return_ic <= 0` 且 `cum_return_ic <= 0.05` 时，不得接受。

## 6. 0061 优秀目标

如果 0061 达到以下目标，则可以认为结构解耦成功：

```text
cum_direction_head_acc        >= 59%
cum_direction_from_return_acc >= 58%
direction_macro_f1            >= 0.35
return_ic                     >= 0.05
cum_return_ic                 >= 0.07
volatility_mae                <= 0.070
risk_f1                       >= 0.52
```

达到优秀目标后，下一轮才进入成熟模型目标：

```text
cum_direction_acc / cum_direction_head_acc >= 60%
return_ic >= 0.08
cum_return_ic >= 0.10
direction_macro_f1 >= 0.36
```

## 7. 选模原则

当前脚本已提供 `score_0061`，训练师应使用 `return_direction_branch` 阶段进行选模。

选模硬门槛：

```text
valid return_ic > 0.02
valid cum_return_ic > 0.02
valid cum_direction_head_acc >= 56%
valid direction_macro_f1 >= 0.33
valid risk_positive_rate_pred / true in [0.8, 1.6]
```

训练师需要特别注意：

- 如果 best model 是 fallback 模式保存，必须在报告中标记，不能直接视为主线可接受。
- 如果 valid 上 `cum_direction_head_acc` 高但 test 上回落明显，说明累计方向头可能过拟合。
- 如果 `cum_return_ic` 提升但 `return_ic` 仍为负，说明累计收益头有效但逐步 horizon return 仍需后续优化。


## 10. 训练过程观察点

训练师在每 5 个 epoch 或每次保存 best 时，应记录：

```text
valid return_ic
valid cum_return_ic
valid cum_direction_head_acc
valid cum_direction_from_return_acc
valid direction_macro_f1
valid risk_positive_rate_pred / true
valid step_cum_return_gap_mae
```

需要提前停止的情况：

```text
return_ic 连续 5 次 <= 0
cum_return_ic 连续 5 次 <= 0
cum_direction_head_acc 连续 5 次 < 54%
risk_positive_rate_pred / true > 1.8
direction_pred_up < 20%
```

需要保留但标记观察的情况：

```text
cum_return_ic 达标，但 return_ic 未达标
cum_direction_head_acc 达标，但 cum_direction_from_return_acc 未达标
risk_f1 小幅下降，但 risk 分布更健康
```

## 11. 结果判断规则

### 11.1 接受

满足：

```text
cum_direction_head_acc >= 58%
cum_return_ic >= 0.05
direction_macro_f1 >= 0.34
risk_f1 >= 0.50
volatility_mae <= 0.075
```

且至少满足以下一项：

```text
return_ic >= 0.04
或 cum_direction_from_return_acc >= 56%
```

### 11.2 条件接受

满足：

```text
cum_direction_head_acc >= 58%
cum_return_ic >= 0.05
```

但 `return_ic < 0.04` 或 `risk_f1 < 0.50`。

条件接受的用途：

- 可以证明新结构有效；
- 不能作为成熟模型；
- 下一轮需要继续调 return step head 或 risk/vol detach 策略。

### 11.3 拒绝

出现任一情况：

```text
cum_direction_head_acc < 56%
cum_return_ic <= 0.02
return_ic <= 0 and cum_return_ic <= 0.05
direction_macro_f1 < 0.32
risk_positive_rate_pred / true > 1.8
```

拒绝后不要回到单纯调参，应检查结构实现或训练初始化。

## 12. 0062 预案

根据 0061 结果，后续分三种情况：

### 情况 A：cum head 有效，step return 仍弱

表现：

```text
cum_return_ic >= 0.05
cum_direction_head_acc >= 58%
return_ic < 0.04
```

下一步：

- 保留 cumulative heads；
- 调整 horizon return head；
- 对 h1-h3 增加 horizon-specific return loss；
- 降低 return consistency weight，避免 step return 被 cum head 牵制。

### 情况 B：step return 有效，cum head 弱

表现：

```text
return_ic >= 0.04
cum_return_ic < 0.05
```

下一步：

- 提高 `cum_return_weight`；
- 检查 `cum_return_pred` 是否塌缩；
- 增加 `cum_direction_head_weight`。

### 情况 C：risk/vol 明显退化

表现：

```text
risk_f1 < 0.48
或 volatility_mae > 0.085
```

下一步：

- 关闭 `detach_risk_vol_heads` 做对照实验；
- 不要提高 risk weight；
- 判断退化是否来自 detach，而不是新 return/direction 分支。

## 14. 结论

0061 的训练目标是验证新结构是否解决 0060 的核心问题，而不是继续在旧指标上做权重交换。

训练师应聚焦：

```text
cum_return_ic
cum_direction_head_acc
cum_direction_from_return_acc
return_ic
direction_macro_f1
```

如果这些指标改善，同时 risk/volatility 保持可用，则说明结构调整方向正确。若这些指标仍无改善，则问题不再是训练参数，而应回到结构实现和目标定义继续排查。
