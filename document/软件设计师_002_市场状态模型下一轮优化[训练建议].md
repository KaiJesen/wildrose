# 软件设计师_002_市场状态模型下一轮优化[训练建议]

## 1. 结论先行

基于 `reports/0049_market_state_smoke` 与 `reports/0050_market_state_formal`：

- 当前多任务架构已证明有效（`return_ic` 转正、`return_mae` 与 `volatility_mae` 显著改善）。
- 当前阶段**不建议继续改模型结构**，应优先做训练策略优化与阈值/权重调整。

---

## 2. 现状问题归纳

1. `direction` 三分类指标偏弱：
   - `direction_acc=34.6%`
   - `direction_macro_f1=0.269`
2. `risk_f1` 提升有限（约 `0.455`），存在类别不平衡影响。
3. 多任务之间存在目标拉扯：
   - return/vol 明显改善
   - direction/risk 改善不足

---

## 3. 下一轮训练建议（0051）

## 3.1 标签阈值调整（仅 train 拟合）

- `direction_threshold_quantile: 0.35 -> 0.25`
  - 目的：减少 flat 类占比，增强方向监督信号。
- `risk_threshold_quantile: 0.80 -> 0.70`
  - 目的：提升 risk 正样本占比，缓解稀疏标签。

> 要求：阈值仍必须仅由 train split 估计，再应用到 valid/test。

## 3.2 多任务损失权重调整

建议配置：

- `return_weight=0.30`
- `direction_weight=0.50`
- `volatility_weight=0.10`
- `risk_weight=0.10`

设计动机：

- 当前 return/vol 已有改进，适当下调其权重；
- 提高 direction/risk 权重，优先补齐分类任务短板。

## 3.3 训练轮数与稳定性

- `epochs: 30 -> 60`
- 模型规模保持不变：
  - `d_model=128`
  - `n_heads=4`
  - `trunk_layers=2`

说明：

- 现阶段信号仍弱，先保证小模型收敛稳定，不建议盲目扩大参数量。

## 3.4 选模策略优化

建议替换当前综合分数为更偏向方向任务：

`score = 0.50*cum_direction_acc + 0.25*direction_macro_f1 + 0.15*return_ic + 0.10*risk_f1`

理由：

- 与当前业务目标更一致（先保证方向质量）；
- 降低单一 `return_ic` 主导选模的风险。

## 3.5 类别不平衡处理（训练层）

建议先后顺序：

1. 方向/风险 CE 增加 class weights（按 train split 统计）。
2. 若仍无提升，再尝试 focal loss（仅用于 direction/risk）。

## 4. 运行建议命令（示例）

以下参数用于下一轮正式训练（示意）：

```bash
python examples/train_market_state_model.py \
  --source binance_vision --symbol BTCUSDT --interval 1h --days 365 \
  --epochs 60 --batch-size 64 --device cpu \
  --direction-threshold-quantile 0.25 \
  --risk-threshold-quantile 0.70 \
  --return-weight 0.30 \
  --direction-weight 0.50 \
  --volatility-weight 0.10 \
  --risk-weight 0.10 \
  --checkpoint-dir checkpoints/0051_market_state_tuned \
  --report-dir reports/0051_market_state_tuned
```

## 5. 本轮验收目标（建议）

建议将以下作为 0051 的最低目标：

1. `cum_direction_acc >= 56%`
2. `direction_macro_f1 >= 0.30`
3. `risk_f1 >= 0.48`
4. `return_ic` 保持为正（或不显著劣于 0050）
5. `volatility_mae` 不显著退化

若以上 3 项核心目标（1~3）仍未改善，再考虑进入结构级改造（例如专门的 direction/risk 解码器分支）。
