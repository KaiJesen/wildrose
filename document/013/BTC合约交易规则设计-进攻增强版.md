# BTC 合约交易规则设计（进攻增强版）

## 1. 目标调整

- 当前观察：近 50 天收益约 3%，进攻性不足。
- 新目标：将 50 天目标收益提升到 `8%~15%` 区间（接受更高波动和回撤）。
- 风格切换：由“稳健优先”改为“收益优先 + 硬风控兜底”。

> 本文档是策略工程方案，不构成投资建议；高收益目标必然伴随更高回撤风险。

---

## 2. 信号不变，决策阈值改为进攻档

延续字段：

- `p_up`, `p_down`, `p_flat`, `p_risk`
- `pred_ret_1...pred_ret_5`
- `pred_cum_ret_5`
- `edge = p_up - p_down`
- `conf = abs(edge)`

### 2.1 开仓阈值（放宽）

**做多开仓（`FLAT -> LONG`）**

- `edge >= 0.05`（原 0.08）
- `p_up >= 0.39`（原 0.42）
- `p_flat <= 0.40`（原 0.34）
- `pred_cum_ret_5 > 0`
- `p_risk <= 0.45`（原 0.38）

**做空开仓（`FLAT -> SHORT`）**

- `edge <= -0.05`
- `p_down >= 0.39`
- `p_flat <= 0.40`
- `pred_cum_ret_5 < 0`
- `p_risk <= 0.45`

### 2.2 反手逻辑（更快）

- LONG 触发反手：`edge <= -0.04` 且 `p_down >= 0.40`
- SHORT 触发反手：`edge >= 0.04` 且 `p_up >= 0.40`
- 满足后直接平旧仓并开新仓，不等待二次确认。

---

## 3. 仓位与杠杆（进攻增强）

## 3.1 风险预算

- 单笔风险预算：`0.75%` 净值（原 0.40%）
- 单方向名义暴露上限：`2.8x`（原 1.6x）
- 最大杠杆：`5x`（原 3x）

## 3.2 动态仓位公式

- `conf_norm = min(conf / 0.16, 1.0)`（更快达到满仓）
- `risk_penalty = clip((0.55 - p_risk) / 0.25, 0.0, 1.0)`
- `pos_scale = 0.50 + 0.80 * conf_norm * risk_penalty`
- `target_notional = base_notional * min(pos_scale, 1.30)`

解释：

- 最低起始仓位更高（0.50），减少“信号对了但仓位太轻”。
- 信号强时允许超配到 `1.30x base_notional`。

### 3.3 顺势加仓

首次入场后，满足以下条件可加 1 次仓：

- 持仓方向正确且浮盈 > `0.6 * ATR(14)`
- 同向信号再次增强（`edge` 较入场提升至少 `0.03`）
- `p_risk <= 0.42`

加仓规模：

- 追加当前仓位的 `35%`
- 单笔总仓位不超过 `2.8x` 名义暴露上限

---

## 4. 止损止盈（进攻但不裸奔）

### 4.1 初始保护

- 初始止损：`1.4 * ATR(14)`（放宽，减少震荡扫损）
- 灾难止损：单笔浮亏达到净值 `-1.2%` 强平

### 4.2 分批止盈

- 触及 `1.2 * ATR`：减仓 30%
- 触及 `2.5 * ATR`：减仓 30%
- 剩余 40% 用追踪止盈：从浮盈峰值回撤 `1.0 * ATR` 平仓

### 4.3 时间止盈

- 若持仓满 `8` 根 1h 仍未到首档止盈，则按当前信号强度减仓到 50%。

---

## 5. 账户级熔断（必须保留）

- 日内回撤 <= `-3.0%`：当天停止开新仓
- 周内回撤 <= `-7.0%`：进入防守模式（仓位系数 * 0.45）
- 连续亏损 `4` 笔：冷却 `10` 根 1h K
- 连续 2 天触发日熔断：第 3 天仅允许 30% 试探仓位

---

## 6. 执行优先级

每根 1h 收盘后的执行顺序：

1. 先检查熔断与异常状态（交易所延迟、滑点异常、连接问题）。
2. 计算目标方向与目标仓位。
3. 若触发反手，先平仓再开新仓（同一根 K 内完成）。
4. 满足加仓条件则执行一次加仓。
5. 更新保护止损与追踪止盈价格。
6. 记录交易日志（必须包含滑点和手续费）。

## 7. 快速参数模板（可直接用于回测）

若沿用现有 `examples/sim_backtest_rule.py`，可先用以下进攻参数启动：

- `--theta-scale 0.10`
- `--target-vol 0.018`
- `--stop-atr 1.4`
- `--take-atr 2.5`
- `--loss-streak-limit 4`
- `--cooldown-bars 10`
- `--fee-bps 4 --slippage-bps 3`（回测别低估交易成本）

建议先跑：

1. 原稳健参数（作为基线）
2. 本进攻参数
3. 本进攻参数 + 资金费率过滤

对比维度：

- 50 天净收益
- 最大回撤
- 胜率与盈亏比
- 每日收益波动（标准差）

---

## 8. 两周上线节奏（进攻版）

- 第 1~3 天：纯回测 + 逐笔检查（重点看反手与加仓是否过度）
- 第 4~7 天：纸面仿真（实时信号，真实执行逻辑）
- 第 8~14 天：小实盘（最大暴露打 0.35 折）

放大条件（满足才加仓）：

- 近 7 天收益为正
- 最大回撤 > `-4%`
- 无风控违规（无跳过止损、无超杠杆）

---

## 9. 必须监控的三项风险

1. **过度换手**：阈值放宽后，交易次数可能上升，费用侵蚀收益。
2. **震荡反复打脸**：反手变快会在横盘期放大亏损，需要结合 `p_flat` 过滤。
3. **高波动滑点**：高杠杆下滑点影响成倍放大，必须把实盘滑点写进复盘。
